## 5. 风险与注意事项

### 5.1 批量文档迁移风险

风险：Git 状态看起来像大量删除，容易误提交或误删历史资料。

处理：每个 PR 只处理一个文档类别，并在 PR 描述列出旧路径到新路径映射。

### 5.2 outputs 产物迁移风险

风险：`outputs/` 可能是生成记录，不一定适合从原路径删除。

处理：PPT 单独决策；未确认前不提交删除。

### 5.3 发布验证污染风险

风险：在脏主工作区跑验证，无法证明最新 main 可发布。

处理：统一使用 `beauty-salon-admin-release-verify` 干净工作区验证。

### 5.4 历史引用失效风险

风险：历史文档迁移后，README、AGENTS、计划文档中的旧路径可能失效。

处理：每个 PR 搜索旧路径；必要时补充索引或更新引用。
