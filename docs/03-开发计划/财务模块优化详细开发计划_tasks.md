## 当前问题摘要

当前财务相关入口过多，用户在侧边栏看到的是多个报表和分析页，而不是业务闭环：

- 财务管理下已有：日结报表、月度结算、数字员工账单、提成规则、提成明细、员工人效、数字员工绩效、利润看板、商品毛利、项目毛利、会员卡储值履约、次卡履约、成本配置。
- 路由层分散在 `/finance/*` 与 `/operation-profit/*`，但菜单上又统一塞进“财务管理”。
- 提成、日结、利润、履约负债本质上是不同业务问题，现在被平铺展示。
- 提成结算状态存在口径风险：提成流水 `settled` 状态当前按员工 + 月份批量更新，未绑定具体结算单明细，容易出现“结算单金额”和“已结算流水金额”不一致。
- 数字员工绩效、数字员工账单、平台收入混在门店财务里，门店侧和平台侧视角不清。

---

### T4.2 修复提成结算单与明细绑定

当前问题：

- `CommissionSettlement` 只记录员工月份总额。
- 确认结算时按 `storeId + staffUserId + settleMonth` 批量将 `CommissionRecord` 改为 `settled`。
- 若结算单生成后又新增提成流水，确认旧结算单会把新流水误标为已结算。

推荐改造：

- [x] 新增结算明细关联表，例如 `CommissionSettlementRecord`：
  - `id`
  - `settlementId`
  - `commissionRecordId`
  - `amountSnapshot`
  - `statusSnapshot`
  - `createdAt`
- [x] 生成月结单时锁定当时纳入结算的 `CommissionRecord`。
- [x] 确认月结单时只更新关联表中的流水。
- [x] 月结单详情展示具体流水明细。
- [x] 若提成流水被修改，已生成但未确认的结算单需要提示“需重新生成”。

验收：

- [x] 已结算金额 = 已确认结算单绑定明细合计。
- [x] 结算单金额 = 结算单明细快照合计。
- [x] 结算后新增同员工同月份提成不会被旧结算单误结。
- [x] 已结算流水不能修改，除非撤销或冲正。

### T4.3 优化提成状态命名

建议业务状态：

- `待确认`：系统生成，财务尚未复核。
- `已确认`：财务复核通过，等待月结。
- `已入结算单`：已被某张月结单锁定。
- `已结算`：月结单已确认。
- `已发放`：月结单已支付。
- `已取消`：作废。

短期兼容：

- 保留数据库现有 `pending/confirmed/settled/cancelled`。
- 前端文案先调整，避免用户把 `confirmed` 理解成已经发钱。

验收：

- [x] 页面不再只显示“未结算/已结算”两个模糊状态。
