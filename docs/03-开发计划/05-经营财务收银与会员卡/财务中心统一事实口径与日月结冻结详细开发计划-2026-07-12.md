# 财务中心统一事实口径与日月结冻结详细开发计划

> **执行要求：** 按测试驱动方式逐项实现；真实数据库 migration/backfill、Git 提交、推送和 PR 必须另行授权。

**目标：** 将现有财务中心升级为可勾稽、可冻结、可追溯的经营财务中心，不建设会计总账。

**架构：** 保留订单、支付、退款、库存、提成和会员资产业务事实表；通过统一财务确认服务计算现金流、收入、成本和负债事实；日结、月度利润和会员负债使用不可变快照。门店财务管理员确认，只有 `super_admin` 可以填写原因后重开。

**技术栈：** NestJS、Prisma、PostgreSQL、React、TypeScript、Jest、Vitest。

---

## 阶段一：财务可信度止血

### 任务 1：日结快照、审计与状态机

- [ ] 为确认、重复生成、重开和版本查询编写失败测试。
- [ ] 新增 `DailySettlementSnapshot` 和 `FinanceAuditLog` Prisma 模型与 migration。
- [ ] 草稿允许刷新；已确认日结禁止重新生成并返回 409。
- [ ] 确认日结时创建不可变快照并写入当前用户。
- [ ] `super_admin` 使用 5–500 字原因重开，原快照保留，再确认生成下一版本。
- [ ] 为现有已确认日结提供版本 1 dry-run 回填脚本。

### 任务 2：日结页面拆分

- [ ] “日结总览”改为持久化日结单，展示状态、版本、确认人和待刷新状态。
- [ ] 实时指标改名“实时经营日报”，只提供重算预览。
- [ ] 财务首页指标显示实时、草稿、已确认、含估算或不可发布标签。

### 任务 3：门店权限隔离

- [ ] 新增 `FinanceRequestContext` 和统一门店范围校验。
- [ ] 所有财务按 ID 查询、修改、确认、发放接口验证对象 `storeId`。
- [ ] 平台收入接口统一使用 `core:platform-revenue:view` 并限制超级管理员。
- [ ] 增加跨门店 IDOR 测试。

### 任务 4：平台收入止血

- [ ] Ami 平台确认收入只统计 `confirmed/invoiced/paid`。
- [ ] 草稿 Ami 账单返回 `estimatedRevenue`。
- [ ] 供应链草稿不计入确认收入。
- [ ] `ltvEstimate` 兼容保留，同时新增 `annualizedRevenueEstimate`。

### 任务 5：退款精确冲销

- [ ] 按 `RefundItem.orderItemId/refundAmount` 精确冲减项目、商品或预收。
- [ ] `sale_return_in` 冲回商品成本，`service_consume_reverse` 冲回耗材成本。
- [ ] `refund_only` 不冲回库存成本。
- [ ] 已结算提成生成负向调整，未结算提成取消。
- [ ] 对账异常覆盖退款明细、库存动作和提成冲销。

## 阶段二：统一财务事实口径

### 任务 6：FinanceRecognitionService

- [ ] 新增只读 `FinanceRecognitionFact` 类型与服务。
- [ ] 统一支付、经营收入、预收、负债释放、成本、提成和经营费用事实。
- [ ] 事实包含 `businessDate`、`recognizedAt`、来源、质量和冲销关系。

### 任务 7：订单履约时间

- [ ] 为 `OrderItem` 增加 `recognizedAt` 和 `recognitionSource`。
- [ ] 服务任务转收银使用服务完成时间。
- [ ] 直接收银项目/商品使用支付完成时间。
- [ ] 历史数据按支付、出库、创建时间生成 dry-run 候选并标记质量。

### 任务 8：指标与利润门禁

- [ ] 平均订单金额按有效经营订单数计算。
- [ ] 消费客单价按整个区间去重顾客数计算。
- [ ] 每日利润使用日级收入和成本事实，不按区间利润率反推。
- [ ] 月度费用按实际日期或自然日分摊。
- [ ] 新增 `FinanceReadiness`，成本、提成、费用、退款或对账缺口存在时阻断利润发布。

## 阶段三：月结、负债与结算闭环

### 任务 9：月度利润快照

- [ ] 新增 `MonthlyProfitClose`，保存完整收入、成本、费用、利润和质量快照。
- [ ] 实现生成、确认、版本查询和平台重开。

### 任务 10：会员负债快照

- [ ] 新增 `MemberLiabilitySnapshot`。
- [ ] 支持 `asOfDate` 重建储值本金、赠送余额和次卡剩余履约价值。
- [ ] 分开披露现金合同负债和赠送权益义务。

### 任务 11：提成调整与付款凭证

- [ ] 新增 `CommissionAdjustment`。
- [ ] 已锁定提成禁止直接修改。
- [ ] `mark-paid` 仅接受已确认结算单并保存付款批次、方式、凭证和操作人。

### 任务 12：Ami 账单状态机

- [ ] 状态固定为 `draft → confirmed → invoiced → paid`，支持 `voided`。
- [ ] 只有草稿可重新生成；已确认账单通过作废和新版本更正。

- [ ] 同步管理端 API、类型、页面和财务口径文档。
- [ ] 运行 Prisma generate、后端定向测试、前端测试、双端 build、API 检查和 `git diff --check`。
