## 1. 阶段 0 结论

本阶段只完成开发准备，不进入代码实现。结论如下：

1. 已新建工作分支 `codex/operation-profit-phase0`。
2. 经营利润 MVP 先按“真实后端聚合 + 5 个管理端页面骨架 + 成本配置可写”推进。
3. 利润口径冻结为“现金收入、经营收入、毛利、经营利润、净利率”五层，不把办卡/充值直接算作利润。
4. 成本缺失、BOM 缺失、提成缺失必须显式返回和展示，不允许按 0 静默计算。
5. 权限采用新增经营利润权限码，不复用 `core:finance:view` 作为全部入口权限。
6. 后续阶段会触碰 Prisma、路由、菜单、权限和 API facade，属于高风险共享链路，必须分阶段小步提交。

## 3. 利润指标口径冻结

### 3.1 收入口径

| 指标 | 冻结定义 | 真实/估算 |
| --- | --- | --- |
| 订单实收 | 已完成订单的实际收款金额，扣除退款前先按订单维度聚合 | 真实 |
| 办卡收入 | 次卡/疗程卡销售产生的现金流 | 真实，但只作为现金流 |
| 充值收入 | 储值充值产生的现金流 | 真实，但只作为现金流 |
| 消课收入 | 会员卡核销对应的履约价值 | 估算优先，按卡售价/总次数折算 |
| 服务收入 | 单次服务订单收入 + 消课收入 | 部分估算 |
| 产品收入 | 商品订单明细收入 | 真实 |
| 退款金额 | 已完成退款金额 | 真实，退款归因不完整时需标记 |

### 3.2 成本口径

| 成本类型 | 冻结定义 | MVP 采集方式 | 真实/估算 |
| --- | --- | --- | --- |
| 耗材成本 | 服务项目消耗的产品和一次性耗材 | `StockMovement` 优先，BOM 标准成本兜底 | 部分估算 |
| 商品成本 | 零售商品出库成本 | `Product.costPrice * quantity` | 估算或真实，取决于成本价维护 |
| 直接提成 | 与服务/销售直接关联的提成 | `CommissionRecord` 优先，缺失则提示 | 部分真实 |
| 人工成本 | 美容师、店长、前台工资 | `OperatingCost` 月度手工录入 | 真实录入 |
| 房租物业 | 房租、物业、管理费 | `OperatingCost` 月度录入 | 真实录入 |
| 营销成本 | 广告、团购、达人、活动权益成本 | `OperatingCost` 月度/活动录入 | 真实录入 |
| 水电杂费 | 水电、布草、维修、办公耗材 | `OperatingCost` 月度录入 | 真实录入 |
| 折旧摊销 | 装修、仪器、设备月摊销 | `OperatingCost` 月度录入 | 估算录入 |
| 其他费用 | 其他经营费用 | `OperatingCost` 月度录入 | 真实录入 |

### 3.3 利润公式

```text
现金收入 = 订单实收 + 办卡实收 + 充值实收 - 退款

经营收入 = 单次服务收入 + 消课收入 + 产品销售收入

毛利 = 经营收入 - 耗材成本 - 商品成本 - 直接服务提成

经营利润 = 毛利 - 房租物业 - 固定工资 - 营销成本 - 水电杂费 - 折旧摊销 - 其他经营费用

净利率 = 经营利润 / 经营收入
```

### 3.4 必须显式标记的数据质量

| 标记 | 触发条件 | 展示要求 |
| --- | --- | --- |
| `complete` | 订单、成本、BOM、提成都有可用数据 | 可正常展示 |
| `estimated` | 消课收入、折旧、耗材等存在估算 | 展示“部分估算” |
| `missing_cost` | 缺少工资、房租、营销等月度成本 | 展示成本缺口 |
| `missing_bom` | 项目无 `ProjectBomItem` | 项目毛利显示“未配置 BOM” |
| `missing_commission` | 没有提成记录 | 展示“提成未计入/待补充” |
| `unavailable` | 经营收入为 0 或核心事实缺失 | 指标不可计算 |

| 角色 | 利润看板 | 项目毛利 | 会员卡履约 | 员工人效 | 成本配置 | 导出 |
| --- | --- | --- | --- | --- | --- | --- |

- `super_admin`
- `store_manager`
- `inventory_manager` 的项目毛利只读

## 5. MVP 范围冻结

### 5.1 MVP 必做

| 模块 | 必做内容 |
| --- | --- |
| 一级菜单 | 新增“经营利润”一级模块 |
| 路由 | 新增 5 个管理端路由 |
| 权限 | 新增权限码并接入角色权限 |
| 成本配置 | `OperatingCost` 表、CRUD、复制上月 |
| 利润看板 | 核心指标、收入结构、成本结构、趋势、异常提醒 |
| 项目毛利 | 项目收入、耗材成本、提成、贡献毛利、缺 BOM 提醒 |
| 会员卡履约 | 剩余次数、估算剩余金额、临期/沉睡/高余额风险 |
| 员工人效 | 美容师服务收入、服务次数、客户数、贡献毛利 |
| 数据质量 | `missingCostReasons` 与 `dataQuality` |
| 测试 | 后端聚合服务单测、前端 API facade 测试、构建与类型检查 |

### 5.2 MVP 暂不做

| 功能 | 暂缓原因 |
| --- | --- |
| 完整财务总账 | 超出经营看板范围 |
| 税务、凭证、个税 | 属于财务软件能力 |
| 自动工资计算 | 依赖完整提成/考勤体系 |
| 多门店健康评分 | 可在利润看板稳定后增强 |
| 活动 ROI 完整归因 | 依赖营销归因成熟度 |
| AI 自动问数 | 等聚合 API 稳定后接入语义层 |
| 生成跟进任务闭环 | 先保留入口，后续接任务系统 |
| 老板手机端 | 管理端完成后再做轻量端 |

### 5.3 首批验收顺序

建议按以下顺序验收，降低返工：

1. 成本配置能真实写入与查询。
2. 利润看板能用真实聚合返回核心指标。
3. 缺成本、缺 BOM、缺提成能被明确提示。
4. 项目毛利能定位到具体项目。
5. 会员卡履约能识别临期和沉睡风险。
6. 员工人效能按美容师聚合。
7. 菜单、路由、权限完整可用。

## 6. 后续阶段边界

### 阶段 1 进入条件

进入阶段 1 前应满足：

- 当前阶段 0 文档已确认。
- 工作分支已创建。
- 明确 `OperatingCost` 字段和成本分类。
- 确认是否需要 seed 演示成本数据。

### 阶段 1 不应处理

阶段 1 只做后端成本模型与成本配置 API，不应顺手改：

- 左侧菜单和路由。
- 前端页面大 UI。
- AI 问数。
- 终端经营摘要。
- 营销归因。

### 阶段 2 不应处理

阶段 2 做利润聚合 API，不应把所有明细查询都暴露给前端，也不应将核心利润公式放到前端。

## 7. 风险确认

| 风险 | 当前判断 | 处理方式 |
| --- | --- | --- |
| 共享文件冲突 | 后续会触碰高风险共享文件 | 分阶段提交，改前复查 `git status` |
| 利润公式争议 | 已按需求文档冻结基础公式 | 页面展示口径说明和数据质量 |
| 成本数据不全 | 中小店首期必然存在 | 缺失显式提示，不假装准确 |
| BOM 不完整 | 项目毛利会受影响 | 高频项目优先维护，缺失返回 `missing_bom` |
| 预收误算利润 | 经营风险高 | 现金收入与经营收入分开展示 |
| 权限过宽 | 财务数据敏感 | 新增独立经营利润权限，不复用财务查看权限 |
| 聚合性能 | 后续数据量上来后可能变慢 | MVP 用索引聚合，后续再考虑日报/月报聚合表 |
