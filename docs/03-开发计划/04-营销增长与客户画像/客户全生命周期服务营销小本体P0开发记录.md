# 客户全生命周期服务营销小本体 P0 开发记录

日期：2026-07-08

## 1. 本轮目标

围绕客户生命周期、服务机会和营销动作建设 P0 小本体。实现采用轻量持久化，不引入图数据库，不重做现有推荐系统。

完成口径：

- 客户生命周期阶段、阶段事件和客户机会可持久化。
- 智能推荐优先合并生命周期机会推荐卡。
- 客户详情和客户画像预测视角能展示生命周期阶段、机会和证据。
- 推荐卡继续承接活动草稿、自动规则草稿和终端跟进任务。

## 2. 已落地能力

### 2.1 数据模型

新增 Prisma 对象：

- `CustomerLifecycleSnapshot`：客户当前生命周期快照。
- `CustomerLifecycleEvent`：客户生命周期阶段初始化/变化记录。
- `CustomerOpportunity`：客户级机会。

新增迁移：

- `packages/server-v2/prisma/migrations/20260708090000_customer_lifecycle_ontology_p0/migration.sql`

### 2.2 后端服务

新增：

- `packages/server-v2/src/marketing/customer-lifecycle-ontology.service.ts`

核心逻辑：

- 从 `CustomerPredictionSnapshot`、客户资料、有效卡项、消费记录、`CustomerBehaviorEvent`、`CustomerAppEvent`、`MarketingAutomationTouch` 计算客户生命周期。
- P0 支持五类机会：`care_cycle_due`、`card_expiring`、`dormant_winback`、`coupon_claimed_unused`、`browse_abandonment`。
- 所有阶段和机会都输出结构化证据，避免只返回自然语言建议。
- 表未迁移时返回 schema pending，不让营销推荐和客户详情直接 500。

### 2.3 API

新增接口：

- `POST /marketing/lifecycle/rebuild`
- `GET /marketing/lifecycle/opportunities`
- `GET /marketing/lifecycle/customers/:id`

扩展接口：

- `POST /marketing/predictions/run`：预测完成后触发生命周期重建。
- `GET /marketing/recommendations`：合并生命周期机会推荐卡。
- `GET /customers/:id/profile`：返回 `lifecycle`。
- `GET /customers/profile-analytics/prediction`：返回生命周期阶段、机会标签和证据摘要。

### 2.4 前端

客户资料页：

- 在客户画像详情中增加“生命周期与下一步机会”模块。
- 展示阶段、LTV、流失风险、触达疲劳、机会标签和关键证据。

客户画像页：

- 在“预测视角”表格中增加生命周期阶段、机会标签和生命周期证据。

智能推荐页：

- 推荐卡识别 `customer_lifecycle` 来源。
- 展示生命周期证据、目标客户数和 P0 风险提示。
- 继续复用现有活动、自动规则和终端跟进承接动作。

## 4. 未纳入 P0

- 不做真实自动发券、自动群发、自动扣库存、自动改卡项或自动创建订单。
- 不新增独立本体管理页面。
- 不做项目级精确护理周期、耗材标准库、库存充足校验和排班产能校验。
- 不做完整统一归因总表，P0 继续复用现有触达、行为、推荐和订单归因对象。

## 5. 后续建议

P1 应优先补三件事：

1. 服务项目护理周期和项目耗材标准库，服务完成后自动生成下一次护理机会。
2. 推荐前校验库存和排班承接能力，避免推荐无法履约的项目。
3. 终端今日跟进直接消费后端生命周期机会，减少端侧启发式重算。
