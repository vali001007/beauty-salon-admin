# 客户全生命周期服务营销小本体 P1/P2 开发记录

日期：2026-07-08

## 1. 本轮目标

按一个大版本完成 P1/P2，但保持内部边界：P1 打通服务项目、护理周期、库存/产能校验和归因闭环；P2 补规则治理、质量快照和经营 Agent 周计划。

边界：

- 继续使用轻量持久化，不引入图数据库。
- 只提交 Prisma schema 和 migration，不默认执行真实数据库 migration。
- Agent 只生成经营计划和待审批动作，不自动发券、不自动群发、不改客户资产、不扣库存、不创建订单。

## 2. 已落地能力

### 2.1 数据模型

新增 Prisma 对象：

- `CustomerServiceCycleState`：客户-项目服务周期状态。
- `CustomerOpportunityFulfillmentCheck`：客户机会库存/产能承接校验。
- `LifecycleAttributionEvent`：生命周期归因事件。
- `CustomerLifecycleRuleVersion`：规则版本、发布和回滚。
- `CustomerLifecycleQualitySnapshot`：本体质量快照。
- `LifecycleBusinessPlan`：经营 Agent 周计划。

新增迁移：

- `packages/server-v2/prisma/migrations/20260708120000_customer_lifecycle_ontology_p1_p2/migration.sql`

### 2.2 后端服务与接口

扩展 `CustomerLifecycleOntologyService`：

- 从划卡记录、项目预约、项目订单和项目配置生成服务周期状态。
- 生成 P1 机会：`project_cycle_due`、`homecare_bundle`、`service_upgrade`、`project_idle_capacity`、`inventory_clearance`。
- 对机会写入库存/产能承接校验和风险。
- 汇总触达、预约、订单、库存流水等归因事件。
- 生成质量快照、规则版本和经营计划草稿。

新增/扩展接口：

- `POST /marketing/lifecycle/rebuild`：新增 `includeServiceCycles`、`includeFulfillmentChecks`、`includeAttribution`。
- `GET /marketing/lifecycle/service-cycles`
- `GET /marketing/lifecycle/opportunities/:id/fulfillment`
- `GET /marketing/lifecycle/attribution`
- `GET /marketing/lifecycle/quality`
- `GET/POST /marketing/lifecycle/rules`
- `POST /marketing/lifecycle/rules/:id/publish`
- `POST /marketing/lifecycle/rules/:id/rollback`
- `POST /marketing/lifecycle/business-plans`
- `POST /marketing/lifecycle/business-plans/:id/submit-actions`

### 2.3 前端与产品体验

客户资料页：

- 生命周期模块展示服务周期、下一次护理时间、库存/产能承接状态和归因事件。

客户画像页：

- 预测视角展示生命周期阶段、机会标签、下一次护理时间和承接风险标签。

智能推荐页：

- 推荐卡展示库存承接、产能承接和归因链。
- 页面顶部展示本体质量指标：字段覆盖率、规则命中率、归因完整率、可承接率。
- 保留活动草稿、自动规则草稿和终端跟进任务入口。

Ami Agent 页面：

- 新增“生成本周经营计划”按钮。
- 生成 `LifecycleBusinessPlan` 草稿并展示待审批动作。
- 提交审批后创建 `AgentRun/AgentApproval`，进入现有审批链路，不直接执行营销或库存动作。
