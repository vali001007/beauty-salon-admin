# 营销 H5（小程序）生成器阶段 2：商品/项目生成器开发计划

日期：2026-06-07  
关联方案：`docs/02-产品设计/营销H5小程序生成器方案.md`  
阶段目标：让商品和服务项目可以直接生成营销 H5/小程序推广页，并逐步抽象为通用 `MarketingPage` 生成器。

## 1. 阶段 2 交付边界

阶段 2 不再只服务“营销活动”，而是把生成器扩展到商品和服务项目两个高频推广对象。

第一轮开发采用“前端生成草稿 + 复用营销活动发布”的落地方式：

- 商品管理页支持“生成推广页”。
- 项目管理页支持“生成推广页”。
- 推广页草稿使用统一的页面 Schema 生成。
- 预览继续复用当前 `ActivityMiniPage` 手机预览组件。
- 发布先写入 `MarketingActivity.pageSchema`，用现有营销活动表承接小程序/H5 发布状态。

第二轮再落通用 `MarketingPage` 后端模型、页面版本、公开 H5 渲染和渠道事件表。

## 2. 详细开发拆解

### 2.1 商品生成器

| 任务 | 说明 | 状态 |
| --- | --- | --- |
| 商品列表增加入口 | 在 `GoodsProductManagement` 行操作增加“生成推广页” | 本轮开始 |
| 商品推广页草稿 | 自动带出商品名、图片、分类、品牌、规格、原价、优惠价、销售说明 | 本轮开始 |
| 商品页默认模板 | 生成 `hero`、`offer`、`benefits`、`product_recommendation`、`faq`、`notice`、`store_info` | 本轮开始 |
| 发布到营销活动 | 生成 `MarketingActivity`，保存 `pageSchema`、`recommendedItemsJson`、`aiGenerationId` | 本轮开始 |
| 商品渠道二维码 | 发布后基于分享域名生成链接，二维码下一轮接入 | 待做 |

### 2.2 项目生成器

| 任务 | 说明 | 状态 |
| --- | --- | --- |
| 项目列表增加入口 | 在 `ProjectManagement` 行操作增加“生成推广页” | 本轮开始 |
| 项目推广页草稿 | 自动带出项目名、图片、项目类型、时长、价格、门店 | 本轮开始 |
| 项目页默认模板 | 生成 `hero`、`offer`、`benefits`、`project_recommendation`、`faq`、`notice`、`store_info` | 本轮开始 |
| 预约 CTA | 项目页默认 CTA 为“立即预约” | 本轮开始 |
| 项目详情素材增强 | 后续补护理流程、适用肤质、禁忌说明、搭配商品 | 待做 |

### 2.3 通用生成器抽象

| 任务 | 说明 | 状态 |
| --- | --- | --- |
| 统一草稿类型 | 增加商品/项目通用营销页草稿类型 | 本轮开始 |
| Schema 构造器 | 把商品和项目转换为统一 `ActivityPageSchema` | 本轮开始 |

### 2.4 后端与公开 H5 下一步

本轮先不直接改后端脏文件，避免和当前工作区大量后端改动冲突。后续需要新增：

- `MarketingPage`
- `MarketingPageVersion`
- `MarketingPageEvent`
- `/marketing/pages/*` 管理端 API
- `/public/marketing/pages/:slug` 公开 API
- `packages/marketing-h5` 前台渲染应用

## 3. 本轮验收口径

本轮属于阶段 2 的启动开发，验收重点是“商品/项目可以产生可预览、可发布的页面草稿”：

1. 商品列表有“生成推广页”入口。
2. 项目列表有“生成推广页”入口。
3. 点击入口后弹出生成器，可看到默认标题、权益、描述。
4. 可打开手机预览，页面展示商品或项目内容。
5. 点击发布后调用现有营销活动创建接口，保存页面 Schema。
6. TypeScript 检查通过，至少不引入新的类型错误。

## 4. 产品影响

完成本轮后，店长不需要先手动创建营销活动，也可以直接从商品或项目发起推广页。交付上先打通“商品/项目 -> 页面草稿 -> 手机预览 -> 发布为活动页”的主链路，下一步再补真正的 `MarketingPage` 独立模型、公开 H5、二维码和渠道统计。
