# Ami Aura Lite 快捷功能与管理端匹配清单

更新时间：2026-06-18

## 对照口径

本清单按当前代码实现对齐：

- 智能终端快捷按钮来源：`src/config/aura.ts` 的 `AURA_ROLE_ACTIONS`，实际动作注册来源为 `packages/Ami-Aura-Lite-Kiosk/src/app/intent/commandRegistry.ts`。
- 终端快捷按钮会过滤 `business.query`，因此“问数”属于智能问答/命令区能力，不算底部快捷按钮。
- 管理端能力以 `src/app/components/Layout.tsx` 菜单和 `src/app/routes.tsx` 路由为准。
- “完全一致”表示管理端已有对应业务模块和核心数据闭环；不代表 UI 交互完全相同。终端更多是现场轻量执行，管理端更多是后台管理、列表、配置和复盘。

## 匹配状态定义

| 状态 | 含义 |
| --- | --- |
| 完全一致 | 管理端已有对应模块，核心业务对象、记录或写入能力能闭环承接。 |
| 字段查阅 | 管理端有相关模块，但终端只展示角色化摘要、风险字段或个人视角，管理端没有同款卡片。 |
| 部分一致 | 管理端可查最终结果或配置基础数据，但缺少终端的一步式现场操作。 |
| 管理端缺失 | 当前管理端菜单/路由未发现直接对应页面或操作入口。 |
| 终端专有 | 更偏设备、打印、现场交互或角色端自助，不建议简单搬到管理端。 |

## 交付判断

当前智能终端快捷功能与管理端的关系可以概括为：

- 后台主数据和后台管理能力基本覆盖：客户、预约、排班、库存、次卡、订单、提成等管理端都有对应模块。
- 现场执行能力由终端领先：收银、充值、打印、服务记录、护理建议、客户跟进待办更偏终端即时处理。
- 管理端主要缺“执行任务中台”：客户跟进任务、服务记录、充值/余额、收银班次/小票这些能力需要以管理端查询、审核、追溯和分派的形式补齐。
