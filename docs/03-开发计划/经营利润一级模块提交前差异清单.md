## 1. 清单目的

本清单用于提交、推送或开 PR 前做范围确认，避免经营利润模块和无关改动混在一起。

当前分支的核心交付是“经营利润一级模块”，但为了让商品毛利、项目毛利、提成、卡项履约和终端开单形成真实闭环，变更不可避免覆盖了订单、提成、卡项、终端和权限链路。提交前应按本清单确认每一类文件的业务目的。

## 7. 当前验证命令

提交前建议至少复跑：

```powershell
npm.cmd run check:operation-profit
npm.cmd run check:operation-profit:full
npm.cmd --prefix packages/server-v2 run operation-profit:confirmation-audit -- --storeId=6 --from=2026-06-01 --to=2026-06-30 --assigneeFile=<服务人候选正式确认JSON> --assigneeManualReviewFile=<无候选服务人正式确认JSON> --beauticianUserFile=<美容师账号绑定正式确认JSON> --staffUserFile=<员工账号创建正式确认JSON> --projectMasterFile=<项目档案正式确认JSON> --requireReady --summaryOnly
npm.cmd --prefix packages/server-v2 run operation-profit:confirmed-dry-run -- --storeId=6 --from=2026-06-01 --to=2026-06-30 --assigneeFile=<服务人候选正式确认JSON> --assigneeManualReviewFile=<无候选服务人正式确认JSON> --beauticianUserFile=<美容师账号绑定正式确认JSON> --staffUserFile=<员工账号创建正式确认JSON> --projectMasterFile=<项目档案正式确认JSON> --summaryOnly
npm.cmd run check:operation-profit:frontend
npm.cmd --prefix packages/server-v2 run operation-profit:scripts:typecheck
npm.cmd run check:operation-profit:build
npm.cmd run check:operation-profit:whitespace
```

`check:operation-profit:full` 已串行执行静态门禁、经营利润前端集合、经营利润 Prisma 脚本 TypeScript 检查、后端经营利润定向测试、前后端构建和 `git diff --check` 空白检查；后端测试固定使用 Jest `--runInBand`，避免 Windows 下 worker 并发触发 `spawn EPERM`；不要把该脚本改回并发测试。

当前经营利润前端测试集合已覆盖成本配置新增、编辑、删除、复制上月、只读权限、非法表单拦截、商品/项目毛利缺口标签、商品/项目毛利分页与筛选重置、会员卡履约风险、人效缺口、Real API 路径和权限/菜单/路由一致性；2026-06-20 最新结果为 7 个文件、35 个测试通过。

如要进入真实业务验收，还必须先完成目标库 migration、成本录入、业务确认包预检、服务人/项目档案/提成补齐，再重跑：

```powershell
npm.cmd --prefix packages/server-v2 run operation-profit:confirmation-audit -- --storeId=6 --from=2026-06-01 --to=2026-06-30
npm.cmd --prefix packages/server-v2 run operation-profit:readiness -- --storeId=6 --periodMonth=2026-06 --from=2026-06-01 --to=2026-06-30
```

注意：用户授权真实写库只代表允许执行写库命令，不代表候选业务事实已确认。`operation-profit-assignee-candidates.pending.json`、`operation-profit-beautician-user-bindings.pending.json`、`operation-profit-staff-user-create.pending.json`、`operation-profit-project-master-candidates.pending.json` 中仍带 `pending_business_confirmation` 的记录不能直接 apply；`operation-profit-confirmation-drafts/*.draft.json` 中仍带 TODO 的记录也不能用于真实写库。正式写库必须按 `docs/04-测试数据/经营利润业务确认后真实写库手册.md` 执行。
