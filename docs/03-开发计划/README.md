# 开发计划文档目录

本目录用于沉淀实施计划、任务拆解、验收记录、发布计划、接口/数据治理计划和项目协作说明。根目录只保留索引与一级主题目录，新增文档优先按主要交付对象放入对应目录。

## 一级目录

- `01-AI智能体与问数能力/`：Ami Agent、Agent V1/V2、AI 问数、经营语义中枢、知识图谱、能力中心、Agent 评测与智能问答相关开发计划。
- `02-AmiAura终端与数字员工/`：Ami Aura Lite、终端管理端、终端客户选择、数字员工、终端数据查询与终端闭环治理。
- `03-AmiGlow小程序与营销H5/`：Ami Glow、小程序、H5 版本、MarketingPage、公开 H5、二维码渠道与 H5 验收/部署说明。
- `04-营销增长与客户画像/`：智能营销、推荐、客户画像、权益匹配、自动营销规则、客户触达和营销闭环。
- `05-经营财务收银与会员卡/`：经营利润、财务系统、收银、套餐优惠分摊、会员卡划扣、提成和成本核算。
- `06-行业数据供应链与库存/`：行业数据平台、供应链平台、库存管理、行业标准品、采购、BOM 与销售链路。
- `07-管理端前端与Mock治理/`：管理端工作台、前端任务、管理平台性能、本地 Mock 退役、vendor-src 边界治理。
- `08-数据接口测试与治理/`：API/Terminal 契约、测试门店、模拟数据、数据补齐、数据完整性、接口收敛和数据治理计划。
- `09-Git发布与项目治理/`：GitHub/Gitee 发布、PR 拆分、分支提交、主工作区清理、版本变更、生产计划和协作参考信息。
- `10-排班与人效/`：美容师排班、一键智能排班、需求热力图、排班策略和人效相关计划。
- `99-通用规划与待确认/`：暂时无法归入明确模块的通用规划、商业计划或历史任务文件；确认业务归属后再迁入正式主题目录。

## 归档规则

- 新增实施计划、任务卡、验收记录和发布计划不要直接放在根目录。
- 产品方案和 PRD 优先放到 `docs/02-产品设计/`；真实测试数据、验收输出和脚本报告优先放到 `docs/04-测试数据/`。
- 一份计划跨多个模块时，按主要交付对象归档；相关模块在文档内用“关联范围”说明。
- `99-通用规划与待确认/` 只作为临时缓冲区，不作为正式开发计划沉淀位置。
