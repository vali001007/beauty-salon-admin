# 库存规格、包装与扣耗单位口径

日期：2026-07-02

## 1. 结论

短期口径：

- 产品档案拆分为 `specQuantity`、`specUnit`、`packageUnit`。
- `specQuantity + specUnit` 表示包装内含规格，例如 `30 ml`、`10 片`。
- `packageUnit` 表示采购、销售、入库常见包装，例如 `瓶`、`盒`、`包`。
- 库存主数量暂不切换到最小规格单位，仍沿用当前系统数量口径。
- 服务 BOM 和服务扣耗页面展示 `specUnit`，例如 `ml`、`片`。
- 商品销售页面展示 `packageUnit`，例如 `瓶`、`盒`。
- 历史库存流水只做巡检和标记，不自动批量改单位。

中期口径：

- 增加包装换算能力，例如 `1 瓶 = 30 ml`、`1 盒 = 10 片`。
- 支持库存按最小规格单位管理，销售/采购按包装单位展示。
- 只有换算字段、批次余量和页面提示全部补齐后，才迁移历史库存主数量口径。

## 2. 当前真实数据结论

来自：

- `docs/04-测试数据/product-unit-consistency-audit-2026-07-02.md`

结论：

- 47 个有效产品的规格数量、规格单位、包装字段已完整。
- 3 条 BOM 明细仍使用包装单位，需要修正为产品规格单位或补充包装换算规则。
- 257 条历史库存流水单位缺失或与规格单位不一致，暂不自动修复。
- 抽样 200 条商品销售明细缺少明确包装/规格口径证据，后续订单 payload 应补写销售单位。

## 3. 页面与接口用词

产品管理：

- `规格数量`：数值，例如 `30`。
- `规格单位`：最小规格单位，例如 `ml`、`片`。
- `包装`：销售/采购包装，例如 `瓶`、`盒`。

BOM 与服务扣耗：

- 使用 `规格单位`。
- 页面避免显示旧字段名 `单位`，如必须保留，应标注为“扣耗单位”。

采购与入库：

- 使用 `包装` 展示采购件数。
- 库存流水写入当前系统数量口径对应单位；在包装换算未上线前，不把 `1 瓶` 自动拆成 `30 ml`。

商品销售：

- 使用 `包装` 展示销售数量。
- 订单明细 payload 后续应补写 `packageUnit`，用于区分销售展示单位和服务扣耗单位。

## 4. 待处理事项

1. 修正 3 条 BOM 单位异常，或为这些产品补充包装换算规则。
2. 新增订单明细销售单位字段写入，至少在 payload 中保留 `packageUnit`。
3. 新增库存流水写入门禁：新流水必须带单位。
4. 包装换算字段上线前，不批量改历史 `StockMovement.unit`。
