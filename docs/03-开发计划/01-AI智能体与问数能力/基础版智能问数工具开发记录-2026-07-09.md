# 基础版智能问数工具开发记录（2026-07-09）

## 交付定位

本次交付的是基础版智能问数能力验证工具，目标是验证“口语问题 -> 找到正确业务表 -> 只读查询 -> 摘要、表格、来源”的闭环。它不是旧 Agent 版本升级，也不接入旧 Agent 治理、权限审批、运行审计、记忆归档或自动化发布体系。

## Clean-room 边界

- 新增后端模块：`packages/server-v2/src/ask-data/`。
- 新增前端入口：`/ask-data`，菜单位置为“工作台 / 智能问数”。
- 不复用旧 `agent`、`agent-v2`、`agent-v3`、`agent-v4`、`agent-v5`、`business-query`、`semantic-query`、`semantic-sql` 的实现、类型、模板或 SQL 方案。
- 仅复用通用工程基础设施：NestJS、Prisma、现有 `AiService.chat` 模型调用能力、Axios API client、React 路由和当前后台 UI 组件。
- 不新增数据库表，不新增 migration，不开放自由 SQL。

## 首版覆盖表

语义目录固定覆盖以下经营核心表：

- `Customer`
- `ProductOrder`
- `OrderItem`
- `PaymentRecord`
- `Reservation`
- `Project`
- `Beautician`
- `Product`
- `StockMovement`
- `DailySettlement`
- `OperatingCost`
- `CommissionRecord`
- `CustomerCard`
- `CardUsageRecord`

首版实际可执行模板：

- 项目收入：`ProductOrder + OrderItem`
- 库存预警：`Product`
- 客户近期消费：`Customer + ProductOrder + OrderItem`
- 预约取消率：`Reservation`

## 验收问题

- “上个月收入按项目看”：返回项目维度收入表格，并声明来源为 `ProductOrder` 和 `OrderItem`。
- “库存低于安全库存的商品有哪些”：返回低库存商品表格，并声明来源为 `Product`。
- “张三最近消费了什么”：先定位客户，再返回最近订单和消费明细，并声明来源为 `Customer`、`ProductOrder` 和 `OrderItem`。
- 多个客户匹配“张三”时：返回追问和候选客户，不强行猜测。
- “本月预约取消率是多少”：返回预约总量、取消量和取消率，并声明来源为 `Reservation`。
- 支持问题但没有记录时：返回 `no_data`，不编造数据。
- 不在首版范围的问题：返回 `unsupported`，提示基础版暂未支持。

## 未覆盖范围

- 暂不支持全库自由问数。
- 暂不支持写操作、自动执行动作、权限审批和治理审计。
- 暂不支持跨任意多表自发推理，只支持固定受控模板。
- 模型只负责尝试输出受控查询计划；实际执行仍由服务端 Prisma 模板完成。
