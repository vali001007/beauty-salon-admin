# Ami Brain P9-P10 最终验收与发布回滚演练记录

生成日期：2026-07-12

## 验收结论

P9 首批真实操作、主动巡检、预测解释与 P10 治理发布、多端复用、性能门禁已完成阶段验收。Ami Brain 已从“只回答”升级为“可预览、可确认、可执行、可回执、可巡检、可评测、可发布、可回滚”。

## P9 真实能力

- 动作：创建/改期/取消预约、客户跟进任务、采购单草稿、营销触达草稿、服务记录。
- 安全：动作执行重新校验门店、用户、权限、对象状态和确认有效期；幂等键唯一，重复确认返回同一回执。
- 巡检：高价值客户沉睡、爽约异常、毛利下降、库存临期/缺货、活动低转化、员工产能空档；两次真实运行均为 57 条发现，开放发现保持 57，去重生效。
- 预测：查询 `CustomerPredictionSnapshot`，返回模型版本、生成时间、流失/复购/响应/LTV、原因和适用边界；过期或缺失快照明确拒答。

## P10 治理与多端

- 语义、角色、技能和巡检资源采用不可变版本；直接激活被禁止，只能通过 release 门禁。
- 异步评测保存逐题回答、引用、grader、失败簇和延迟；空评测不能发布。
- 管理端、Ami Aura Lite Kiosk、`packages/app` 统一使用 `/api/brain/*` 会话协议。
- Kiosk 将 Brain 回答映射到既有 Agent UI 合同，动作按钮调用 Brain 确认/拒绝接口，反馈写入 Brain 反馈队列。

## 性能与自动化

- 真实 SSE 首字：20 次请求平均 1.715 秒，P95 2.547 秒，最大 2.553 秒。
- 完整 650 题请求：平均 3.243 秒，P95 5.272 秒，最大 16.516 秒；该指标包含会话、消息、Run、上下文和记忆落库。
- Brain 全套：49 suites、589 tests 通过，包含安全对抗、跨店、roleHint、动作幂等、巡检、预测和发布回滚。
- Playwright：桌面聊天闭环、治理台切换、390px 移动宽度共 3/3 通过。

## 未关闭风险

- 完整请求完成耗时受远程数据库网络波动影响，需继续做连接池与部署地域优化；用户感知首字已达到 3 秒门禁。
- 真实业务写入的浏览器验收未在生产数据上执行，避免无额外授权创建预约或业务单据；Service 集成与幂等测试已覆盖。
- 巡检真阳性率需要试点人工标注集验证，当前已完成规则执行、落库、去重和处置反馈链路。
