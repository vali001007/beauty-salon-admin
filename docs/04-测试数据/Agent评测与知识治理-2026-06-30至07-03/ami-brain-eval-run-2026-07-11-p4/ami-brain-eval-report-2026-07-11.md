## 产品结论与改进建议

1. 当前 Ami Brain 主链路已能真实落库和执行指标问数，但覆盖面仍停留在指标问答 MVP。
2. 六大能力要变成可用产品，下一阶段必须补角色意图路由和 domain adapter，不能只扩关键词。
3. 对 650 题中“经营概览、收入对账、客户管理、员工管理、前台预约、库存采购、营销增长、财务风控”分别建立可执行技能，而不是统一返回通用提示。
4. 对已识别但未接入真实口径的指标，继续保持拒答策略；这是比返回 0 更安全的产品行为。
5. 后续评测门禁建议采用：可用率、带 citation 比例、拒答正确率、写操作确认率、敏感/注入拦截率五个指标并行。
