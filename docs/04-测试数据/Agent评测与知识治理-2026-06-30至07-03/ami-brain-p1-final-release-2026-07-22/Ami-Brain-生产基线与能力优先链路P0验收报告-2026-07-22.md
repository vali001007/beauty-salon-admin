# Ami Brain 生产基线与能力优先链路 P0 验收报告

日期：2026-07-22

## 一、结论

本轮完成了五项生产 P0 的代码重建：生产不再允许回退到规则空壳，数据库 release 成为生产运行唯一事实源，能力目录先于意图编译参与决策，时间/对象/指标/条件/答案形态进入确定性校验，真实错题及同义表达进入发布门禁。

代码与构建门禁已通过。生产发布尚未完成，因此当前结论是“代码具备上线条件”，不是“生产已经切换”。旧 Release 3 在新版本上线后会被判定为无效生产基线，必须紧接着创建、评测并激活新的 model release。

## 二、修复范围

### 2.1 真实生产基线

- `BRAIN_RUNTIME_SOURCE=database` 为默认生产模式。
- 生产请求必须解析到有效 model release、非空能力目录和语义快照。
- 无 release、规则 release、空能力 release 分别返回明确的生产基线错误，不再静默走规则问答。
- 全量发布自动标记 `productionBaseline=true`，回滚只寻找此前有效生产基线。

### 2.2 语义快照一致性

- 创建 release 时按 skill 的 `definitionRefs` 生成 SHA-256 语义快照指纹。
- 激活、事务锁定和回滚时重新计算指纹。
- 指纹不一致、缺少 skill、运行模式不是 model/shadow 时阻断发布。

### 2.3 能力优先意图判断

新主链路为：

```text
用户问题
  -> 身份、门店、角色、权限与风险过滤
  -> 能力目录语义发现
  -> 模型语义意图编译
  -> 时间/对象/指标/条件/答案形态确定性校验
  -> 单一能力执行
  -> 证据与完成度校验
```

模型预算或供应商不可用时，只允许采用唯一、高置信且已治理的能力候选。该降级路径不是关键词路由，不会绕过权限和语义合同。

### 2.4 假成功归零

- 当前管理端/后台不具备的数据事实返回 `failed`。
- 能力边界问题返回 `failed + incomplete`。
- 失败答案不带业务 citation，不计入真实可用率。
- 发布门禁新增 `release_false_success` 类型，验证系统不会把缺口包装成成功。

## 六、生产发布步骤

1. 提交并推送本轮代码，等待 Zeabur 部署到准确 commit SHA。
2. 使用最新 41 个能力资源创建新的 evaluation-only shadow release；该 release 必须携带新语义快照指纹。
3. 运行完整发布门禁，目标约 100 题；要求产品失败 0、provider unavailable 0、假成功 0、缺失能力 0。
4. 以通过评测的 release evidence 创建 model canary release，执行商品排行、支付拆分、员工排行、预约文案、供应商缺口及安全问题 smoke。
5. canary 通过后激活 100% full release，并确认 `productionBaseline=true`。
6. 验证后续发布可回滚到该真实生产基线，不再允许回滚到规则空壳。

## 七、最终验收标准

- 生产 active release 为包含真实语义快照和 41 个能力资源的 model release。
- 当前错题及同义表达全部通过。
- 假成功率为 0。
- 时间误退化全量为 0。
- 文案、动作、排行、名单不误命中标量指标。
- 跨门店读取、roleHint 绕权和假动作确认全部为 0。
- 回滚目标是上一版真实生产基线，不是 rules 空壳。
