# Ami Brain 长期记忆治理收口报告

日期：2026-07-21

范围：长期偏好/显式决定的保存、召回、冲突、遗忘、审计和模型上下文注入

结论：P1 长期记忆原子任务已完成代码、接口、页面、测试和只读真实数据核对；未修改线上历史记忆。

## 一、原链路问题

1. 普通问题只要包含“决定/确定”等片段就可能被写成长记忆，造成错误事实固化。
2. 未验证的客户偏好可以从用户原话直接进入长期记忆，缺少客户身份和事实证据。
3. 长期记忆没有进入模型意图编译和规划上下文，模型实际不能依赖用户偏好完成规划。
4. 召回排序优先置信度，门店默认可能压过用户后续纠正。
5. 用户无法在会话中自然地查看、记住或忘记偏好；治理台也看不到版本历史。
6. 恢复旧记忆时未检查同一主体是否已有更新版本，可能重新激活冲突。

## 二、本轮实现

### 2.1 写入门禁

- 只有“请记住、以后、今后、设为默认、作为默认”等显式指令才生成长期记忆候选。
- 普通问句、裸“决定/确定”、短期经营数值、敏感字段和未验证客户画像全部拒绝写入。
- 个人记忆与门店共享记忆分开；门店共享写入/删除要求 `core:brain-governance:manage`。
- 同一作用域、类型和主体的新记忆会替代旧版本并留下 revision，不并行保留两个生效定义。

### 2.2 模型召回

- 长期偏好在每次模型规划前检索，并作为受治理 `longTermMemory` slot 注入语义意图编译器。
- 显式决定只在用户提出“照之前、上次决定”等问题时召回，普通问数不自动带入历史决定。
- 同一主体按“用户纠正 > 门店默认 > 模型推断”处理，最多注入 8 条已脱敏记忆。
- 召回写入 `long_term_memory_recall` trace，保存使用的 memory id 和数量。

### 2.3 用户控制和治理

- 支持自然语言“你记得我什么”“以后默认先说结论”“忘记我的偏好”等指令。
- 独立记忆指令直接返回保存/删除/列表结果和 memory citation，不用先完成无关业务查询。
- 新增 `GET /brain/governance/memories/:id/revisions`，按当前门店和用户范围读取版本历史。
- 管理端记忆治理页新增版本记录入口，可查看纠正、删除和恢复的内容、原因与时间。
- 恢复旧版本前检查同主体生效冲突，已有新版本时拒绝恢复旧记忆。
