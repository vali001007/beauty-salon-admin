# Ami Brain Release 325 多轮回归与缺口报告

生成日期：2026-07-19

## 一、结论

本轮没有启动 120 题和 650 题全量评测。先用 15 个真实多轮 P0 场景检查上下文继承、改口纠正、结果引用和动作边界，确认当前严格通过 `10/15`。

Release 325 仍为 evaluation-only，不进入 canary，不改变生产 active release。

## 二、已修复

1. P0 评测由“把两轮问题拼成一条文本”改为同一会话真实发送。
2. 支持时间改口、重新查询、相对周期比较和表达形式纠正。
3. 精确客户查询支持“客户马美琳，手机号后四位 6325”，第二轮可继承客户并回答最近服务项目。
4. “给她们发一条召回消息”不再作为普通文案处理，明确进入受控动作语义。
5. 动作缺少客户、项目、时间或唯一对象时返回结构化澄清，不再返回“当前无法完成查询”。
6. Supervisor 节点间数据改用受控 `mappingOutputs`，消除 `brain_observation_mapping_value_missing`。
7. 评分器将有 DB 依据的查无结果计为有效查询，并修正预约名单意图识别。
