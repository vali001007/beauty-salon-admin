## 正式版复用建议

1. V3 的受控 Text-to-SQL 适合作为事实型问数主链路，但必须继续加强语义路由、中文表头、日期格式、金额/小数标准化和问题意图校验。
2. V4 适合作为正式版经营动作层：负责生命周期机会、经营计划、审批草稿、归因复盘；事实查询仍复用 V3 的只读执行能力。
3. V2 的 Manifest、能力治理、发布门禁、dry-run、审计视图应作为“能力治理系统”保留，不建议继续作为用户自然语言主运行入口。
4. V1 可作为历史工具编排和角色路由参考，正式版不应继续扩展 V1 新能力。
5. Ami AI 只作为兜底话术和异常解释层，不应承担事实问数或经营决策。

## V3/V4 优先修复清单

1. 对“最受欢迎/最好/不足/风险/趋势”等高频业务词建立强语义路由，先定位业务对象，再生成 SQL。
2. 统一所有表格输出中文表头、中文日期、金额两位小数、数量两位以内展示。
3. 对 no_data 与 unsupported 做可解释诊断：说明缺哪个语义视图、字段、权限或数据。
4. 把用户“无用”反馈和本评测失败样本打通，形成自动诊断和修复建议池。
5. V4 需要把生命周期本体、营销计划、审批状态、执行归因和质量校验持续沉淀到统一运行审计。
