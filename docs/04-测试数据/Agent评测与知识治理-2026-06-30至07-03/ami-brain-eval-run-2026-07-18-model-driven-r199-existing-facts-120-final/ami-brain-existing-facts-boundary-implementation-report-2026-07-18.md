# Ami Brain 现有事实接入与外部缺口防误答实施报告

生成日期：2026-07-18

## 1. 本轮目标

只完成当前管理端和后端已有业务、功能与数据的 Ami Brain 接入；缺少后台事实的能力不继续开发业务模块，只登记外部任务并阻止相邻能力冒充回答。

## 2. 已完成实现

### 2.1 现有客户与营销事实

- 客户消费金额分层复用管理端 RFM M 值阈值。
- 优惠敏感客户按近 180 天订单和优惠使用比例筛选。
- 基础项目未升单客户按现有项目类型和订单事实识别。
- 疗程续购候选按活跃卡剩余次数和到期时间识别，仅生成候选。

### 2.2 现有员工与接待事实

- 员工周业绩下滑按当前期与上一同长度周期比较。
- 今日接待能力按待到店、可接待员工和服务超时影响判断。
- 项目级成本上涨缺少归因事实时明确拒答。

### 2.3 外部缺口防误答

在模型规划前拦截已经确认无统一后台事实的查询：投诉/满意度、等待流失、优惠授权、设备、储值提现、员工客户流失、服务事故/过敏、员工离职带客、消防检查。执行器层继续保留试用期、客户归属历史和项目成本归因边界。

所有缺口回答均满足：

- `status=completed`
- `grounding=none`
- `citations=[]`
- 具体 `unsupportedReason`
- trace 记录 `current_backend_fact_gap`
