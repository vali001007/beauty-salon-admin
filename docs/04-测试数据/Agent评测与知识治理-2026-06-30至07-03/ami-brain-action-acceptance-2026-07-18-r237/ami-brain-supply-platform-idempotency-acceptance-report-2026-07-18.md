# Ami Brain R237 供应链批量采购与收货幂等验收报告

## 结论

供应链平台已有管理端采购链路已完成业务表级强幂等：批量补货按供应商拆单、重复或并发创建、采购收货、丢响应重放和并发抢收均通过隔离 PostgreSQL 真实写入验收。

本轮没有新增管理端页面，没有执行真实采购、共享数据库或生产数据库写入。

## 已完成合同

| 合同 | 完成结果 |
| --- | --- |
| 批次根键 | 同门店补货批次使用哈希后的根幂等键，不保存原始键 |
| 供应商子键 | 每个供应商派生独立子键，同一批次只生成一张该供应商采购单 |
| 不可变指纹 | 门店、来源、商品、报价、数量和到货时间变化时拒绝复用旧键 |
| 事务拆单 | 映射解析、报价锁定和全部供应商子单在同一事务内完成 |
| 收货事实 | 新增 `ProcurementReceipt`，记录收货业务回执与库存副作用证据 |
| 并发保护 | 发货明细和商品行锁保证不同请求键也不能重复收同一余量 |
| 操作员审计 | 收货操作员只取当前登录用户，不接受请求体伪造 |
| 前端重试 | 现有管理端页面按业务参数保留失败请求键，丢响应后重试复用；参数变化自动生成新键 |

## 验证结果

- 供应链定向测试：`5 suites / 22 tests` 通过。
- 管理端 API 门面测试：`14/14` 通过。
- `server-v2` build：通过。
- 管理端 typecheck/build：通过。
- 全量后端 Jest：`327` suites 通过、`2` suites 失败；`3602` tests 通过、`5` tests 失败、`10` tests 跳过。
- 5 条失败均来自未修改的语义查询目录：缺少 `product_gross_margin_rate`、`staff_service_count` 指标，以及支付方式维度旧断言不一致。定向复现结果一致，当前工作区对 `semantic-data` 和 `semantic-query` 无本轮差异。

## 剩余边界

- 管理端尚未展示批次根键、供应商子单关联、重放状态和收货回执对账，继续作为后续管理端任务。
- 本轮只收口现有采购与收货能力，不建设真实采购审批外部系统、供应商支付或渠道通知。
- 营销真实发送、渠道回执和失败恢复仍是 Ami Brain 真实动作任务的主要未完成项。
