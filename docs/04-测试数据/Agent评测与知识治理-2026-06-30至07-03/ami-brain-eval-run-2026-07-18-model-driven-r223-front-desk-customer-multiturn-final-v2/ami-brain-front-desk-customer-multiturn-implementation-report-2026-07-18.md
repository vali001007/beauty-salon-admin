## 1. 结论

前台明确客户和多轮指代已从“依赖模型每轮复述客户”升级为“模型理解 + 服务端确定性实体继承”。R223 两组场景、六个轮次全部通过，供应商不可用和假阳性均为 0。

## 2. 修复内容

- 首轮姓名与手机号后四位联合定位后，将完整身份写入会话快照。
- 后续“她/这个客人”由服务端继承上一轮客户，不重新猜测。
- 无上下文代词必须澄清；补充姓名和手机号后四位后恢复原查询。
- 精确客户历史、卡项、来源、标签、备注和服务记录固定使用 `customer_facts`。
- 最近服务读取最近已完成 `ServiceTask`，不使用未来预约替代。
- `customer_facts` 补齐客户、预约、项目、美容师及对应维度合同。
- 评测器识别结构化澄清为成功执行和部分完成，不改变外部 API 状态合同。
