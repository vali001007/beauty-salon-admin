# Ami Brain 现有增长、库存与紧急风险能力实施报告

生成日期：2026-07-18

## 一、范围

本轮只接入当前管理端、后端和真实数据库已经具备的事实，不新增管理端页面、Prisma schema、migration、审批流程或真实写操作。所有需要新业务模块的数据缺口继续具体拒答，并登记到独立管理端/后端任务。

## 二、完成能力

| 问题 | 实现 | 回答边界 |
| --- | --- | --- |
| 有什么产品积压太久了 | 商品库存 + 在库批次记录年龄 + 出库速度形成积压候选排行 | 只称“积压候选”；无批次商品不参与；披露 30 天龄期和 180 天覆盖阈值 |
| 新客中哪些人最有潜力转成长期客户 | 最新完成预测批次中筛选近 90 天新客、到店不超过 2 次、复购评分不低于 70 | 明确这是预测候选，不承诺一定转化 |
| 有没有客户对某个项目特别感兴趣但还没办卡 | 汇总最近 90 天已绑定客户的项目浏览、预约行为，排除任何活跃卡 | 行为信号只代表运营候选，不等同购买承诺 |
| 今天有没有需要我马上处理的紧急事项 | 汇总预约接待、服务超时、退款/毛利、低库存和临期库存 | 缺少设备、消防、客户反馈和事故事实时明确披露，不推断为无风险 |

## 四、评测结果

### 4.1 定向评测 R203

4 个目标题全部 `usable_exact`：

1. 库存积压返回前 10 商品排行和批次/出库证据。
2. 紧急事项返回 1 项当前真实低库存风险，并披露未接入风险域。
3. 新客潜力返回 2 名预测候选和模型批次信息。
4. 项目兴趣未办卡返回 2 条绑定客户行为候选，并排除活跃卡。

## 五、人工审计与数据质量

- 库存积压答案粒度、排序和事实来源匹配问题。
- 紧急事项没有因设备、消防、反馈等缺失域而输出“无风险”。
- 新客潜力披露模型 `rules-v2.1` 和预测日期，不把评分当确定性事实。
- 项目兴趣严格排除活跃卡，但真实结果包含“H5真实联调客户-20260707”。这是客户主数据缺少测试/生产 provenance 标识，不是 Ami Brain 路由错误。
- 本轮不使用客户姓名黑名单过滤。后续必须在客户主数据和事件层统一提供 `dataProvenance/isTest`，由所有报表、指标和 Agent 共用。
