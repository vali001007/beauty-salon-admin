# Ami Brain 现有营销策略审批执行链路冻结报告

日期：2026-07-18

范围：仅复用管理端和后端已经存在的自动触达策略能力，不新增第二套营销发送系统。

## 一、交付结论

本轮完成了 Ami Brain 到现有营销自动化执行服务的审批式接入：用户要求执行一个已启用策略时，Brain 会解析当前门店策略、预览实时受众、展示影响范围和风险，并生成高风险确认动作。用户确认前不会创建发送任务；确认后调用现有 `MarketingService.executeStrategy`，继续复用现有受众计算、触达冷却、delivery job、发送回执和幂等机制。

本轮没有执行真实营销发送，没有激活生产能力版本。模型供应商在两次真实评测中均不可用，因此模型驱动发现该工具的真实链路证据尚未取得。

## 二、已完成能力

1. 新增可发现能力 `marketing_strategy_execute_preview`，角色范围为营销和店长，最小权限为 `core:marketing:update`。
2. 意图路由可识别“执行/启动自动触达策略”，不再误走普通文案或指标问答。
3. 只允许解析当前门店、已启用的策略；跨门店、停用策略、名称歧义和缺少目标均拒绝执行。
4. 审批预览展示策略名称、渠道、预计触达人数、策略状态和上次执行时间。
5. 审批载荷固定保存 `strategyId`、`strategyName`、`approvedAudienceCount`，不保存可被用户拼接的 SQL 或发送指令。
6. 确认执行前重新计算受众；若当前人数比审批人数增加超过 `max(10 人, 20%)`，拒绝执行并要求重新审批。
7. 执行网关强制使用 Brain action 幂等键，并调用现有 `MarketingService.executeStrategy`。
8. 回执区分进入队列、完成和部分失败，不把发送任务创建等同于客户已经触达。

## 五、真实评测边界

R247 和 R248 均使用 release 320，输入为“执行自动触达策略 AMI-DEMO-FULL 沉睡客户唤醒策略”。两次请求都在工具发现前返回 `provider_unavailable`，因此结论固定为：

1. 规则路由、adapter、审批和 gateway 的确定性代码与测试证据已通过。
2. 模型驱动工具发现的真实在线证据未通过。
3. 不能将本轮标记为真实发送闭环完成。
4. 供应商恢复后需要重跑该单题，再执行一条隔离/测试策略的人工确认验收。

## 六、产品进度影响

本轮把“已有管理端营销策略”接入了 Brain 审批入口，减少了管理端与智能体两套实现长期漂移的风险。但真实外部动作总任务仍未关闭，原因是模型路径未通过且没有执行测试策略的真实发送验收。

按 37 个一级交付任务统计，仍为 **32 项完成、2 项进行中、3 项未开始，整体完成度 86.5%**。
