# Ami Brain 合法候选子集诊断与新客双排行修复报告

日期：2026-07-18

基线 release：`316`

修复后 release：`318`

性质：evaluation-only，不激活生产

## 一、结论

本轮针对当前共享开发库中能够通过权限和源码新鲜度门禁的 23 张能力卡进行诊断。修复后 120 题真实可用率达到 `52/118 = 44.1%`，假阳性由 `1` 降为 `0`，安全绕过项全部为 `0`。

本轮证明“最近哪个时间段新客最多，从哪些渠道来”已能同时返回时间排行和来源排行，不再只回答渠道。该结果仅代表合法候选子集，不代表 Ami Brain 已达到正式发布条件。正式 release `315` 仍被共享开发库 9 条 pending migration 和 4 张能力卡权限未注册阻塞。

## 二、修复内容

1. 保留带有“最多、最高、排行”等明确排序线索的问题为 ranking intent，不再降级为普通 list。
2. 通用英文 Ontology 标签 `Customer/Staff/Product` 不再被误判为指定客户、员工或商品实体。
3. 新客来源能力在问题同时要求时间和渠道时，按自然周输出新客时间排行，并继续输出来源排行。
4. 结构化回答增加两个 ranking block，分别承载时间段和客户来源。
5. 回答标签增加 `timePeriod -> 时间段`，避免前端展示内部字段名。

## 六、剩余阻塞

1. R243 仍有 38 条 `metric_failed`。主要原因是共享开发库未应用当前 9 条 migration，Prisma Client 已引用新字段，但数据库 Reservation 等结构尚未对齐。
2. 仍有 6 条执行失败进入执行预算或重规划失败，需要在数据库对齐后复测，再区分真实编排缺陷和数据库连锁失败。
3. 28 条 `unsupported_intent` 中包含客户投诉、满意度、等待离店原因等管理端和后台尚未完成的数据能力；按当前任务边界只登记缺失，不在 Ami Brain 内另造业务事实。
4. release `318` 仅包含当前合法的 23 张能力卡，是诊断 release，不得替代正式全量 release。
5. 正式发布前必须完成共享开发库 `preflight -> deploy -> verify`，重新验证权限目录、release `315`/后续全量 release 和 120 题门禁。

## 七、产品进度影响

本轮关闭了一个真实假阳性并提高候选子集可用率，但没有解除数据库发布阻塞，也没有完成第 33 项“能力目录全量发布与门禁”。按 37 个一级交付任务统计，仍为 `32 项完成、2 项进行中、3 项未开始`，工程任务完成度为 `86.5%`。
