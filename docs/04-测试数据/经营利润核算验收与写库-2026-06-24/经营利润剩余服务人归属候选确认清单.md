## 5. 可转 JSON 的确认格式

业务确认后，将 `operation-profit:assignee-audit` 输出的 `candidateDraft.assignments` 复制为正式 JSON，并把每条已确认记录的 `confirmedBy` 从 `pending_business_confirmation` 改为业务确认人：

```json
{
  "assignments": [
    {
      "orderItemId": 1072,
      "beauticianId": 40,
      "source": "card_usage:428",
      "reason": "业务确认：同客户同项目消课",
      "confirmedBy": "业务确认人"
    }
  ]
}
```

执行前必须先 dry-run：

```powershell
npm.cmd --prefix packages/server-v2 run operation-profit:assignee-backfill -- --storeId=6 --from=2026-06-01 --to=2026-06-30 --file=<确认后的JSON文件>
```

确认 dry-run 无异常后，才允许写库：

```powershell
npm.cmd --prefix packages/server-v2 run operation-profit:assignee-backfill -- --storeId=6 --from=2026-06-01 --to=2026-06-30 --file=<确认后的JSON文件> --apply --yes
```
