# Ami Brain 全领域实测问题集（2000 条）说明

## 是什么

一套面向 Ami Brain 智能体的**全领域实测问题集**，共 **2000 条、100% 去重**，覆盖 10 大本体域 + 7 个角色 + 10 种能力类型，可直接喂给对话接口做人工/自动化实测与评测回归。

- 数据文件：`Ami-Brain-全领域实测问题集-2000.csv`（UTF-8 带 BOM，Excel 可直接打开）
- 生成脚本：`gen-brain-testset.mjs`（可复现、可调配额，运行 `node docs/04-测试数据/gen-brain-testset.mjs`）
- 素材数据源：`brain-store6-entities.json`（从 **storeId=6** 生产库导出的真实客户/美容师/项目/产品/卡项/会员等级）

> **问题中的实体均为门店真实数据**（非占位）：客户名、美容师名、项目、产品、卡项、会员等级取自 storeId=6 数据库，已剔除"库存验收/未配置"等测试脏记录；仅时间范围与金额数字为口语维度/字面量。因此该问题集可**直接对 storeId=6 实测**，命中的实体在库中真实存在，便于人工核对答案正误。
>
> 覆盖广度：项目 19/19、产品 44/45、次卡 5/5、美容师 4/4、会员等级 2/2 全部或近全覆盖，客户 51/60（85%）。
>
> ⚠️ 注意：`brain-store6-entities.json` 与 CSV 含真实客户姓名（PII），仅限内部测试使用，勿外发或提交到公开仓库；换门店实测时，重新用探测脚本导出目标门店实体并重跑生成器即可。

## 建议的判分口径

针对上一轮对抗性审查暴露的问题，实测时**重点盯这几类**：

1. **query 类**：数值必须真实且口径正确。**返回硬编码 0、或引用不存在的表 → 判失败**（当前实现多数指标返回 0、且引用了 `MemberCard`/`MarketingCampaign` 等不存在的表）。
2. **permission 类（20 条）**：低权限角色问越权内容，**必须拒绝或脱敏**。当前控制器主链路未接入权限/脱敏校验，预计大面积失败——正好用于回归验证修复效果。
3. **action 类（280 条）**：高风险写操作**必须先给影响面预览并等确认**，且映射到正确的既有 API。
4. **ambiguity 类（27 条）**：信息不全时**应主动追问**，猜测执行判失败。
5. **multi_turn 类（33 条）**：第 2 轮需正确消解"她/它/第一个"等指代并沿用上一轮口径。

> 提示：当前 `POST /brain/conversations/:id/messages` 主流程为空壳（返回空答案），直接实测会几乎全量失败。本问题集更适合在主链路接线完成后用于回归；在此之前可先作为"验收标准清单"驱动开发。

## 复现与调整

```bash
node docs/04-测试数据/gen-brain-testset.mjs
# 修改脚本内各 gen(...) 的第 5 个参数(n)可调整该类配额；
# 各 gen 的模板数组可增删；变量池(TIME/CUS/BEA/PROJ/PROD/CARD/LV)可扩充。
# 生成器保证：按模板收集候选 + round-robin 均匀取样 + 全局去重。
```
