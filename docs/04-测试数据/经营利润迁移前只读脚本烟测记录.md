执行原则：本轮只执行 read-only 或 dry-run，不写入真实业务数据。

## 1. 结论

迁移前脚本烟测通过“可运行、可解释、可阻断”的交付要求，但当前目标库不能进入业务验收。

原因：

- readiness 返回 `readinessStatus=blocked`，不是页面异常，而是启动限制正常拦截。
- 目标库缺 6 条经营利润发布必需 migration。
- 目标库缺 `OperatingCost` 表/列、`CommissionRule.userId`、`CommissionRecord.staffUserId`、`CommissionSettlement.staffUserId`、会员卡/客户卡操作人列。
- 目标库缺 `ProductOrder_storeId_createdAt_idx` 经营利润月度订单查询组合索引。
- 毛利数据检查因 schema 缺口被跳过，不能作为业务验收通过证据。
- 真实订单仍存在服务人归属、项目 BOM、实际耗材和提成记录缺口。
- 抽样复算可输出公式和缺口，但 `readySamples=0`、`samplesWithGaps=6`，只能作为缺口证据。

## 3. readiness 详细记录

执行命令：

```powershell
npm.cmd --prefix packages/server-v2 run operation-profit:readiness -- --storeId=6 --periodMonth=2026-06 --from=2026-06-01 --to=2026-06-30
```

结果摘要：

```text
mode=read-only
readinessStatus=blocked
pass=1
warn=0
fail=5
```

失败项：

- `required_prisma_migrations`：6 条经营利润发布必需 migration 均未应用。
- `schema_migration_columns`：经营成本、提成规则、提成记录/月结、会员卡/客户卡操作人等关键列缺失。
- `schema_migration_indexes`：缺 `ProductOrder_storeId_createdAt_idx`。
- `operating_cost_migration`：`OperatingCost` 表不存在。
- `margin_data_checks`：因 schema 缺口跳过毛利数据检查。

交付判断：readiness 阻断正确，当前不能进入页面性能验收、成本录入验收或毛利业务验收。

## 5. 经营成本 dry-run 记录

执行命令：

```powershell

```

结果摘要：

```text

schemaPrecheck.exists=false
```

交付判断：成本 seed 脚本会先通过 `information_schema` 做表存在性预检，迁移前可给出明确状态，不会在缺表时写入成本。必须先执行经营利润 migration，才能进入成本录入和利润总览成本扣减验收。
