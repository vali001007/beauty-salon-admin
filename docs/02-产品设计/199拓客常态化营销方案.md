# 199 拓客常态化营销方案

来源文档：`docs/02-产品设计/2_199拓客逻辑与拓客启动前期准备（新）.pptx`

更新时间：2026-06-24

## 1. 一句话结论

这份 PPT 讲的不是单纯卖一张 199 体验卡，而是把美容门店的获客、裂变、到店激活、客户建档、沉睡客唤醒和 2991 留客承接做成一套日常经营系统。

产品落地时，应把它定位为“拓客增长战役 + 转介绍裂变 + 到店激活 SOP + 留客转化承接”的组合能力，而不是普通营销活动页。

## 2. PPT 核心内容总结

### 2.1 为什么门店必须拓客

PPT 的核心判断是：美容院经营有三个关键指标，分别是业绩、顾客满意度和新客持续导入。门店天然会有客户流失，原因可能是搬家、换工作、怀孕、被其他门店吸引等，不一定都来自服务问题。因此门店不能只靠存量客户续费，需要一套常态化拓客留客系统。

拓客在 PPT 中被定义为：

- 一把手工程。
- 门店经营战略。
- 门店常态化工作。
- 美容师必备职业技能。
- 美疗师收入的重要组成部分。

产品含义：系统不能只服务市场部或老板看活动数据，还要服务店长、顾问、美容师每天执行。

### 2.2 拓客方式

PPT 中列出的主要拓客方式包括：

- 老必带新：老顾客必须推荐新顾客，才能激活 199 宠粉卡使用。
- 新带新：新客到店激活后继续生成自己的链接，推荐其他朋友。
- 蜂王裂变：识别高影响力、高分享意愿客户，重点激励。
- 沉睡顾客激活：对久未到店客户用真诚道歉、礼品和重新建联的方式唤醒。
- 团购顾客和外拓客户：作为补充客源。

产品含义：需要把客户来源、推荐链路、推荐人、被推荐人、激活状态和后续成交串起来，形成可追踪的裂变链。

### 2.3 国颜集团拓客优势

PPT 用三个关键词描述拓客系统的交付价值：

- 结果：首次拓客目标是基于现有常到店顾客数翻 2-3 倍。
- 标准：提供标准化接待流程、拓客流程和留客流程。
- 打造员工：让员工在拓客、接待、留客过程中形成能力和习惯。

另一个表达是“简单、易成交、易裂变、自循环”：

- 门店：获得现有顾客基数 2-3 倍的新客和持续进店。
- 员工：按每人每月 10 个新客、6 个月 60 个新客测算。
- 顾客：成功分享 3 人可获得 120 元红包和额外礼品，例如 598 元高端洗护套。

产品含义：系统需要支持目标分解、员工 PK、奖励核算、裂变排行和复盘，而不是只展示活动浏览量。

### 2.4 199 卡项设置逻辑

199 卡项设计原则是站在顾客角度，而不是站在门店销售角度：

- 老客一看就买。
- 老客分享给新客，新客一看就买。
- 不玩复杂数字游戏。
- 不做强套路。
- 卡项简单、超值、容易理解。

线上链接被强调为关键工具，因为老客带新客线下到店的概率不高，但线上购买不受时间和地点限制。

PPT 的三步路径是：

1. 老必带新：老客购买后，必须转介绍新客才能激活使用。
2. 新带新：新客到店后按标准流程接待、建档、预约，并继续转介绍。
3. 2991 留客：在 199 激活和服务过程中铺垫更高客单的 2991 卡。

产品含义：199 是入口产品，2991 是留客和转化产品。系统必须把低价拓客和高客单承接设计成同一个漏斗。

### 2.5 拓客前中后 SOP

拓客前：

- 填写调研表。
- 设置 199 卡项。
- 收集门店资料。
- 制作线上链接。
- 确认启动时间和爆卡时间。
- 开线上或线下启动会。

拓客中：

- 顾客盘点：有效顾客、常到店顾客、沉睡顾客。
- 全员目标：按门店顾客数、员工数、床位数拆目标。
- 准备物料：海报、地贴、展架、赠品、激活礼。
- 提前 3-5 天预热：朋友圈、老顾客微信、电话沟通。
- 爆卡当天集中执行：不接普通服务，集中卖老客、促分享、激活沉睡客。
- 当天总结：公布结果，生成喜报。

拓客后：

- 邀约新客到店激活。
- 制定年度、月度、每日、个人目标。
- 每天后台数据监测。
- 每晚群内发送业绩报表。
- 每周拓客复盘会。
- 推动门店转介绍，保底 1 家，目标 3 家。
- 继续新店对接。

产品含义：需要活动生命周期管理，分为筹备期、预热期、爆卡期、激活期、留客期、复盘期。

### 2.6 门店前期准备

PPT 要求门店在启动前完成：

- 条幅、展架、地贴等氛围物料。
- 文宣内容与 199 链接卡项内容保持一致。
- 门店资料提交给技术方制作 199 链接。
- 洗护套等赠品准备。
- 顾客详细盘点。
- 朋友圈和抖音预热。
- 员工五好文化表达训练。
- 199 新客进店接待流程训练。
- 练习、打卡、脱稿视频考核，持续 3 天。

五好文化包括：

- 行业好。
- 企业好或平台好。
- 老板好。
- 团队好。
- 品质好。

产品含义：要有启动清单、素材一致性校验、员工训练任务和考核记录。

### 2.7 顾客盘点与目标分解

PPT 给出两个核心表格：

- 老顾客盘点表：顾客姓名、分类、电话、顾客类型、关心项目、成交率、电话备注。
- 199 拓客目标分解表：床位数、门店年限、员工数、常到店顾客数、员工个人目标、未完成处理方式、实际达成、有效客人、沉睡客人。

顾客类型包括：

- 领袖顾客。
- 种子顾客。
- 股东顾客。
- 常到店顾客。
- 沉睡顾客。
- 99 元开业卡顾客。
- 流失顾客。

产品含义：系统应支持客户标签、顾客成交率预估、员工名下客户池、目标分解和进度追踪。

### 2.8 触达内容与话术体系

PPT 包含大量可沉淀为模板库的内容：

- 朋友圈每日 3 条预热内容。
- 三天倒计时内容。
- 给顾客发信息的三步骤：文字提醒、卡项图片、个人链接。
- 电话爆卡顺序：先高成交率老客，再按标记分层拨打，再打沉睡客、流失客和体验卡顾客。
- 老客爆卡话术。
- 老客跟进话术。
- 十大抗拒点处理话术。
- 新客电话邀约话术。
- 新客带参观话术。
- 老带新、新带新、转介绍话术。
- 沉睡客激活话术。
- 2991 铺垫话术。

产品含义：应做成“话术任务 + 模板库 + 触达记录 + 客户反馈标签”，而不是存在文档里。

### 2.9 新客激活流程

新客进店五件事：

1. 电话邀约，集中激活。
2. 带参观，讲五好文化。
3. 建档，反预约。
4. 做转介绍。
5. 铺垫 2991 卡。

新客资料表包含：

- 基础资料：姓名、年龄、生日、职业、婚否、电话、地址。
- 消费与生活习惯：家居品牌、美容院护理项目、医美项目。
- 面部护理习惯、养生护理习惯。
- 皮肤状态、护理建议。
- 身体情况、失眠、生理期、亚健康、疼痛部位、禁忌、特殊情况。
- 购买记录、金额、美容师、顾客签名。

产品含义：系统需要支持 199 新客建档表，并沉淀到客户画像、健康档案和后续推荐。

### 2.10 老带新机制

PPT 中的核心机制：

- 老顾客成功分享 1 位新顾客后，才激活宠粉卡服务。
- 新顾客可按月支付，减少一次性付费压力。
- 老顾客每成功介绍 1 位可获得 40 元红包。
- 推荐 3 位可获得 120 元红包和高端洗护套。
- 如果老顾客 3 个月内没有成功转介绍 1 位新顾客，可退还 199 元，降低顾客风险。

产品含义：需要支持购买、待激活、推荐达标、已激活、退款资格、奖励发放等状态机。

### 2.11 沉睡客激活

沉睡客激活的底层逻辑：

- 负责人是门店院长。
- 本质是真诚。
- 形式是道歉。
- 目标是重新建立信任和到店机会。

三步触达：

1. 第一次微信：老板身份真诚沟通，请顾客反馈问题，邀约到店重新认识。
2. 第二次微信：提醒是否看到信息，希望有机会见面。
3. 第三次电话：说明已发信息，希望再给一次合作机会。

产品含义：沉睡客唤醒应作为单独自动规则或跟进任务，不能只混在普通活动触达里。

### 2.12 2991 卡承接

2991 卡是 199 拓客之后的留客承接产品。PPT 要求到店时通过堆头展示、产品介绍、十大权益、分段支付和老顾客背书进行铺垫。

核心卖点：

- 创始会员卡大礼包。
- 一年可购买一次。
- 分段支付，满意继续支付，不满意停止支付。
- 老顾客每年人手一张。
- 通过产品价值、项目价值和品质背书，让顾客觉得 2991 是合理且低风险的升级选择。

产品含义：199 活动不能以“购买 199”为终点，必须追踪到 2991 预约、成交、分段支付和后续履约。

## 3. 产品定位

### 3.1 产品名称

建议命名：`199 拓客增长战役`

在系统中可归属到 `智能营销 > 拓客增长`，与现有营销活动、自动营销规则、智能推荐、营销 H5、终端跟进任务形成联动。

### 3.2 目标用户

- 老板：看拓客结果、员工执行、ROI、2991 转化和长期客基增长。
- 店长：配置活动、分配目标、盯执行、开复盘会。
- 顾问：打电话、跟进高价值客户、处理抗拒点、推动到店激活。
- 美容师：分享链接、执行新客接待、建档、转介绍、服务预约。
- 新老顾客：购买 199 卡、生成分享链接、查看奖励、预约到店。

### 3.3 核心目标

- 把门店一次性拓客活动变成可复制、可追踪、可复盘的系统化 SOP。
- 把老客分享、新客购买、到店激活、建档服务、2991 转化串成数据闭环。
- 把员工执行从微信群手工汇报升级为系统任务和进度看板。
- 把话术、朋友圈素材、抗拒点处理沉淀为可复用模板库。

## 4. 业务流程设计

### 4.1 全流程阶段

| 阶段 | 业务目标 | 系统能力 |
| --- | --- | --- |
| 筹备期 | 明确卡项、目标、物料、员工训练 | 活动配置、卡项配置、启动清单、员工训练任务 |
| 预热期 | 老客触达、朋友圈造势、预约爆卡 | 素材任务、触达任务、客户分层、个人链接 |
| 爆卡期 | 集中销售 199，推动老必带新 | 实时销售榜、裂变链路、待激活状态、奖励预估 |

## 5. 功能模块设计

### 5.1 拓客战役管理

核心页面：`199 拓客战役列表`、`战役详情`、`创建战役`

关键字段：

- 活动名称。
- 门店。
- 活动时间。
- 爆卡时间。
- 激活期时间。
- 199 卡项内容。
- 卡项原价与活动价。
- 适用项目。
- 老客激活条件。
- 新客购买条件。
- 奖励规则。
- 退款规则。
- 2991 承接产品。
- 活动状态。

关键能力：

- 从模板一键创建 199 拓客战役。
- 支持按门店复制活动。
- 支持生成 H5 或小程序活动页。
- 支持生成员工个人分享链接。
- 支持活动启动清单。

### 5.2 卡项与权益配置

199 卡项需要配置：

- 卡项名称。
- 总价值展示，例如原价 5200。
- 活动价 199。
- 项目组合。
- 次数或权益说明。
- 使用限制。
- 是否专人专卡。
- 是否必须推荐成功后激活。
- 激活后有效期。

奖励权益需要配置：

- 推荐 1 人：40 元红包。
- 推荐 3 人：120 元红包 + 洗护套。
- 门店自定义礼品。
- 奖励发放方式。
- 发放审核人。

2991 承接需要配置：

- 2991 卡名称。
- 大礼包内容。
- 分段支付规则。
- 十大权益。
- 适用项目。
- 介绍话术。

建议复用现有权益资产库能力，新增 `referral_reward`、`group_deal`、`trial_price`、`package_upgrade` 场景模板。

### 5.3 顾客盘点

核心页面：`顾客盘点工作台`

客户分层：

- 领袖顾客。
- 种子顾客。
- 股东顾客。
- 常到店顾客。
- 高成交率顾客。
- 沉睡顾客。
- 流失顾客。
- 体验卡顾客。
- 团购顾客。

字段：

- 顾客姓名。
- 手机号。
- 所属员工。
- 顾客分类。
- 关心项目。
- 成交率预估。
- 推荐潜力。
- 最近到店时间。
- 最近消费项目。
- 近期触达记录。
- 电话备注。

能力：

- 从客户画像、消费记录、到店频次、营销响应分自动推荐分层。
- 支持人工调整标签。
- 支持批量分配给员工。
- 支持导入 PPT 中的表格模板。
- 支持按成交率生成电话顺序。

### 5.4 目标分解与员工任务

核心页面：`拓客目标与执行看板`

目标维度：

- 门店总目标。
- 员工个人目标。
- 老客购买目标。
- 新客购买目标。
- 到店激活目标。
- 2991 转化目标。
- 沉睡客唤醒目标。

员工任务：

- 每日朋友圈发布任务。
- 老客微信触达任务。
- 电话爆卡任务。
- 新客邀约任务。
- 到店接待任务。
- 新客建档任务。
- 2991 铺垫任务。
- 奖励跟进任务。

看板指标：

- 已触达客户数。
- 已购买 199 数。
- 已生成链接数。
- 已分享数。
- 新客购买数。
- 老客激活数。
- 新客到店数。
- 建档完成数。
- 2991 转化数。
- 员工目标达成率。

### 5.5 裂变链接与归因

核心能力：

- 每个员工有员工分享链接。
- 每个顾客有顾客分享链接。
- 每个链接带活动 ID、门店 ID、员工 ID、推荐人客户 ID。
- 新客购买后绑定推荐关系。
- 支持裂变树查看：员工 -> 老客 -> 新客 -> 新客再推荐。

关键事件：

- link_generated。
- link_shared。
- page_viewed。
- purchase_started。
- paid_199。
- activation_invited。
- arrived_store。
- profile_completed。
- service_booked。
- referral_reward_earned。
- converted_2991。

归因规则：

- 同一新客多次打开不同链接时，默认最后一次有效推荐归因。
- 如已有明确推荐人，可由店长人工调整。
- 归因调整需要留操作记录。

### 5.6 触达任务与话术模板

模板库分类：

- 朋友圈预热。

- 微信私聊。
- 老客爆卡。
- 老客跟进。

- 新客转介绍。
- 老带新。
- 新带新。
- 沉睡客激活。
- 2991 铺垫。

执行能力：

- 根据客户类型推荐话术。
- 一键复制微信文案。
- 记录客户反馈。
- 标记抗拒点。

### 5.7 到店激活与新客建档

核心页面：`199 到店激活`

流程：

1. 搜索客户或扫码核销。
2. 确认 199 购买状态和推荐人。
3. 执行带参观 SOP。
4. 填写新客资料表。
5. 生成首次服务预约。
6. 教客户生成个人链接。
7. 记录是否完成 2991 铺垫。

建档字段应进入客户画像和健康档案：

- 面部护理习惯。
- 养生护理习惯。
- 皮肤状态。
- 身体调理建议。
- 亚健康情况。
- 疼痛部位。
- 禁忌与特殊情况。
- 关心项目。
- 预算偏好。
- 分段支付接受度。

### 5.8 2991 留客承接

核心页面：`2991 转化跟进`

能力：

- 在新客激活后自动生成 2991 跟进任务。
- 根据新客建档信息推荐 2991 卖点。
- 记录铺垫状态：未铺垫、已介绍堆头、已讲权益、已预约沟通、已成交、已拒绝。
- 支持分段支付订单或卡项订单承接。
- 2991 成交归因到原 199 活动和推荐链路。

核心指标：

- 199 到店激活率。
- 2991 铺垫率。
- 2991 成交率。
- 2991 客单价。
- 199 到 2991 转化周期。

### 5.9 沉睡客激活

可作为自动营销规则或拓客战役子模块。

触发条件：

- 最近 90/180/365 天未到店。
- 曾购买但长期未服务。
- 曾有投诉或流失风险标签。
- 营销响应分较高。

执行流程：

- 院长身份第一次微信。
- 未回复则第二次微信。
- 仍未回复则电话。
- 到店后赠送无服务性质礼品。
- 重新建档或更新客户画像。

系统能力：

- 生成沉睡客名单。
- 分配给院长或顾问。
- 提供三步话术。
- 记录回复、拒绝、到店、成交。
- 排除近期已预约或已触达客户，避免打扰。

## 13. 验收标准

P0 完成后，应能用一场真实或演示门店活动验证：

1. 店长能创建 199 拓客战役，并配置 199 卡项、奖励和 2991 承接。
2. 系统能按客户画像和到店数据生成顾客盘点名单。
3. 员工能领取客户任务，复制话术并记录触达结果。
4. 员工和顾客都能生成个人分享链接。
5. 新客通过链接购买后，系统能记录推荐关系。
6. 老客推荐 1 位新客购买后，系统自动把老客 199 状态改为可激活。
7. 新客到店后，前台或终端能完成建档、预约和转介绍引导。
8. 199 到店客户能进入 2991 跟进任务。
9. 看板能展示购买、分享、激活、到店、建档、2991 转化和员工排行。
10. 活动结束后能生成复盘报表，并保留活动、客户、员工、奖励和成交归因。

## 14. 风险与产品取舍

| 风险 | 影响 | 建议 |
| --- | --- | --- |
| 只做活动页，不做员工任务 | 门店执行仍靠微信群，数据断层 | P0 必须做员工任务和触达记录 |
| 只记录购买，不记录推荐关系 | 无法核算老客激活和奖励 | P0 必须做推荐关系和激活状态 |
| 话术只放文档 | 员工不会按 SOP 执行 | P0 做话术复制和反馈记录，P1 再做考核 |
| 199 和 2991 分离 | 低价拓客无法证明经营价值 | P0 必须做 2991 跟进状态和归因 |
| 自动发送微信过早 | 渠道、合规和交付风险高 | P0 先做任务、复制和记录，不做强自动发送 |
| 奖励发放不审核 | 容易产生纠纷 | P0 可先人工确认，P1 做自动核算和审核流 |

## 15. 推荐实施顺序

1. 先做数据闭环：战役、链接、推荐关系、激活状态、基础看板。
2. 再做执行闭环：顾客盘点、员工任务、话术模板、到店建档。
3. 再做转化闭环：2991 跟进、分段支付承接、成交归因。
4. 最后做智能化：高潜客户推荐、沉睡客自动规则、员工分配优化、活动复盘优化。

## 17. 交付结论

这份 PPT 可以转化为一个完整的“199 拓客增长战役”产品模块。它的开发重点不是低价卡售卖，而是：

- 用 199 卡做获客入口。
- 用老必带新和新带新做裂变。
- 用顾客盘点和员工任务保证执行。
- 用到店建档和五好文化完成信任建立。
- 用 2991 卡完成留客和收入承接。
- 用数据看板和归因证明活动效果。

因此，建议作为智能营销模块的一个 P0 增长场景落地，并优先打通现有营销活动、营销 H5、客户画像、智能推荐、自动营销规则、终端跟进任务和权益资产库。
