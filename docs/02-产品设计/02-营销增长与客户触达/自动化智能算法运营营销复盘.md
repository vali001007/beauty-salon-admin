# 自动化与智能算法运营营销复盘

生成日期：2026-06-14
复盘范围：管理端、`server-v2`、Ami Aura Lite 智能终端、Ami Glow 客户小程序、智能营销与经营工作台相关文档和代码。

## 1. 执行摘要

当前系统已经不是单点 AI 文案工具，而是形成了一个“数据采集 -> 客户预测 -> 智能推荐 -> 自动营销 -> 终端执行 -> 反馈归因”的闭环雏形。

但从美容门店真实经营来看，现在的强项是“规则模型 + 自动化工作流 + LLM 辅助内容生成”，还不是完整的“自主增长系统”。下一阶段最关键的不是继续堆页面，而是把触达渠道、顾问任务、预约履约、服务交付、库存采购、归因复盘这些执行环节打通。

结论可以浓缩成三句话：

1. 减少人工方面：系统已经覆盖智能排班、自动圈人、自动营销建议、AI 文案、测肤报告、库存补货、日结/月账单等高频工作，但还缺“真实触达执行、顾问任务催办、异常处理、采购/对账闭环”。
2. 增长客户方面：系统已经覆盖流失唤醒、复购窗口、LTV 分层、活动响应、卡券到期、护理周期、小程序行为、终端现场推荐，但还缺“拉新裂变、私域渠道、会员权益、A/B 测试、频控、渠道归因、内容分发”。
3. 产品路线方面：优先把现有算法建议变成门店每天能执行的任务闭环，再考虑更复杂的机器学习模型。

| 财务日结、Ami 绩效与月账单 | `recordAmiContribution()`、每日结算、每月自动生成 Ami 月账单 | 量化 AI 带来的收入和节省工时 | 已落地，需加强异常解释和对账 |

## 3. 当前算法本质判断

当前系统的“智能”主要分四层：

| 层级 | 当前表现 | 说明 |
| --- | --- | --- |
| L1 规则自动化 | 自动营销规则、终端自动化、库存预警、日结/月账单 | 依赖明确条件和触发器，适合 MVP 和可解释交付 |
| L2 可解释评分模型 | 流失分、复购分、营销响应分、LTV 分层、项目推荐评分 | 根据消费、到店、卡项、肤质、触达转化等特征计算，不依赖黑盒训练 |
| L3 LLM 辅助生成 | 文案、活动页、邀约话术、服务建议、下一步行动、意图解析 | 提升内容和操作效率，但需要审批、审计和兜底 |
| L4 学习型优化 | A/B 测试、uplift、渠道预算、多臂老虎机、自动调权 | 目前基本欠缺，需等事件、触达和转化数据稳定后再做 |

因此，当前最适合对外表达为：

> Ami 已具备可解释智能经营算法和自动化执行闭环雏形，下一阶段会把真实渠道、任务执行、效果归因和模型自学习补齐。

## 8. 关键产品判断

### 8.1 不建议马上投入“大模型全自动经营”

原因：

1. 美容门店的核心问题不是缺少更多建议，而是建议有没有被真实执行。
2. 自动触达、顾问跟进、客户反馈、成交归因的数据还没有完全闭环，直接做复杂 ML 容易学不到真实因果。
3. 美业场景涉及皮肤、护理、价格、疗效表达，AI 自动输出必须有边界和审批。

### 8.2 应优先做“可控自动化”

更适合当前阶段的方向：

1. 规则可解释：店长能知道为什么命中客户。
2. 动作可审批：活动、文案、触达前能预览和确认。
3. 执行可追踪：每条触达、每个顾问任务、每笔转化都有记录。
4. 效果可复盘：能看命中、触达、打开、点击、预约、到店、成交、复购。
5. 模型可迭代：先用规则，等数据稳定后再做权重学习和 uplift。

## 9. 优先级路线图

### P0：先把“建议”变成“每天自动执行”

目标周期：2-4 周。

| 事项 | 交付物 | 价值 |
| --- | --- | --- |
| 真实触达执行层 | 短信/小程序订阅消息/企微或微信模板接入，记录发送、失败、点击、退订 | 让自动营销不用人工复制粘贴 |
| 顾问任务闭环 | 自动分派、截止时间、催办、完成原因、转化记录 | 让算法建议真正落到人 |
| 行为埋点补齐 | 小程序首页、项目详情、预约漏斗、领券、卡项、推荐点击、取消预约 | 提高推荐准确性 |
| 频控与免打扰 | 客户维度频控、策略优先级、黑名单、退订 | 避免过度营销 |
| 数据完整度工作台 | 手机号、生日、肤质、会员等级、上次到店、卡项状态缺失提醒 | 提升算法基础数据质量 |
| 审批与审计 | AI 文案、活动页、自动触达上线前审批，保留版本 | 降低误发和合规风险 |

### P1：把“运营自动化”和“增长旅程”做成闭环

目标周期：1-2 个月。

| 事项 | 交付物 | 价值 |
| --- | --- | --- |
| 预约自动化 | 自动确认、改期推荐、候补补位、爽约挽回 | 减少前台人工，提高到店率 |
| 会员生命周期旅程 | 新客欢迎、首访后回访、复购提醒、沉默唤醒、生日/周年 | 提升留存和复购 |
| 券/积分/权益体系 | 券包、积分、等级权益、生日礼、老带新权益 | 让营销动作有稳定抓手 |
| A/B 测试和保留组 | 文案、渠道、权益、频次测试 | 判断真实增量 |
| 容量与库存约束 | 推荐活动前检查排班、房间、设备、库存 | 防止营销带来履约风险 |
| 服务 SOP 与耗材联动 | 项目 SOP、耗材扣减、服务照片、回访计划 | 提高服务一致性和成本准确性 |
| 评价和口碑闭环 | 服务后评价邀约、差评预警、好评沉淀 | 增强新客转化 |

### P2：做真正的增长优化算法

目标周期：2-3 个月以上，依赖 P0/P1 数据闭环。

| 事项 | 交付物 | 价值 |
| --- | --- | --- |
| 权重自学习 | 根据转化反馈调整推荐权重 | 从规则模型走向半自动优化 |
| Uplift/保留组模型 | 识别“触达才会转化”的客户 | 减少无效营销和打扰 |
| 渠道预算优化 | 按渠道成本、转化率、LTV 分配预算 | 提升 ROI |
| 个性化权益优化 | 根据客户价格敏感度推荐券门槛和权益 | 提升成交同时保护毛利 |
| 智能客服/企微助手 | 顾客咨询、预约、改期、售后自动应答 | 减少人工客服 |
| 员工能力模型 | 美容师技能、转化、满意度和项目匹配 | 提升人效和服务质量 |

## 10. 建议衡量指标

### 10.1 减少人工指标

| 指标 | 说明 |
| --- | --- |
| 自动触达占比 | 触达中由系统自动执行的比例 |
| 顾问任务自动生成数 | 系统每天为顾问生成多少可执行任务 |
| 顾问任务完成率 | 任务是否真正被跟进 |
| 排班耗时 | 店长每周排班从多少小时降到多少分钟 |
| 预约人工确认量 | 需要前台手动确认的预约数量 |
| 库存人工盘点异常数 | 低库存、临期、耗材异常是否减少 |
| 财务日结异常处理时长 | 对账差异定位是否更快 |
| AI 贡献工时 | `AmiPerformanceRecord.workMinutes` 汇总 |

### 10.2 增长客户指标

| 指标 | 说明 |
| --- | --- |
| 高风险客户挽回率 | 高流失客户触达后到店或成交比例 |
| 30 天复购率 | 复购窗口客户的实际复购 |
| 预约转化率 | 浏览、领券、测肤后的预约转化 |
| 到店率 | 预约后实际到店比例 |
| 卡项核销率 | 次卡/套餐到期提醒后的核销 |
| 客单价和升单率 | 终端推荐或顾问跟进带来的升单 |
| LTV 提升 | 推荐和自动营销后客户长期价值变化 |
| 活动真实增量 | 通过保留组判断的增量收入 |
| 触达疲劳率 | 退订、屏蔽、投诉、连续未读 |

## 11. 行业趋势参考

外部资料显示，美业软件竞争已经从“预约/收银工具”进入“自动化运营 + 客户个性化 + 经营增长”阶段：

1. McKinsey 2025 年美容行业报告指出，美容行业增长仍在，但品牌和门店需要应对增长放缓、消费者更挑剔、数字影响和个性化需求增强的问题。参考：https://www.mckinsey.com/industries/consumer-packaged-goods/our-insights/state-of-beauty
2. Square 2025 美业趋势资料提到，美业经营者正在把技术用于营销自动化、AI 经营分析、员工排班、预约和支付，并强调客户留存、会员和新收入来源。参考：https://squareup.com/gb/en/the-bottom-line/reaching-customers/top-beauty-trends
3. Square 美业解决方案强调 24/7 在线预约、客户档案、自动通知、无爽约保护、员工排班、营销活动和库存提醒是一体化美业软件的关键能力。参考：https://squareup.com/us/en/beauty/hair-salon
4. Salon Today 2025 技术方案指南把 salon technology tools、marketing、client retention、finance、employee management 等作为美业经营技术关注领域，说明行业软件竞争已经覆盖经营全链路。参考：https://www.salontoday.com/1095328/salon-today-2025-technology-solutions-guide

对 Ami 的启发：

1. 已有的智能推荐和自动营销方向是对的。
2. 行业标配能力如 24/7 预约、自动提醒、双向沟通、会员权益、无爽约保护、客户评价、库存和员工管理要继续补齐。
3. 真正形成差异化的点不是“也有 AI”，而是 AI 能否直接减少门店人工、提高到店和复购，并能用数据证明 ROI。

## 12. 建议下一步产品落点

最建议先做一个“自动化经营闭环 MVP”，把现有能力收束成老板和店长每天能感知的结果。

### 12.1 MVP 页面与流程

| 模块 | 核心动作 |
| --- | --- |
| 今日自动化任务 | 展示今天自动生成的高风险唤醒、复购提醒、预约放弃、卡券到期、顾问跟进任务 |
| 一键执行/审批 | 店长确认策略后，系统批量发送触达或派发顾问任务 |
| 顾问工作台 | 每个顾问看到自己的客户、话术、截止时间、跟进结果 |
| 触达与转化看板 | 展示发送、打开、点击、预约、到店、成交、复购 |
| 异常与风险 | 展示触达失败、客户退订、过度触达、缺少联系方式、活动容量不足 |
| ROI 看板 | 展示节省工时、带来收入、转化客户、Ami 贡献 |

### 12.2 MVP 成功标准

| 标准 | 可验收口径 |
| --- | --- |
| 店长不用人工筛客户 | 每天自动生成可执行客户清单 |
| 运营不用复制粘贴文案 | 文案、渠道、触达对象、发送状态都在系统里 |
| 顾问知道该跟谁 | 任务自动分派并有截止时间 |
| 老板能看到结果 | 每条自动化都能追踪收入、工时和转化 |
| 客户不会被过度打扰 | 有频控、退订、黑名单和审批 |

## 13. 结论

当前系统的基础已经足够做“美业智能经营助手”的 MVP，能力覆盖面比普通预约/收银系统更强。真正的短板不是算法种类不够，而是算法建议到真实经营动作之间还有断点。

下一阶段应该优先补齐：

1. 渠道触达执行闭环。
2. 顾问任务和服务 SOP 闭环。
3. 行为埋点、推荐反馈和营销归因闭环。
4. 预约、库存、财务异常自动化。
5. 会员、裂变、口碑和内容渠道增长闭环。

只要把这些闭环打通，现有规则模型和 LLM 能力就可以持续产生可量化价值；等数据积累稳定后，再升级到更复杂的自学习推荐和增长优化算法。
