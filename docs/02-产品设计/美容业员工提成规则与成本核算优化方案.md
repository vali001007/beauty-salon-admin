# 美容业员工提成规则与成本核算优化方案

更新时间：2026-06-21

适用范围：Ami_Core 管理端、经营利润模块、员工人效与提成模块、Ami Aura 数字员工绩效归因。

## 1. 结论

美容院成熟的提成规则通常不是简单地按利润或售价二选一，而是采用“员工激励按可感知业绩，老板核算按真实毛利”的双账口径。

建议本产品采用以下原则：

1. 员工提成基数优先使用订单明细实收金额、耗卡价值、服务次数和固定手工费，员工能理解、能复核、能预期。
2. 老板经营核算使用收入、耗材成本、商品成本、提成成本、固定工资、房租、营销和折旧，算清真实毛利和经营利润。
3. 提成规则必须和成本核算分开展示，但底层数据要能关联到同一笔订单、同一个明细、同一位员工。
4. 不建议让一线员工直接按净利润提成，因为利润受房租、广告、折扣、耗材、赠送、历史卡项等因素影响，员工难以验证，容易产生不公平感。
5. Ami Aura 数字员工不应参与真人手工提成，应按可归因增量绩效、有效值守和任务完成度结算。

## 2. 行业成熟提成规则

### 2.1 常见薪酬结构

美容院常见结构是：

```text
员工收入 = 保底/底薪 + 销售提成 + 手工费/服务提成 + 耗卡/到店奖励 + 阶梯奖金 + 团队奖励 - 扣回项
```

公开招聘信息中，“底薪 + 手工费 + 销售提成 + 奖金”是高频表述，说明行业已形成销售和服务双提成的心智。系统不能只建一个“销售额提成”字段，否则无法覆盖真实门店薪酬。

### 2.2 销售提成

适用对象：顾问、美容师、店长、前台或参与销售的员工。

常见基数：

| 基数 | 适用场景 | 建议 |
| --- | --- | --- |
| 实收金额 | 单次项目、产品、套餐、充值、办卡 | 默认推荐，最容易核对 |
| 明细实收金额 | 混合订单、套餐价、整单优惠分摊 | 产品必须支持，避免重复或错算 |
| 首款到账金额 | 高客单分期、预付订单 | 只按已收款部分计提 |
| 毛利金额 | 高成本产品、仪器类、医美类 | 可作为店长/顾问高阶规则，不作为新手默认 |

建议规则：

```text
销售提成 = 可计提实收金额 x 岗位/品类提成比例 x 归因权重
```

成熟做法是按品类区分比例：

| 品类 | 提成建议 |
| --- | --- |
| 标准护理项目 | 中等比例，按实收或耗卡计提 |
| 高毛利项目 | 可给较高比例，鼓励主推 |
| 低毛利引流项目 | 低比例或不提成，只给服务次数奖励 |
| 零售产品 | 按实收提成，同时在利润模块扣商品成本 |
| 办卡/充值 | 低比例计提，且支持退款保护期 |
| 赠送权益 | 不计提或按 0 收入计提 |

### 2.3 手工费/服务提成

适用对象：实际完成护理、身体项目、仪器辅助、皮肤管理等服务的美容师。

常见基数：

| 基数 | 适用场景 | 员工感受 | 成本清晰度 |
| --- | --- | --- | --- |
| 固定手工费 | 标准化项目，例如单次护理 | 稳定、易懂 | 高 |
| 服务实收比例 | 不同价格项目 | 和收入挂钩 | 中 |
| 耗卡单次价值比例 | 卡项核销 | 贴近履约价值 | 中 |
| 服务时长系数 | 长时项目、身体项目 | 更公平 | 需要排班/工时数据 |

建议本产品支持两种基础模式：

```text
模式 A：固定手工费 = 项目配置的服务费 x 服务次数 x 技师权重

模式 B：服务提成 = 项目明细实收金额 x 项目提成比例 x 技师权重
```

优先推荐模式 A 给中小门店，因为员工容易理解，老板也能把它作为直接人工成本扣进项目毛利。

### 2.4 耗卡/消课提成

美容院一个常见问题是：销售时员工拿了办卡提成，但后续服务消耗由其他美容师承担。如果只按办卡提成，会鼓励卖卡，不鼓励认真服务。

建议拆成两段：

```text
办卡销售提成 = 办卡实收 x 低比例 x 销售归因

耗卡服务提成 = 单次耗卡价值 x 服务提成比例 x 实际服务人权重
```

其中：

```text
单次耗卡价值 = 卡项实收金额 / 可用次数
```

如果有赠送次数，应显示两种口径：

```text
财务单次价值 = 实收金额 / 总可履约次数
销售展示单次价值 = 标准售价 / 标准次数
```

提成建议使用财务单次价值，避免赠送越多、利润越虚。

### 2.5 阶梯提成

阶梯提成是调动积极性的核心，但必须防止月底压单、诱导过度销售。

建议采用“区间内阶梯”，而不是“全额跳档”：

```text
0-3 万：3%
3-6 万：5%
6 万以上：7%
```

若员工当月销售 8 万：

```text
提成 = 3万 x 3% + 3万 x 5% + 2万 x 7%
```

不建议：

```text
8万 x 7%
```

原因是全额跳档会导致临界点前后收入剧烈变化，员工容易为了冲档牺牲价格、服务质量和客户体验。

### 2.6 团队分成与归因拆分

成熟门店通常存在多人参与：

- 顾问负责成交。
- 美容师负责服务。
- 店长负责压单或审批优惠。
- 前台负责预约和复访。
- Ami Aura 负责推荐、话术、提醒或客户资料查询。

建议产品支持“贡献角色”而不是只存一个销售员：

| 角色 | 常见权重 |
| --- | --- |
| 主成交人 | 50%-70% |
| 服务人 | 20%-40% |
| 协助人 | 5%-20% |
| 店长/团队池 | 5%-10% |
| 数字员工 | 仅参与可归因增量池，不参与手工池 |

系统应支持同一订单明细配置多个参与人，且总权重必须等于 100% 或明确存在团队池。

## 3. 员工心态与机制风险

### 3.1 员工真正关心什么

一线员工通常不只关心比例高低，更关心以下问题：

| 员工问题 | 产品需要解决 |
| --- | --- |
| 这单算不算我的？ | 订单归因、服务人、协助人清晰 |
| 为什么这个月少了？ | 提成明细、扣回、退款、未结算原因可追溯 |
| 我服务了客户，销售提成给了别人怎么办？ | 销售提成和服务提成拆开 |
| 老客户是谁维护的？ | 客户归属和复购贡献 |
| AI 或店长参与后会不会抢我的钱？ | 明确数字员工和管理岗只参与增量池或团队池 |
| 老板说没利润所以不给提成，是否合理？ | 员工提成不要直接依赖全店净利润 |

### 3.2 容易伤害积极性的规则

需要避免：

1. 只按销售额，不看服务质量，容易过度销售。
2. 只按利润，不给员工清晰计算过程，容易不信任。
3. 办卡一次性高提成，后续服务没有奖励，容易重销售轻履约。
4. 全额跳档，容易月底冲单、压单、提前收款。
5. 退款扣回没有周期和明细，容易造成工资争议。
6. AI 推荐、店长压单、员工服务混在一起，容易引发抢单。

### 3.3 更健康的激励方向

建议把员工激励从“只卖出去”升级为“四类贡献”：

| 贡献 | 指标 | 激励方式 |
| --- | --- | --- |
| 成交贡献 | 项目、产品、卡项实收 | 销售提成 |
| 履约贡献 | 服务次数、耗卡价值、服务时长 | 手工费/服务提成 |
| 复购贡献 | 30/60/90 天复购、续卡 | 复购奖励 |
| 质量贡献 | 好评、投诉率、到店率、流失挽回 | 奖金或等级晋升 |

这样可以减少“只会卖卡”的短期行为。

## 4. 成本核算口径

### 4.1 员工提成和老板利润要分开

建议系统采用双口径：

```text
员工可见提成 = 员工能理解的实收、耗卡、手工费、奖励、扣回

老板可见利润 = 经营收入 - 商品成本 - 耗材成本 - 直接提成 - 固定工资 - 房租 - 营销 - 折旧 - 其他成本
```

员工端不展示全店净利润，只展示本人服务、销售、复购和提成明细；老板端展示提成作为人工成本进入毛利和经营利润。

### 4.2 订单明细实收是统一基础

本产品当前已有订单级优惠分摊设计，提成和利润应优先使用订单明细 `netAmount`。这对美容院非常关键：

- 套餐价、整单优惠、手工改价必须分摊到每个明细。
- 提成不能按原价算，否则员工收入虚高。
- 利润不能按原价算，否则老板误判毛利。
- 赠品应为 0 收入，不参与提成和利润。

建议所有提成记录都保留：

| 字段 | 说明 |
| --- | --- |
| 订单 ID | 追溯原始订单 |
| 明细 ID | 对应具体项目/商品/卡项 |
| 员工 ID | 提成归属 |
| 角色 | 主成交、服务、协助、店长、数字员工 |
| 基数类型 | 实收、耗卡、固定手工、毛利、奖励 |
| 基数金额 | 参与计算的金额 |
| 比例/固定金额 | 规则配置 |
| 归因权重 | 多人拆分 |
| 提成金额 | 最终金额 |
| 结算状态 | 待结算、已结算、退款扣回、人工调整 |
| 数据来源 | 系统规则、人工确认、历史导入 |

### 4.3 毛利保护线

为兼顾激励和成本，建议每个项目和商品维护最低毛利保护线。

示例：

```text
项目贡献毛利 = 明细实收 - 耗材成本 - 直接服务提成
商品贡献毛利 = 商品明细实收 - 商品成本 - 商品销售提成
```

若毛利率低于阈值：

- 普通员工仍可拿基础手工费，保障服务积极性。
- 销售提成降低或进入店长审批。
- 系统提示“低毛利成交”，让老板知道问题出在折扣、成本、项目结构还是提成规则。

### 4.4 退款、撤单和异常扣回

提成必须支持扣回，但规则要透明：

| 情况 | 建议 |
| --- | --- |
| 当月退款 | 未结算提成直接取消 |
| 跨月退款 | 下月生成负向提成记录 |
| 部分退款 | 按退款比例扣回 |
| 员工离职后退款 | 进入老板/财务处理，不自动追扣历史工资 |
| 服务质量投诉退款 | 可标记责任人，但需要人工审核 |

不要只在工资里扣一个总数，必须让员工看到对应订单和原因。

## 5. 推荐的产品化提成模型

### 5.1 规则层级

建议规则从高到低匹配：

```text
门店特殊规则
> 岗位等级规则
> 项目/商品/卡项品类规则
> 全店默认规则
```

每条规则需要有生效时间，避免历史工资被新规则影响。

### 5.2 角色模型

建议至少支持：

| 角色 | 是否计销售 | 是否计服务 | 是否计成本 |
| --- | --- | --- | --- |
| 美容师 | 可选 | 是 | 是 |
| 顾问 | 是 | 可选 | 是 |
| 店长 | 可选 | 否 | 是 |
| 前台 | 可选 | 否 | 是 |
| 数字员工 Ami Aura | 仅可归因增量 | 否 | 作为数字员工服务成本或绩效费用 |

### 5.3 推荐默认规则

给中小美容院的默认模板：

```text
美容师：
- 服务项目：固定手工费或项目实收 5%-12%
- 产品销售：实收 3%-8%
- 办卡/充值：实收 0.5%-3%，可设置保护期
- 耗卡服务：单次耗卡价值 5%-10% 或固定手工费
- 复购奖励：客户 60 天内复购，按复购订单给固定奖励或低比例

顾问：
- 项目/套餐成交：实收 3%-10%
- 办卡/充值：实收 1%-5%，建议低于即时项目销售
- 产品销售：实收 3%-8%
- 复购维护：按归属客户复购给低比例

店长：
- 不建议逐单高比例提成
- 推荐门店毛利达标奖金、团队目标奖金、服务质量奖金

前台：
- 不建议高销售提成
- 推荐预约到店、核销准确率、客户回访完成奖励
```

比例只是系统模板建议，最终应由门店根据城市、客单价、毛利、岗位等级和经营目标确认。

### 5.4 数字员工 Ami Aura 规则

Ami Aura 不建议参与以下提成：

- 美容师手工费。
- 实际护理服务提成。
- 员工已明确归属的老客服务提成。
- 全部门店流水抽成。

建议参与：

| 贡献 | 结算方式 |
| --- | --- |
| AI 推荐带来成交 | 可归因订单实收的低比例，或增量毛利分成 |
| 老客唤醒到店 | 每个有效到店固定奖励 |
| 预约自动跟进 | 每个有效预约/核销固定奖励 |
| 话术辅助成交 | 只参与辅助归因池 |
| 经营提醒带来调整 | 纳入订阅费或值守费，不逐单提成 |

推荐模型：

```text
Ami Aura 收入 = 基础订阅费 + 数字员工值守费 + 可归因绩效提成
```

其中可归因绩效必须满足：

1. 有客户触点记录。
2. 有推荐、提醒、报价、预约或跟进动作。
3. 有成交或到店结果。
4. 能排除纯线下自然成交。
5. 支持店长审核。
6. 有退款保护期和封顶机制。

## 6. 系统需要补齐的能力

### 6.1 提成规则配置

必须支持：

- 按岗位、等级、员工、项目、商品、卡项设置规则。
- 按实收、固定金额、耗卡价值、毛利、服务次数设置基数。
- 支持多人归因权重。
- 支持区间阶梯。
- 支持生效时间和失效时间。
- 支持低毛利保护线。
- 支持退款扣回。
- 支持人工调整和审批记录。

### 6.2 员工端透明账单

员工应能看到：

- 本月预计提成。
- 已结算提成。
- 待结算提成。
- 被退款或撤单扣回的明细。
- 每笔提成对应客户、订单、项目、基数、比例和金额。
- 为什么某笔订单“不计提成”。

员工端不应看到：

- 全店净利润。
- 其他员工工资。
- 商品采购成本明细。
- 老板固定成本。

### 6.3 老板端利润视角

老板应能看到：

- 项目收入、耗材成本、提成成本、贡献毛利。
- 商品收入、商品成本、销售提成、商品毛利。
- 员工服务收入、提成、贡献毛利。
- 办卡/充值现金收入与未来履约负债。
- 低毛利项目、低毛利员工组合、异常折扣。
- 缺成本、缺 BOM、缺提成规则、缺服务人归属的数据质量提醒。

### 6.4 审核和治理

需要有审核机制：

| 场景 | 审核建议 |
| --- | --- |
| 大额订单提成 | 店长或老板确认 |
| 手工改价订单 | 自动标记，影响毛利 |
| 低毛利成交 | 提示或审批 |
| 人工调整提成 | 必须记录调整原因 |
| 归属争议 | 保留修改前后记录 |
| 数字员工归因 | 店长可审核通过或驳回 |

## 7. 分阶段落地建议

### 阶段 1：先算清楚，不急着复杂激励

目标：让老板和员工都信任账。

- 使用订单明细实收作为统一基数。
- 服务人、销售人、协助人必须落到订单明细。
- 支持固定手工费和简单比例提成。
- 支持退款扣回。
- 员工端展示提成明细。
- 老板端把提成作为项目/商品毛利成本。

### 阶段 2：加入阶梯和毛利保护

目标：既激励增长，也防止亏损成交。

- 支持区间阶梯。
- 项目/商品维护最低毛利线。
- 低毛利订单提示。
- 办卡和耗卡拆分计提。
- 复购奖励和服务质量指标进入员工人效。

### 阶段 3：加入数字员工绩效

目标：让 Ami Aura 作为增量助手，而不是抢员工提成。

- 建立 Aura 事件归因链。
- 支持 AI 推荐、预约、唤醒、报价到成交的追踪。
- 数字员工只参与增量池或辅助归因池。
- 店长可审核数字员工贡献。
- 给老板展示“Aura 带来的新增收入、节省人力、提升复购”，而不是简单说 AI 要拿提成。

## 8. 产品验收标准

1. 同一笔订单可以同时生成销售提成、服务提成和成本毛利记录。
2. 整单优惠能分摊到明细，提成和利润都使用明细实收。
3. 员工能看到本人每笔提成的来源和计算过程。
4. 老板能看到提成作为人工成本进入项目/商品毛利。
5. 办卡收入不会被直接当成利润，耗卡服务能单独产生服务提成。
6. 退款能生成负向提成或取消待结算提成。
7. 阶梯提成按区间内分段计算，不默认全额跳档。
8. 低毛利订单能提示，不静默按高业绩奖励。
9. 数字员工不参与真人手工费，只参与可归因增量。
10. 缺服务人、缺成本、缺规则时，系统显示数据缺口，不按 0 静默计算。

## 9. 对当前产品的直接建议

结合当前经营利润模块的建设方向，建议把“提成规则”从单纯财务功能升级成“员工激励 + 利润成本 + 数字员工归因”的共同底座。

优先级建议：

| 优先级 | 能力 | 原因 |
| --- | --- | --- |
| P0 | 订单明细服务人/销售人/协助人归属 | 没有归属就无法算提成和人效 |
| P0 | 明细实收提成基数 | 解决优惠、套餐、赠品导致的错算 |
| P0 | 固定手工费 + 简单比例提成 | 覆盖大部分中小门店 |
| P0 | 员工提成明细账单 | 提升员工信任 |
| P0 | 提成进入项目/商品毛利 | 让老板看到真实成本 |
| P1 | 阶梯提成和低毛利保护 | 兼顾激励和利润 |
| P1 | 办卡/耗卡拆分 | 避免只卖卡不服务 |
| P1 | 退款扣回和结算周期 | 降低争议 |
| P2 | 复购奖励、客户维护奖励 | 从短期销售转向长期经营 |
| P2 | Ami Aura 可归因绩效 | 支持数字员工商业化 |

## 10. 参考来源

- 《中华人民共和国劳动法》规定工资分配遵循按劳分配、同工同酬，并允许用人单位依法自主确定工资分配方式，同时工资不得低于当地最低工资标准。
- 《最低工资规定》及多地政策解读均强调，实行计件工资或提成工资等形式时，支付给劳动者的工资不得低于相应最低工资标准。
- BOSS 直聘美容师招聘样例中，高频出现“底薪 + 手工费 + 销售提成 + 奖金”“销售、手工双提成”等薪资结构，可作为行业薪酬心智参考，但不应直接作为精确比例来源。
- 本项目现有经营利润设计已明确：办卡/充值不直接等于利润，毛利需要扣除耗材成本、商品成本和直接服务提成；员工人效需要展示服务收入、服务次数、直接提成和贡献毛利。
