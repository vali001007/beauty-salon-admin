# 商品与项目智能营销推荐方案

版本：v1.0  
日期：2026-06-14  
适用范围：Ami_Core 管理端智能营销、商品管理、项目管理、库存、排班、Ami Glow 小程序、Ami Aura Lite 终端  
关联文档：`docs/02-产品设计/智能推荐算法升级需求文档.md`、`docs/marketing-trigger-rules-requirements.md`、`docs/customer-profile-recommendation-pipeline-plan.md`

## 1. 背景

当前智能推荐体系已经具备客户生命周期、流失风险、复购窗口、LTV、营销响应、优惠券到期、卡项到期、护理周期等能力。下一步要把推荐从“客户运营推荐”升级为“人、货、项目、排班、时间”的经营推荐。

美业门店的真实增长机会通常不是单一客户标签触发，而是多个经营信号叠加：

- 商品临期：库存批次即将过期，如果不及时消化会形成损耗。
- 商品消耗增长：某些项目或季节带动耗材、家居护理商品消耗变快，需要及时补货、搭配销售或引导客户复购。
- 商品滞销积压：库存周转变慢，占用现金和库位，需要组合促销或跨店调拨。
- 美容师排期不满：低峰时段、空闲美容师和客户复购窗口叠加时，可以用低成本权益填充产能。
- 项目产能和毛利：高毛利项目、有空余产能的项目、关联商品充足的项目，适合优先推荐。

本方案目标是把这些经营信号转化为可解释、可执行、可归因的智能营销推荐卡。

## 2. 产品目标

### 2.1 一句话定位

把“商品库存压力、项目服务能力、美容师排班空档”自动转成可落地的营销动作，让门店知道此刻应该推什么、推给谁、怎么推、预期收益多少。

### 2.2 业务目标

- 降低商品临期损耗，优先把库存压力变成到店理由和套餐权益。
- 提升商品周转，识别消耗增长、滞销积压、复购周期到期等机会。
- 提升项目预约率，把低峰时段和美容师空闲转化为可售产能。
- 提升客单价，通过项目与商品组合、项目升级、疗程卡承接带动连带销售。
- 降低人工运营成本，推荐卡可直接生成活动草稿、自动营销规则、顾问任务和小程序推荐位。
- 打通归因闭环，能复盘推荐命中、触达、预约、核销、下单、库存消化和排期填充效果。

## 3. 推荐范围

### 3.1 商品推荐

| 场景 | 触发信号 | 推荐目的 | 典型动作 |
| --- | --- | --- | --- |
| 商品临期消化 | 批次剩余有效期小于 60/30/15/7 天，临期库存大于自然销量预测 | 降低过期损耗 | 项目搭赠、会员限时购、跨店调拨、顾问任务 |
| 商品消耗增长 | 近 7/14 天消耗环比或同比明显增长，库存可售天数下降 | 防断货并放大需求 | 补货建议、热销商品推荐、项目后家居护理带货 |
| 商品补货复购 | 客户购买后达到预计消耗周期，且未再次购买 | 提升复购 | 小程序补货提醒、复购券、同功效升级推荐 |
| 商品滞销积压 | 库龄长、库存周转慢、销量低于预期 | 回笼现金和释放库位 | 搭配畅销项目、第二件优惠、会员专享包 |
| 商品连带销售 | 客户做过项目且项目 BOM 或护理方案关联商品 | 提升客单价 | 项目后带回家装、疗程维养包、顾问推荐话术 |
| 新品冷启动 | 新品上架但浏览/购买/服务搭配不足 | 获取首批反馈 | 种子客户体验、达人顾客试用、评价返券 |

### 3.2 项目推荐

| 场景 | 触发信号 | 推荐目的 | 典型动作 |
| --- | --- | --- | --- |
| 美容师排期不满 | 未来 1-7 天可预约工时多，预约占用率低 | 填充低峰产能 | 低峰专享券、当天预约礼、老客关怀 |
| 项目产能闲置 | 某项目可服务美容师多，但预约少 | 提高项目利用率 | 项目体验价、会员复购提醒、项目教育内容 |
| 高毛利项目提升 | 项目毛利高、库存/BOM 充足、客户匹配度高 | 提升利润 | 高价值客户升级护理、疗程卡转化 |
| 护理周期到期 | 上次项目服务后达到 21/28/45 天周期 | 提升复购 | 周期提醒、同类项目复购券、升级项目推荐 |
| 项目带动商品消耗 | 项目 BOM 关联商品临期或库存充足 | 项目消化库存 | 项目套餐、项目后带商品、护理组合包 |
| 新项目冷启动 | 新项目上线，预约少但匹配人群存在 | 验证新品 | 小范围试运营、种子客户体验、评价沉淀 |

## 4. 成熟营销套路沉淀

### 4.1 商品临期

成熟打法不是一开始就大幅降价，而是按“损耗风险和自然消化能力”分层：

| 风险层级 | 判断 | 推荐动作 |
| --- | --- | --- |
| 轻度临期 | 31-60 天，预测自然销量可覆盖大部分库存 | 小程序推荐位、项目后顾问顺带推荐、会员轻权益 |
| 中度临期 | 16-30 天，预测自然销量覆盖不足 | 与强相关项目做套餐，满额赠小样，顾问跟进高匹配客户 |
| 重度临期 | 8-15 天，缺口库存明显 | 限时会员购、跨店调拨、员工内购、低峰项目搭赠 |
| 紧急临期 | 0-7 天或已过期风险 | 停止对客户销售过期风险商品，优先内部消化、报损或合规处理 |

推荐原则：

- 优先调拨到有需求门店，再做降价。
- 优先做“项目搭赠”和“会员权益”，避免伤害商品原价体系。
- 高价值客户避免强折扣推销，优先用“护理礼遇、专属赠品、顾问关怀”表达。
- 临期商品必须校验适用人群、禁忌、过敏风险和合规可售期。

### 4.2 商品消耗增长

消耗增长既可能是好事，也可能是风险。系统要区分三类：

| 类型 | 说明 | 推荐动作 |
| --- | --- | --- |
| 真实需求增长 | 商品订单、项目 BOM 消耗、浏览行为同步增长 | 补货、设置热销推荐位、项目后带货 |
| 服务带动增长 | 某项目预约增长导致耗材消耗增长 | 预测未来 7/14 天缺口，提醒补货，同时推荐项目套餐 |
| 异常消耗增长 | 实际消耗明显高于标准 BOM | 先做经营预警，不直接营销；提示复核服务记录或损耗 |

成熟打法：

- 爆款商品用“小程序补货提醒 + 项目后带回家护理”承接。
- 耗材类先保障供应，不盲目做促销，避免越卖越缺货。
- 家居护理商品可做周期复购，面膜、精华、防晒、修护类按不同消耗周期提醒。
- 若增长来自季节或天气，活动主题要围绕场景表达，例如夏季防晒控油、秋冬补水修护。

### 4.3 美容师排期不满

排期不满不是简单发优惠券，而是把“空闲时段、合适客户、合适项目、合适美容师”匹配起来。

推荐原则：

- 只在低峰时段使用低峰权益，避免影响黄金时段价格。
- 优先推荐美容师擅长项目和曾服务过的客户，提高转化。
- 优先触达高营销响应、近期浏览未预约、护理周期到期、沉睡轻度客户。
- 对已预约客户不重复触达同类预约活动。
- 对高价值客户优先顾问邀约和专属时段，而不是低价券。

典型动作：

- 工作日 14:00-17:00 低峰专享护理券。
- 空闲美容师老客回访任务。
- 当天余位提醒，仅展示剩余可约时段。
- 美容师专长项目小范围定向活动。

### 4.4 项目与商品组合

成熟组合不是随意捆绑，而是基于“项目效果闭环”设计：

- 清洁项目后推荐修护面膜或补水精华。
- 抗衰项目后推荐家居维养套装。
- 敏感修护项目后推荐温和修护商品，不推刺激性功效品。
- 防晒季项目后推荐防晒、舒缓、补水组合。
- 次卡/疗程卡推荐时，搭配周期护理商品提升持续效果。

组合推荐要同时满足：

- 客户画像匹配。
- 商品库存充足或存在可控临期消化压力。
- 项目可预约容量充足。
- 毛利和优惠后利润为正。
- 近期触达不过度。

## 5. 推荐引擎设计

### 5.1 推荐生成流程

```text
经营数据采集
  -> 信号计算
  -> 机会候选生成
  -> 客户/商品/项目匹配
  -> 评分排序
  -> 风控与频控
  -> 推荐卡生成
  -> 活动/自动规则/顾问任务承接
  -> 效果归因
```

### 5.2 数据输入

| 数据域 | 当前可用基础 | 用途 |
| --- | --- | --- |
| 商品 | `Product`、`StockBatch`、`StockMovement`、`ProductOrder` | 库存、临期、销售、周转、价格 |
| 项目 | `Project`、`ProjectType`、`ProjectBomItem` | 项目分类、价格、时长、BOM 关联商品 |
| 排班 | `Schedule`、`Reservation`、`BeauticianProjectSkill`、智能排班需求热力图 | 空闲时段、服务能力、项目技能匹配 |
| 客户 | `Customer`、健康档案、消费记录、卡项、预测快照 | 人群筛选、复购、LTV、流失、肤质 |
| 行为 | `CustomerBehaviorEvent` | 浏览未预约、商品兴趣、活动页意图 |
| 营销 | `MarketingAutomationTouch`、`RecommendationEvent`、活动归因 | 频控、反馈、转化归因 |

### 5.3 机会候选

系统先不直接对所有客户、商品、项目做全量组合，而是先生成经营机会候选：

| 候选类型 | 示例 |
| --- | --- |
| `product_expiry_clearance` | 某门店 80 盒修护面膜 30 天内临期 |
| `product_consumption_growth` | 某精华近 14 天消耗增长 45%，预计 9 天后低于安全库存 |
| `product_replenishment` | 客户购买防晒 45 天后进入补货周期 |
| `product_slow_moving` | 某套盒库存周转天数超过 120 天 |
| `project_idle_capacity` | 下周三下午美容师预约占用率仅 38% |
| `project_cycle_due` | 120 位客户进入补水护理 28 天复购窗口 |
| `project_high_margin_upsell` | 高 LTV 客户适合从基础护理升级到抗衰护理 |
| `project_product_bundle` | 补水项目可带动临期面膜库存消化 |

### 5.4 推荐评分

每张推荐卡按 0-100 分排序。建议使用规则评分起步，后续逐步引入模型。

```text
recommendationScore =
  businessUrgencyScore * 0.22
  + audienceFitScore * 0.18
  + itemProjectFitScore * 0.16
  + timingScore * 0.14
  + operationFitScore * 0.12
  + profitScore * 0.10
  + channelFitScore * 0.08
  - fatiguePenalty * 0.10
  - riskPenalty * 0.08
```

| 维度 | 含义 | 商品例子 | 项目例子 |
| --- | --- | --- | --- |
| `businessUrgencyScore` | 经营紧迫度 | 临期天数、滞销库存、断货风险 | 低峰空档、项目冷启动、产能浪费 |
| `audienceFitScore` | 人群匹配度 | 肤质、历史购买、补货周期 | 历史项目偏好、护理周期、LTV |
| `itemProjectFitScore` | 商品与项目适配 | 商品功效、BOM、搭配项目 | 项目功效、商品库存、技能匹配 |
| `timingScore` | 当前时机 | 节令、天气、商品到期、复购周期 | 可约时段、节日、护理周期 |
| `operationFitScore` | 承接能力 | 库存是否足够、可调拨 | 美容师排班、房间/仪器资源 |
| `profitScore` | 利润质量 | 毛利、折扣后利润、损耗避免 | 项目毛利、疗程转化空间 |
| `channelFitScore` | 触达渠道适配 | 小程序、短信、顾问 | 小程序预约、顾问任务 |
| `fatiguePenalty` | 触达疲劳惩罚 | 近期已推同类商品 | 近期已推同类项目 |
| `riskPenalty` | 风险惩罚 | 过敏、合规、库存不足 | 过度预约、技能不匹配 |

### 5.5 商品临期评分

```text
expiryUrgencyScore =
  daysToExpiryScore * 0.35
  + sellThroughGapScore * 0.30
  + lossAmountScore * 0.20
  + productProjectBundleScore * 0.15
```

| 指标 | 计算口径 |
| --- | --- |
| `daysToExpiryScore` | 0-7 天 100 分，8-15 天 85 分，16-30 天 70 分，31-60 天 45 分 |
| `sellThroughGapScore` | `max(0, 临期库存 - 到期前自然销量预测)` 越大分越高 |
| `lossAmountScore` | `预计缺口库存 * 成本价` 越高分越高 |
| `productProjectBundleScore` | 商品能否和当前热门项目、可约项目、客户肤质匹配 |

推荐动作选择：

- 缺口小：小程序推荐位、顾问顺带推荐。
- 缺口中：项目套餐、会员权益、满额赠。
- 缺口大：跨店调拨、限时活动、员工内购、报损预警。

### 5.6 商品消耗增长评分

```text
growthScore =
  recentGrowthRateScore * 0.30
  + forecastStockoutScore * 0.25
  + projectDemandScore * 0.20
  + repeatPurchasePotentialScore * 0.15
  + marginScore * 0.10
```

建议判断口径：

- 近 7 天日均消耗 vs 前 28 天日均消耗，增长超过 30% 进入候选。
- z-score 大于 2 视为异常增长，需要区分真实需求和异常耗用。
- 若增长来自项目 BOM 消耗，同步读取未来 7/14 天预约和排班，预测耗材缺口。
- 若增长来自客户购买，进入商品复购和小程序热销推荐。

输出建议：

- 真实增长且库存充足：推荐热销商品或项目后带货。
- 真实增长但库存不足：先生成补货建议，再控制营销曝光。
- 异常消耗：生成经营预警，不生成客户营销活动。

### 5.7 美容师排期不满评分

```text
idleCapacityScore =
  emptySlotScore * 0.35
  + customerDemandScore * 0.25
  + beauticianSkillFitScore * 0.15
  + lowPeakValueScore * 0.15
  + conversionProbabilityScore * 0.10
```

判断口径：

- 未来 1-7 天按时段计算预约占用率。
- 占用率低于 60% 且可服务美容师不少于 1 人，进入候选。
- 黄金时段不默认降价，低峰时段可以配置低峰权益。
- 如果某美容师空闲，但其擅长项目和客户偏好高度匹配，优先生成顾问任务。

推荐动作：

- 低峰专享券：仅可用于指定日期和时段。
- 顾问邀约：优先给曾服务客户、护理周期到期客户。
- 小程序余位提醒：展示可约时段和项目。
- 美容师专属活动：用美容师擅长项目承接空档。

### 5.8 项目与商品组合评分

```text
bundleFitScore =
  projectBomRelationScore * 0.25
  + skinGoalFitScore * 0.25
  + associationRuleLiftScore * 0.20
  + inventoryFitScore * 0.15
  + marginAfterOfferScore * 0.15
```

算法来源：

- BOM 关系：项目真实使用的产品天然具备搭配解释。
- 关联规则：消费 A 项目的人，有多大比例会购买 B 商品。
- 协同过滤：相似客户还购买过哪些商品或项目。
- 护理知识库：根据肤质、问题、季节、项目功效推荐组合。
- 库存策略：临期或库存充足商品优先，但必须经过肤质和合规校验。

## 6. 算法分层

### 6.1 MVP：规则引擎 + 可解释评分

先使用规则引擎，原因是当前业务数据仍在补齐阶段，规则更可控、可解释、容易让门店信任。

MVP 必须覆盖：

- 商品临期消化。
- 商品补货复购。
- 商品消耗增长预警。
- 美容师排期不满。
- 项目护理周期到期。
- 项目与商品组合推荐。

### 6.2 增强：统计模型

当订单、核销、触达、行为事件达到一定量后，引入统计模型：

- RFM / LTV / 流失模型：判断客户价值和触达策略。
- 关联规则 Apriori / FP-Growth：发现项目、商品、卡项的连带关系。
- ABC/XYZ 库存分析：识别高价值稳定品、波动品、长尾滞销品。
- 移动平均 / EWMA / z-score：识别商品消耗增长和异常消耗。
- 生存分析 / 补货周期模型：预测客户何时需要再次购买商品。
- 容量利用率模型：预测未来某时段是否需要做低峰促销。

### 6.3 高级：个性化优化

中后期可引入：

- Uplift Model：识别“不给券也会买”的客户，避免无效让利。
- Contextual Bandit：自动学习不同人群、渠道、优惠力度的最优组合。
- 动态定价保护：在库存压力、排期压力和品牌价格体系之间自动权衡。
- 多目标优化：同时优化收入、毛利、库存损耗、排期利用率和触达疲劳。

## 8. 推荐类型设计

### 8.1 商品临期消化推荐

规则编码：`product_expiry_clearance`

触发条件：

- `StockBatch.expiryDate` 距今天小于等于 60 天。
- 批次库存大于 0。
- 预测自然销量小于临期库存。
- 商品仍在合规可售期内。

推荐逻辑：

1. 先判断是否有其他门店同 SKU 消耗更快，若有，优先推荐跨店调拨。
2. 若可用项目 BOM 或护理知识库匹配项目，推荐“项目 + 商品”组合。
3. 若客户近期购买过同功效商品或做过相关项目，进入目标客群。
4. 若临期风险高但客户匹配不足，推荐员工内购或报损预警，不强行营销客户。

推荐动作：

- 创建限时活动。
- 创建顾问任务。
- 小程序推荐位。
- 生成跨店调拨建议。

### 8.2 商品消耗增长推荐

规则编码：`product_consumption_growth`

触发条件：

- 商品近 7/14 天销售或 BOM 消耗明显增长。
- 增长率超过 30%，或 z-score 大于 2。
- 当前库存可售天数小于安全阈值，或存在商品热销机会。

推荐逻辑：

- 若库存不足：生成补货建议和断货风险，不扩大营销。
- 若库存充足：推荐热销商品曝光、项目后带货、复购提醒。
- 若增长来自某项目：推荐对应项目套餐和家居维养商品。
- 若增长异常：生成“消耗异常复核”运营卡。

推荐动作：

- 商品补货。
- 热销推荐位。
- 项目后带货话术。
- 自动复购提醒。

### 8.3 商品补货复购推荐

规则编码：`product_replenishment`

触发条件：

- 客户购买某商品后达到预计消耗周期。
- 最近未再次购买同 SKU 或同功效商品。
- 客户触达频控允许。

消耗周期默认建议：

| 商品类型 | 默认周期 |
| --- | --- |
| 面膜 | 21-30 天 |
| 精华 | 30-45 天 |
| 洁面 | 45-60 天 |
| 面霜 | 45-60 天 |
| 防晒 | 30-45 天，夏季缩短 |
| 身体护理 | 45-90 天 |

推荐动作：

- 小程序补货提醒。
- 同款复购券。
- 同功效升级商品。
- 与下一次护理项目组合推荐。

### 8.4 商品滞销积压推荐

规则编码：`product_slow_moving`

触发条件：

- 库龄超过阈值，例如 90/120 天。
- 近 30 天销量低于阈值。
- 库存金额较高或库存周转天数过长。

推荐动作：

- 与畅销项目组合。
- 第二件优惠或会员专享包。
- 小程序专题页。
- 跨店调拨。
- 若毛利不足或过敏风险高，不推荐客户促销，只给经营提醒。

### 8.5 美容师排期不满推荐

规则编码：`project_idle_capacity`

触发条件：

- 未来 1-7 天某门店或某美容师预约占用率低于阈值。
- 至少存在可预约时段和可服务项目。
- 目标客户中存在护理周期到期、近期浏览未预约、轻度沉睡或高响应人群。

推荐动作：

- 低峰专享项目券。
- 美容师老客回访任务。
- 小程序余位推荐。
- 顾问一键邀约。

默认策略：

- 优先工作日低峰，不打扰黄金时段价格。
- 优先推可快速决策、服务时长可控、库存/BOM 充足的项目。
- 优先命中曾由该美容师服务过且满意度高的客户。

### 8.6 项目护理周期推荐

规则编码：`project_cycle_due`

触发条件：

- 客户上次完成某项目后达到护理周期。
- 当前无未来预约。
- 对应项目近期有可预约容量。

推荐动作：

- 周期提醒。
- 项目复购券。
- 项目升级包。
- 次卡续费或疗程卡承接。

### 8.7 高毛利项目提升推荐

规则编码：`project_high_margin_upsell`

触发条件：

- 项目毛利较高，项目可服务产能充足。
- 客户 LTV 或消费能力较高。
- 客户肤质、护理目标、历史项目偏好匹配。

推荐动作：

- 基础项目升级。
- 高端护理体验。
- 疗程卡推荐。
- VIP 顾问邀约。

### 8.8 项目商品组合推荐

规则编码：`project_product_bundle`

触发条件：

- 项目与商品存在 BOM、功效或关联规则关系。
- 商品库存充足，或存在可控临期消化压力。
- 项目有可预约容量。
- 目标客户画像匹配。

推荐动作：

- 项目套餐。
- 项目后带回家装。
- 小程序 H5 活动页。
- 顾问话术。

## 11. 风控与频控

### 11.1 客户触达频控

- 同客户同商品 14 天内最多触达 1 次。
- 同客户同项目 7 天内最多触达 1 次。
- 同客户同渠道 1 天内最多触达 1 次。
- 已预约客户不再推同类预约活动。
- 已购买客户不再推同款临期活动，除非进入补货周期。
- 近 30 天触达多次未响应的客户降低推荐分。

### 11.2 库存风控

- 库存不足时不生成对外营销活动，只生成补货建议。
- 临期商品必须校验可售期、禁忌、过敏风险。
- 预计活动销量不得超过可用库存。
- 多门店调拨优先于深度折扣。
- 临期严重时停止客户营销，转为内部处理或报损。

### 11.3 排班风控

- 推荐活动必须绑定可预约时段或动态校验余位。
- 不能超卖美容师工时。
- 高技能项目必须校验美容师技能。
- 低峰权益只在指定低峰时段可用。
- 已有预约、请假、不可用时段必须排除。

### 11.4 价格与品牌风控

- 高端项目和高价值客户不默认低价促销。
- 折扣后毛利必须高于底线。
- 限时权益要有清晰理由，例如临期、低峰、会员日，避免长期价格锚点下移。
- 优惠券、赠品、套餐权益要能被活动页和收银核销一致识别。

## 12. 推荐卡示例

### 12.1 商品临期示例

标题：`修护面膜 30 天内临期，建议绑定补水护理消化 55 盒`

推荐理由：

- 当前门店修护面膜临期库存 80 盒。
- 按近 30 天销量预测，到期前自然消化约 25 盒，缺口 55 盒。
- 补水护理、敏感修护项目与该商品功效匹配。
- 312 位干性/敏感肌客户近 60 天有护理需求或浏览行为。

推荐动作：

- 创建“补水护理 + 修护面膜带回家”活动。
- 面向干性/敏感肌、高营销响应、近期未预约客户。
- 小程序推荐位 + 顾问任务。
- 预计避免损耗：`55 * 成本价`。

风险提示：

- 敏感肌客户需校验过敏史。
- 活动库存上限不能超过临期可用库存。

### 12.2 商品消耗增长示例

标题：`防晒乳消耗 14 天增长 46%，建议补货并开启夏季防护复购提醒`

推荐理由：

- 近 14 天日均销量较前 28 天增长 46%。
- 当前库存预计 11 天后低于安全库存。
- 夏季防晒浏览行为和控油项目预约同步增长。

推荐动作：

- 先生成补货建议。
- 库存安全后开启防晒复购提醒。
- 在夏季控油/美白项目详情页推荐防晒搭配。

风险提示：

- 当前库存不足时不建议做大范围促销。

### 12.3 排期不满示例

标题：`周三下午低峰预约占用率 38%，建议推低峰补水护理`

推荐理由：

- 下周三 14:00-17:00 可预约工时 18 小时，已预约 7 小时。
- 3 位美容师可服务补水护理和敏感修护项目。
- 126 位客户进入 28 天护理复购窗口，其中 48 位近期浏览项目未预约。

推荐动作：

- 创建低峰专享护理券，仅限周三 14:00-17:00 使用。
- 先触达高响应客户和浏览未预约客户。
- 同步生成美容师老客回访任务。

风险提示：

- 低峰券不可用于周末或晚间黄金时段。
- 需实时校验余位，避免超卖。

- 店长可看本周推荐带来的库存消化和排期填充。

### Phase 4：算法优化

目标：从规则推荐升级为个性化推荐。

范围：

- 关联规则。
- 补货周期模型。
- 排期容量预测。
- Uplift 优惠优化。
- Contextual Bandit 渠道和权益优化。

交付：

- 推荐分数支持模型权重。
- 推荐理由保留可解释证据。
- 支持 A/B 测试和自动调参。

- 系统至少能生成 6 类推荐：商品临期、商品补货、商品消耗增长、排期不满、护理周期、项目商品组合。
- 每张推荐卡必须包含推荐理由、数据证据、推荐动作、目标客户、预计收益和风险提示。
- 商品推荐必须包含库存或批次证据。
- 项目推荐必须包含排期或客户需求证据。

### 14.2 执行承接

- 商品临期推荐可生成活动草稿或顾问任务。
- 商品补货推荐可生成自动营销规则草稿。
- 排期不满推荐可生成低峰活动草稿，并绑定可用时段。
- 项目商品组合推荐可带入活动页推荐商品和项目。

### 14.3 风控

- 库存不足不允许生成大范围营销活动。
- 临期商品超过合规可售边界不允许对客户销售。
- 排期活动不能超卖美容师时段。
- 同客户触达满足频控要求。
- 高价值客户不默认强折扣。

### 14.4 归因

- 推荐采纳、活动创建、自动规则创建、触达、预约、下单、核销能关联到推荐来源。
- 商品临期推荐能统计消化库存、避免损耗。
- 排期推荐能统计填充时段和新增收入。
- 项目推荐能统计预约率、到店率、核销率、客单价和毛利。

## 15. 优先级建议

P0：

- 商品临期消化推荐。
- 美容师排期不满推荐。
- 商品补货复购推荐。
- 项目护理周期推荐。

P1：

- 商品消耗增长推荐。
- 项目商品组合推荐。
- 商品滞销积压推荐。
- 高毛利项目提升推荐。

P2：

- 新品冷启动推荐。
- 多门店调拨优先算法。
- Uplift 优惠优化。
- Contextual Bandit 自动权益优化。

## 16. 结论

商品与项目智能营销推荐的核心，不是简单把库存预警或排班空档展示给运营，而是把经营信号转成可执行动作：

- 商品临期：从“损耗预警”变成“合规可控的消化方案”。
- 商品消耗增长：从“库存变化”变成“补货、热销推荐和复购机会”。
- 排期不满：从“美容师空闲”变成“低峰时段可售产能”。
- 项目商品组合：从“单品促销”变成“完整护理方案和客单提升”。

建议先用规则引擎和可解释评分落地 P0 场景，优先打通推荐卡到活动草稿、自动营销规则和顾问任务的执行链路。等触达和转化数据积累后，再引入关联规则、补货周期模型、Uplift 和 Bandit 优化权益，逐步形成真正的经营智能推荐中枢。
