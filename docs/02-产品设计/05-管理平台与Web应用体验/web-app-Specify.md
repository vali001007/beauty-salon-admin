# 美业智能助手 Web App — 产品规格文档

## 1. 产品概述

### 1.1 产品定位

美业智能助手是一款面向美业门店日常运营的对话式 Web 应用。它将现有后台管理系统的全部业务能力（客户、营销、门店、商品、订单、库存、系统设置）封装在一个自然语言对话界面中，让店长和店员通过"说人话"完成原本需要在多个页面间跳转才能完成的管理操作。

### 1.2 目标用户

| 角色 | 典型画像 | 核心诉求 |
|------|---------|---------|
| 店长 (store_manager) | 30-45岁，管理1-3家门店，不熟悉复杂后台 | 快速查看经营数据、处理审批、管理排班和营销 |
| 美容师 (beautician) | 20-35岁，手机重度用户 | 查看自己的排班和预约、确认客户到店 |
| 收银员 (cashier) | 25-40岁，操作频率高 | 快速开单、核销次卡、查询订单 |
| 库存管理员 (inventory_manager) | 25-45岁 | 查库存、处理入库/采购/调拨、关注临期 |

### 1.3 核心价值

- **零学习成本**：用自然语言代替菜单导航，新员工无需培训即可上手
- **操作效率提升**：一句话完成原本需要 3-5 次页面跳转的操作
- **移动优先**：手机端单手可完成 80% 的日常管理工作
- **智能辅助**：主动推送异常提醒（临期产品、低库存、客户流失预警）

## 2. 交互设计理念

### 2.1 设计原则

| 原则 | 说明 |
|------|------|
| 对话即操作 | 每一轮对话都应产生可见的业务结果（数据卡片、表格、操作确认），而非仅返回纯文本 |
| 渐进式披露 | 先给摘要，用户追问再展开详情；避免一次性倾倒大量数据 |
| 确认再执行 | 所有写操作（创建/修改/删除）必须经过明确的确认步骤，展示变更摘要后由用户点击确认 |
| 上下文连贯 | 同一会话内保持业务上下文，支持代词指代（"把她的等级改成VIP"） |
| 快捷兜底 | 对话框下方常驻快捷入口，覆盖高频操作，用户不必每次都打字 |

### 2.2 与传统后台的核心区别

| 维度 | 传统后台 | 对话式助手 |
|------|---------|-----------|
| 导航方式 | 侧边栏菜单 → 页面 → 表格 → 操作按钮 | 直接描述意图，系统自动路由到对应功能 |
| 信息密度 | 整页表格，一次展示几十条 | 按需返回摘要卡片，追问展开 |
| 操作路径 | 点击 → 弹窗 → 填表 → 提交 | 说 → 确认卡片 → 完成 |
| 学习成本 | 需要了解菜单结构和各页面功能 | 用自然语言描述需求即可 |
| 多任务切换 | 需要在不同页面间跳转 | 在同一对话流中连续处理不同业务 |

## 3. 用户角色与权限

复用现有 `src/config/permissions.ts` 的 `ROLE_PERMISSIONS` 权限体系。AI 助手在执行任何操作前，先校验当前用户的 `permissions` 数组。

### 3.1 角色能力矩阵

| 能力域 | super_admin | store_manager | beautician | cashier | inventory_manager |
|--------|:-----------:|:-------------:|:----------:|:-------:|:-----------------:|
| 仪表盘数据查询 | 全部门店 | 本店 | 个人相关 | 本店订单 | 本店库存 |
| 客户管理 | ✓ | ✓ | — | — | — |
| 智能营销 | ✓ | ✓ | — | — | — |
| 门店项目/美容师管理 | ✓ | ✓ | — | — | — |
| 排班查询 | ✓ | ✓ | 仅自己 | — | — |
| 排班编辑 | ✓ | ✓ | — | — | — |
| 项目预约 | ✓ | ✓ | 仅自己 | — | — |
| 商品/次卡管理 | ✓ | ✓ | — | — | — |
| 订单管理 | ✓ | ✓ | — | ✓ | — |
| 次卡开卡/核销 | ✓ | ✓ | — | ✓ | — |

### 3.2 权限拦截对话示例


## 4. 核心功能模块

### 4.1 仪表盘 — 经营数据查询

**用户指令示例：**
- "今天营业额多少？"
- "这周新增了几个客户？"
- "对比上月的业绩"
- "哪个美容师业绩最好？"

**AI 响应方式：**
- 数据摘要卡片（关键指标 + 环比变化箭头）
- 迷你图表（折线/柱状，嵌入消息气泡）
- 排行榜列表

**示例对话：**
```
用户: "今天店里情况怎么样？"

      ┌──────────────────────────┐
      │ 营业额  ¥12,580  ↑15%   │
      │ 到店客户  18人   ↑3人    │
      │ 新客户    3人             │

      └──────────────────────────┘
      "还有3个预约未完成，需要查看详情吗？"
```

**用户指令示例：**
- "查一下张美丽的信息"

- 操作确认卡片（修改/删除前展示变更内容）

**示例对话：**
```
用户: "新增一个客户"
助手: "好的，请提供以下信息：
      [内联表单]

      手机号*: [______]
      性别: [女 ▼]
      客户来源: [门店 ▼]

      [提交] [取消]"
```

- 删除客户需二次确认 + 密码验证（复用 PasswordConfirmDialog 逻辑）
- 批量操作展示影响范围后确认

### 5.2 上下文管理

采用会话栈（Context Stack）模式管理多轮对话状态：

```typescript
interface ConversationContext {
  // 当前会话的实体引用（支持代词指代）
  entities: {
    customer?: { id: number; name: string };
    beautician?: { id: number; name: string };
    product?: { id: number; name: string };
    appointment?: { id: number };
  };

  // 待确认的操作（用户中途切换意图时保留）
  pendingAction?: {
    type: IntentType;
    params: Record<string, any>;
    prompt: string; // "要继续创建新客户吗？"
  };

  // 最近 N 轮对话摘要（用于上下文理解）
  recentTurns: Array<{
    role: 'user' | 'assistant';
    intent?: IntentType;
    entities?: Record<string, any>;
    summary: string;
  }>;
}
```

**上下文规则：**
1. 实体引用在同一会话内持续有效，直到用户明确切换对象
2. 代词解析优先匹配最近提到的同类实体（"她" → 最近提到的女性客户）
3. 意图切换时，前一个未完成的操作存入 `pendingAction`，完成新操作后提示恢复
4. 会话超过 30 分钟无活动自动清空上下文

### 5.3 多轮对话场景

**场景一：渐进式信息收集**
```
用户: "帮客户预约"
助手: "好的，请问是哪位客户？"
用户: "张三"
助手: "张三，想预约什么项目？"
用户: "深层清洁"
助手: "请问希望哪个时间段？需要指定美容师吗？"
用户: "明天下午3点，小王"
助手: [预约确认卡片]
      客户: 张三 | 项目: 深层清洁 (60分钟)
      美容师: 小王 | 时间: 明天 15:00-16:00
      [确认预约] [修改] [取消]
```

**场景二：上下文指代**
```
用户: "查一下张三的信息"
助手: [张三的客户详情卡片]
用户: "她最近消费了什么？"          ← "她"指代张三
助手: [张三的消费记录列表]
用户: "给她的次卡核销一次面部护理"   ← "她"仍指代张三
助手: [核销确认卡片]
```

**场景三：中途切换意图**
```
用户: "创建一个新客户"
助手: "好的，请告诉我客户的姓名和手机号。"
用户: "等一下，先帮我查一下今天的预约"    ← 意图切换
助手: [今日预约列表]
      "预约信息如上。要继续创建新客户吗？"  ← 保留 pendingAction
用户: "继续"
助手: "好的，请告诉我新客户的姓名和手机号。"
```

**场景四：纠错与修改**
```
用户: "帮张三预约明天上午10点"
助手: [预约确认卡片] 时间: 明天 10:00-11:00
用户: "改成下午2点"
助手: [更新后的确认卡片] 时间: 明天 14:00-15:00
      [确认预约] [修改] [取消]
```

## 6. UI/UX 规格

### 6.1 页面布局（移动优先）

```
┌─────────────────────────────────┐
│  顶部栏 (56px)                   │
│  [≡] 美业智能助手    [门店▼] [👤] │
├─────────────────────────────────┤
│                                 │
│  消息区域（可滚动）              │
│                                 │
│  [AI消息 - 左对齐]              │
│  [用户消息 - 右对齐]            │
│  [AI消息 + 数据卡片]            │
│  [AI消息 + 内联表单]            │
│  [用户消息 - 图片]              │
│  [用户消息 - 语音波形]          │
│                                 │
├─────────────────────────────────┤
│  快捷操作栏 (按角色动态显示)     │
│  [💳核销] [📅预约] [🛒收银]     │
├─────────────────────────────────┤
│  输入区域（多模态）              │
│  [📷] [🎤] [输入框...] [发送 ➤] │
└─────────────────────────────────┘
```

**响应式断点：**
- 移动端 (<768px)：全屏对话，快捷栏横向滚动
- 平板/桌面 (≥768px)：左侧历史会话列表 + 右侧对话区域

### 6.2 多模态输入

对话助手支持三种输入方式，可自由切换或组合使用：

#### 6.2.1 文本输入
- 标准键盘输入，支持中文 / 英文 / 表情符号
- 回车发送，Shift+Enter 换行
- 输入过程中自动提示快捷指令（如输入"查客户"自动弹出匹配的客户列表）

#### 6.2.2 语音输入 🎤
- 长按麦克风按钮录音，松开发送；支持取消（上滑手势）
- 语音识别采用 **Whisper API** 或浏览器原生 `SpeechRecognition`（降级方案）
- 识别后展示文字草稿，用户可编辑后确认发送
- 语音消息气泡显示波形图 + 时长 + 识别文字
- 支持录音超时提示（默认 60 秒上限）
- 适用场景：美容师双手忙碌时口述预约、店长开车途中查询数据

#### 6.2.3 图片输入 📷
- 从相册选择 / 拍照上传 / 粘贴截图
- 支持单次最多 3 张图片
- 通过 **GPT-4o Vision** 理解图片内容，提取结构化信息
- 典型应用：

### 7.3 AI/NLP 集成方案

#### 7.3.1 文本意图识别

采用 OpenAI function calling 实现意图识别和参数提取：

```typescript
// engine/IntentClassifier.ts
const tools = [
  {
    type: 'function',
    function: {
      name: 'query_customer',
      description: '查询客户信息',
      parameters: {
        type: 'object',
        properties: {
          keyword: { type: 'string', description: '客户姓名或手机号' },
          filters: {
            type: 'object',
            properties: {
              memberLevel: { type: 'string' },
              lastVisitDays: { type: 'number', description: '最近N天未到店' },
            }
          }
        }
      }
    }
  },
  {
    type: 'function',
    function: {
      name: 'create_appointment',
      description: '创建预约',
      parameters: {
        type: 'object',
        properties: {
          customerName: { type: 'string' },
          projectName: { type: 'string' },
          dateTime: { type: 'string' },
          beauticianName: { type: 'string' },
        },
        required: ['customerName', 'projectName']
      }
    }
  },
  // ... 其他 17 个意图的 function 定义
];
```

**调用流程：**
1. 用户输入 → 附加上下文摘要 → 发送给 OpenAI
2. OpenAI 返回 function call（意图 + 参数）
3. `ActionExecutor` 根据意图调用对应的 `src/api/` 模块
4. `ResponseBuilder` 将 API 结果转换为消息卡片
5. 渲染到对话界面

**降级方案：**
- OpenAI 不可用时，使用本地关键词匹配做基础意图识别
- 无法识别意图时，展示快捷操作面板引导用户选择

#### 7.3.2 语音识别（Speech-to-Text）

| 方案 | 使用场景 | 优缺点 |
|------|---------|-------|
| **Whisper API** (首选) | 主方案，高识别率 | 准确率高，支持多语言；需联网，每分钟约 ¥0.04 |
| **Web Speech API** (降级) | 浏览器原生，离线可用 | 免费；识别率受浏览器和环境影响，iOS Safari 不支持 |

处理流程：
1. 录音 → 本地 WAV/MP3 编码
2. 上传到 Whisper API → 返回识别文字
3. 识别文字注入文本意图识别链路，复用 function calling
4. 原始语音文件保存在消息记录中（用户可重放）

#### 7.3.3 图片理解（Vision）

使用 **GPT-4o** 的多模态能力处理图片输入：

```typescript
// engine/VisionService.ts
async function analyzeImage(imageBase64: string, userPrompt?: string) {
  const response = await openai.chat.completions.create({
    model: 'gpt-4o',
    messages: [{
      role: 'user',
      content: [
        { type: 'text', text: userPrompt || '请识别图片中的关键信息，并判断可能的业务场景（客户录入/采购单/库存盘点/肌肤档案等）' },
        { type: 'image_url', image_url: { url: `data:image/jpeg;base64,${imageBase64}` } }
      ]
    }],
    tools, // 复用文本意图的 function 定义
  });
  return response;
}
```

典型处理链路：

| 图片类型 | 识别结果 | 后续动作 |
|---------|---------|---------|
| 名片/证件 | 姓名、手机号、身份证号、生日 | 触发 `create_customer` 意图，预填表单 |
| 采购单据 | 商品名、数量、单价、供应商 | 触发 `create_stock_in` 或 `create_purchase`，预填明细 |
| 货架照片 | 商品识别 + 大致数量估算 | 与系统库存对比，生成盘点差异报告 |
| 客户肌肤照 | 肌肤类型、问题点 | 关联到客户档案的肌肤分析 |

#### 7.3.4 多模态消息的统一处理

`ChatEngine.sendMessage` 接受统一的消息对象：

```typescript
interface UserMessage {
  text?: string;              // 文本内容
  voice?: Blob;               // 语音文件
  images?: File[];            // 图片列表（最多 3 张）
  voiceTranscript?: string;   // 语音识别后的文字（由 SpeechService 填充）
}

// 处理流程
async sendMessage(msg: UserMessage) {
  // 1. 语音 → 文字
  if (msg.voice) {
    msg.voiceTranscript = await speechService.transcribe(msg.voice);
  }

  // 2. 图片 → 结构化信息
  let imageContext = '';
  if (msg.images?.length) {
    const results = await Promise.all(msg.images.map(img => visionService.analyze(img)));
    imageContext = results.map(r => r.summary).join('\n');
  }

  // 3. 合成最终的意图识别输入
  const finalText = [msg.text, msg.voiceTranscript, imageContext]
    .filter(Boolean).join('\n');

  // 4. 走统一的意图识别
  return intentClassifier.classify(finalText, context);
}
```
