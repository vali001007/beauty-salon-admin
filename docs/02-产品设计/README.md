# 产品设计文档目录

本目录用于沉淀产品方案、PRD、调研分析、模块设计和运营策略文档。根目录只保留索引与一级主题目录，新增文档优先按主题放入对应子目录。

## 一级目录

- `01-AI智能体与问数能力/`：Ami Agent、Ami Aura、AI 问数、Text-to-SQL、Agent 编排、底层 API + Agent 模式、Agent 评测与智能体产品需求。
- `02-营销增长与客户触达/`：拓客、插秧营销、智能营销、自动营销规则、推荐算法、权益资产、营销 H5、小程序触达与客户行为整合。
- `03-经营财务与人效管理/`：经营利润、提成规则、成本核算、美容师排班、人效管理与财务口径相关方案。
- `04-行业数据供应链与库存/`：行业数据平台、供应链平台、库存管理、主数据、数据字典、数据库/知识库建设。
- `05-管理平台与Web应用体验/`：管理平台性能、Web App 体验、Figma / Specify 设计提示词与前端体验类方案。
- `99-临时草稿与待确认/`：内容过短、命名实验或暂无法判断用途的临时草稿；后续确认价值后再归入正式主题目录。

## 既有专项目录

- `Admin Panel/`：管理端专项设计资料。
- `Ami_Glow/`：Ami Glow 专项设计资料。
- `Ami_Greet/`：Ami Greet 专项设计资料。

## 归档规则

- 新增产品文档不要直接放在根目录，先判断业务主题后进入对应一级目录。
- 一份文档如果同时涉及多个模块，按主要交付对象归档；其他关联模块可在文档内加引用说明。
- 调研、PRD、方案、操作说明可以放在同一主题目录内；开发任务拆解优先放到 `docs/03-开发计划/`。
- `99-临时草稿与待确认/` 只作为临时缓冲区，不作为正式产品方案沉淀位置。
