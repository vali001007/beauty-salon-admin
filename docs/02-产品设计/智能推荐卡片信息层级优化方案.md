# 智能推荐卡片信息层级优化方案

## 背景

当前智能推荐卡片中，“权益”“匹配项目/商品”“渠道”被做成三个等宽卡片。但实际信息量不均衡：

- 权益信息包含权益内容、权益资产来源、适配度、主推/备选权益，内容最多。
- 匹配项目/商品通常只有一个名称。
- 渠道通常只有 2-3 个触达渠道。

等宽布局会导致权益卡拥挤，项目和渠道卡空白过多，视觉上主次不分。

## 优化目标

- 让运营一眼看清“这条推荐主推什么权益”。
- 项目/商品和渠道作为辅助信息，不抢权益主位。
- 减少卡片高度和空白，提升列表扫描效率。
- 保留“权益资产库”与推荐之间的关联表达，方便后续活动投放和效果复盘。

## 推荐布局

### 第一层：推荐标题与匹配度

保持现状：

- 优先级：紧急 / 推荐 / 机会。
- 机会标签：流失预警、库存机会、策略推荐等。
- 推荐标题。
- 匹配度。

### 第二层：主行动信息区

改为“左主右辅”的结构，不再三列等分。

建议布局：

```text
[权益主卡，占 60%-70%宽度]  [辅助信息区，占 30%-40%宽度]

权益主卡：
  主推权益：推荐成功双方各得护理券
  来源：权益资产库 / 老带新双边礼 / 适配度64
  备选：专属顾问服务 + 优先预约、指定商品临期专享价

辅助信息区：
  项目/商品：玻尿酸保湿精华
  渠道：小程序、顾问跟进
```

### 权益主卡展示规则

权益卡作为主卡展示，宽度更大，内容分层：

- 第一行：`主推权益：xxx`
- 第二行：`权益资产库：xxx · 适配度 xx`
- 第三行：备选权益，以小标签展示。

不建议继续使用：

- `权益：xxx`
- `来自权益资产库：xxx，适配度 xx`
- `主推 · xxx`

推荐改成更接近业务语言：

- `主推权益`
- `权益来源`
- `备选权益`

### 项目/商品展示规则

项目/商品不是权益方案，只在有真实匹配对象时展示。

展示方式建议从卡片降级为轻量行或标签：

```text
匹配项目/商品：玻尿酸保湿精华
```

如果没有真实项目/商品匹配，则不展示该字段。

### 渠道展示规则

渠道信息较短，不需要独立大卡片。

建议展示为轻量标签：

```text
触达渠道：小程序 / 顾问跟进
```

渠道应放在辅助信息区，和项目/商品并列。

## 经营指标区

保留当前紧凑标签样式：

```text
预计营收 ¥6,556    建议周期 30天    批次 2026/6/17 12:00
库存54    剩12天    缺口54    毛利¥2,860    避损¥9,072
```

原则：

- 数字指标一行展示。
- 去掉“预计毛利”“预计避免损耗”等重复前缀。
- 库存、周期、批次不与权益信息混在一起。

## 操作区

保持现在的动作顺序：

```text
目标客户    开启自动触达    发一次活动    下发终端跟进    跟进记录
```

其中：

- `目标客户`：查看命中客户名单。
- `开启自动触达`：适合规则长期触发时展示。
- `发一次活动`：适合一次性活动投放时展示。
- `下发终端跟进`：需要人工跟进时展示。

## MVP改造范围

### 前端改造

- 调整 `MarketingRecommendation.tsx` 推荐卡片的第二层布局。
- 权益卡从等宽卡片改为主卡，占更多宽度。
- 项目/商品、渠道合并到辅助信息区。
- 没有真实项目/商品时不展示项目/商品字段。
- 优化权益文案：
  - `权益` -> `主推权益`
  - `来自权益资产库` -> `权益来源`
  - `主推 / 备选` -> `备选权益`

### 不改动范围

- 不改推荐算法。
- 不改权益匹配接口。
- 不改活动创建逻辑。
- 不改自动触达逻辑。

## 验收标准

- 权益内容较多时不再挤在三等分小卡片里。
- 项目/商品和渠道不再占用与权益相同的视觉权重。
- 没有真实项目/商品时，不显示空泛的建议内容。
- 运营能在 3 秒内判断：
  - 这条推荐给谁。
  - 主推什么权益。
  - 通过什么渠道触达。
  - 下一步该点哪个动作。
