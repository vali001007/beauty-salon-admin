# 美容行业成熟热门权益资产库清单

更新时间：2026-06-17

适用模块：智能营销 > 权益资产库 / 推荐动作卡 / Ami Glow 小程序活动 / Ami Aura Lite 终端核销

## 1. 设计口径

本清单用于为 Ami Core 内置一套可推荐、可发布、可领取、可预约、可核销的美业权益资产库。资产口径尽量贴合当前 `Promotion` 模型字段：`code`、`name`、`description`、`discountText`、`type`、`scenario`、`audienceTags`、`thresholdAmount`、`discountAmount`、`discountRate`、`giftText`、`validDays`、`maxIssueCount`、`estimatedCost`、`grossMarginGuard`、`stackable`、`metadata`。

调研来源包括：有赞美业小程序对“线上预约 + 团购进店 + 新客套餐”的玩法说明，国内美业系统对次卡/体验卡/裂变券/生日关怀/定向发券的描述，以及 Zenoti、Booksy、GlossGenius、Mindbody 等海外美业 SaaS 对 referral、loyalty、birthday、gift card、holiday package、off-peak booking、add-on service 等成熟活动的整理。参考链接见第 7 节。

## 2. 类型枚举建议

| type | 中文口径 | 适合权益 |
| --- | --- | --- |
| `money_off` | 满减券 / 立减券 | 新客、复购、沉睡唤醒、到期提醒 |
| `discount` | 折扣券 | 生日月、会员日、节日预售、指定项目 |
| `trial_price` | 体验价 | 新客首单、浏览未预约、爆品引流 |
| `gift` | 赠品 / 赠服务 | 生日礼、续卡礼、评价礼、到店礼 |
| `package_upgrade` | 套餐升级 | 季节护理、项目组合、加项升级 |
| `member_privilege` | 会员权益 | VIP、等级、优先预约、专属顾问 |
| `stored_value_bonus` | 储值赠送 | 充值、预售、礼品卡、年卡 |
| `points_reward` | 积分 / 集点 | 会员忠诚、评价、分享、复购 |
| `referral_reward` | 推荐有礼 | 老带新、双边奖励、裂变券 |
| `group_deal` | 拼团 / 同行 | 闺蜜同行、亲友团、节日组团 |

## 6. 落库与产品化建议

1. 资产库分两层：系统内置资产 `source=system`，门店自建资产 `source=store`。
2. P0 先内置 24 个高频资产；完整 88 个作为后续扩展库。
3. 每个资产默认 `approvalStatus=approved`、`status=active`、`stackable=false`，门店复制后可编辑。
4. 低价和大额满减类必须配置 `grossMarginGuard`，至少包含最低毛利、适用项目、不可叠加、每人领取上限。
5. 推荐引擎输出不要只给“建议做活动”，应直接给资产卡：目标客户、推荐资产、预计成本、有效期、发布渠道、后续核销动作。
6. 小程序展示以“领取权益/立即预约”为主动作；终端展示以“核销权益/转预约/转开卡”为主动作。
7. 对高价值会员优先推荐服务权益、优先权、专属顾问，少用大额折扣。
8. 对新客和沉睡客户允许更强利益点，但必须限定首单、短有效期、指定项目和不可叠加。
