# 美业插秧营销需求文档

版本：v1.0  
日期：2026-06-26  
适用范围：Ami_Core 管理端智能营销、营销 H5、Ami Glow 小程序、Ami Aura Lite 终端、Ami_Agent 营销增长 Agent、后端营销与客户数据能力  
文档定位：产品需求文档，用于定义“插秧营销”的用户故事、业务流程、现有数字化承接方案与 AI 应用点。

---

## 1. 一句话结论

美业插秧营销不是一次性促销，也不是简单群发优惠券，而是把门店客户按生命周期分到不同“田块”，在合适时间把内容、权益、服务提醒、复购理由和转介绍激励持续种下去，再通过员工跟进、到店服务、数据复盘完成“生根、养护、收割、再播种”的经营闭环。

产品上应把它设计为：

```text
客户分田 -> 营销种子 -> 自动插秧 -> 员工养护 -> 到店转化 -> 效果归因 -> 下一轮补苗
```

其中：

- 客户分田：按新客、体验客、疗程中、待复购、高价值、沉睡、转介绍潜力等分群。
- 营销种子：权益、内容、服务理由、案例、预约入口、顾问任务、转介绍激励。
- 自动插秧：系统按客户状态、时间节点和触达频控生成自动触达或跟进任务。
- 员工养护：美容师、顾问、前台按任务跟进，不依赖店长手工盯人。
- 到店转化：承接预约、核销、办卡、升单、复购、转介绍。
- 效果归因：用真实订单、预约、核销、页面行为和员工任务结果复盘。

## 2. 背景与问题

### 2.1 门店经营现状

美业门店获客越来越难，单次团购、达人种草、节日活动或朋友圈刷屏往往只能带来短期曝光。真正影响利润的是客户能否持续进店、疗程能否消耗、复购是否及时发生、老客是否愿意推荐新客。

门店常见问题：

- 店长靠经验筛客户，无法稳定知道谁该唤醒、谁该复购、谁适合转介绍。
- 美容师知道客户情况，但缺少统一任务和话术，跟进靠个人习惯。
- 活动、优惠、营销页、员工任务和订单结果割裂，复盘只能看报名数或销售额。
- 老客、新客、沉睡客的触达节奏相同，容易骚扰客户或错过最佳时机。
- 朋友圈、微信、电话、到店沟通、终端推荐、小程序入口各自为战。
- 店长想做常态化营销，但缺少“每天该插哪一批客户、谁负责、说什么、怎么验收”的系统。

### 2.2 插秧营销的产品机会

“插秧”的核心不是一次把所有客户轰炸一遍，而是有节奏、有格子的精细化种植：

| 插秧类比 | 美业经营含义 | 系统能力 |
| --- | --- | --- |
| 分田块 | 客户分层和生命周期分群 | 客户标签、预测分、消费/预约/卡项状态 |
| 选秧苗 | 选择适合客户的内容、权益和项目 | 营销资产、优惠权益、项目/卡项推荐 |
| 插秧 | 在合适时间触达或派发跟进任务 | 自动营销、员工任务、H5/小程序触点 |
| 浇水施肥 | 持续回访、护理建议、服务提醒 | 终端任务、顾问话术、预约提醒 |
| 补苗 | 对未响应客户二次跟进 | 失败原因、补触达、换权益/换话术 |
| 收割 | 到店、核销、办卡、复购、转介绍 | 订单、预约、核销、归因 |
| 留种 | 把高响应客户沉淀为转介绍种子 | 推荐链路、裂变奖励、会员权益 |

产品目标是让门店从“临时做活动”升级为“持续经营客户田地”。

## 3. 产品目标

### 3.1 业务目标

| 目标 | 说明 | 核心指标 |
| --- | --- | --- |
| 提升客户复购 | 把护理后、疗程中、到期前、沉睡前的客户及时拉回 | 复购率、复购周期、到店间隔 |
| 提升转介绍 | 将高满意、高 LTV、高响应客户转化为推荐种子 | 推荐人数、推荐成交、裂变层级 |
| 降低人工盯人成本 | 店长不再手工拉表分配跟进 | 自动生成任务率、任务完成率 |
| 提高活动 ROI | 活动从曝光复盘转为收入和利润复盘 | ROI、归因收入、核销率、毛利 |
| 提升员工执行力 | 员工知道每天跟谁、说什么、用什么权益 | 任务完成率、员工转化排行 |
| 打通 AI 增长闭环 | AI 不只写文案，还要发现机会、生成策略、复盘优化 | AI 建议采纳率、AI 触发转化 |

### 3.2 产品目标

- 建立插秧营销工作台，统一展示客户田块、今日插秧任务、待养护客户、转化结果和补苗建议。
- 让店长能从一个营销目标出发，快速生成客户分群、权益、内容、触达节奏和员工任务。
- 让系统自动把营销动作分配到 H5、小程序、终端、员工跟进和自动营销规则。
- 让 AI 能解释“为什么选这批客户、为什么用这个权益、下一步怎么做”。
- 让复盘能追踪从触达、打开、预约、到店、核销、订单到员工跟进的完整链路。

### 3.3 非目标

| 非目标 | 说明 |
| --- | --- |
| 不做单纯群发工具 | 群发短信/微信不是核心，核心是分群、节奏、承接和归因 |
| 不替代营销活动模块 | 插秧营销应复用活动、页面、优惠、自动营销和效果分析能力 |
| 不绕过人工确认 | 批量触达、发券、公开发布页面、影响价格和权益的动作必须确认 |
| 不承诺全自动成交 | AI 和自动化负责建议与执行辅助，员工服务和到店体验仍是关键 |
| 不先做复杂外部广告投放 | MVP 先覆盖私域、到店、H5、小程序、终端和员工跟进 |

## 4. 用户角色

| 角色 | 核心诉求 | 插秧营销价值 |
| --- | --- | --- |
| 老板 / 多店管理者 | 看增长结果、ROI、门店对比 | 知道哪家店会种客户、哪类客群最赚钱 |
| 店长 | 每天安排营销和员工跟进 | 从“手工分配”变成“系统推荐 + 一键派发” |
| 营销运营 | 做活动、内容、权益和复盘 | 从“做素材”升级为“经营客户旅程” |
| 美容师 / 顾问 | 知道该联系谁、怎么说、推什么 | 获得客户理由、话术、权益和跟进节奏 |
| 前台 | 承接预约、到店、核销、办卡 | 到店时看到来源、权益和推荐下一步 |
| 顾客 | 收到有用、不骚扰的服务提醒和权益 | 感知为护理关怀，而不是硬推销 |
| 平台运营 / 总部 | 沉淀标准打法并复制到多店 | 模板化客户田块、活动 SOP 和复盘模型 |

## 6. 业务流程

### 6.1 总流程

```text
1. 设定增长目标
   -> 2. 客户分田
   -> 3. 选择营销种子
   -> 4. 生成插秧节奏
   -> 5. 发布触点和派发任务
   -> 6. 员工养护与客户互动
   -> 7. 到店承接和收银核销
   -> 8. 效果归因和复盘
   -> 9. 补苗和下一轮再播种
```

### 6.2 流程一：设定增长目标

店长或营销运营选择目标：

- 新客拓客：例如 199 体验卡、老带新、新带新。
- 沉睡唤醒：例如 60/90/180 天未到店客户补苗。
- 复购提升：例如项目周期到期、卡项剩余权益未消耗。
- 升单转化：例如体验后转 2991 或疗程卡。
- 节日活动：例如女神节、周年庆、生日月。
- 清库存联动：例如临期商品、组合护理项目。
- 高价值客户维护：例如 VIP 专属护理、私密邀约。

系统需要记录：

| 字段 | 说明 |
| --- | --- |
| 目标类型 | 拉新、复购、唤醒、升单、转介绍、清库存等 |
| 目标客户 | 可由系统推荐或人工选择 |
| 时间周期 | 预热、执行、补苗、复盘周期 |
| 预算限制 | 权益成本、赠品、投放或员工奖励预算 |
| 风险要求 | 触达频控、审批、不可触达客户、敏感客群 |
| 成功指标 | 预约、到店、核销、订单、复购、转介绍、ROI |

### 6.3 流程二：客户分田

系统根据真实业务数据生成客户田块：

| 客户田块 | 筛选逻辑 | 推荐动作 |
| --- | --- | --- |
| 新客待激活田 | 新增客户、已留资未到店、购买体验卡未核销 | 预约提醒、到店激活、前台接待任务 |
| 体验转卡田 | 已体验但未办卡、已做单次项目 | 体验反馈、项目方案、限时转卡权益 |
| 疗程消耗田 | 有次卡/疗程卡且剩余权益 | 消耗提醒、护理计划、预约任务 |
| 复购成熟田 | 到达项目复购周期、最近消费满意 | 复购提醒、同类项目/升级项目推荐 |
| 高价值维护田 | 高 LTV、高频到店、会员等级高 | 专属关怀、生日/节日权益、转介绍邀请 |
| 沉睡补苗田 | 60/90/180 天未到店或预约中断 | 道歉关怀、回店礼、顾问电话任务 |
| 转介绍种子田 | 高满意、高响应、近期到店、社交意愿强 | 推荐链接、裂变奖励、员工跟进 |
| 风险暂停田 | 近期投诉、退费、频繁拒绝、触达过载 | 暂停营销、人工关怀、风险标签 |

### 6.4 流程三：选择营销种子

营销种子由权益、内容、渠道、服务理由和员工动作组成。

| 种子类型 | 示例 | 承接对象 |
| --- | --- | --- |
| 权益种子 | 优惠券、护理体验、赠品、生日礼、推荐红包 | Promotion / 权益资产 |
| 内容种子 | 护理知识、案例、节日文案、客户专属建议 | 营销页面 / 小程序推荐位 / 话术 |
| 服务种子 | 肌肤检测、项目复查、疗程计划、售后回访 | 预约 / 服务任务 / 终端提醒 |
| 转介绍种子 | 老带新链接、推荐奖励、好友体验卡 | 营销 H5 / 裂变关系 |
| 升单种子 | 体验转疗程、项目组合、卡项升级 | 活动 / 收银 / 顾问任务 |
| 补苗种子 | 二次关怀、换权益、换话术、人工电话 | 自动营销 / 员工任务 |

### 6.5 流程四：生成插秧节奏

系统根据目标和客群生成触达节奏：

| 阶段 | 时间 | 动作 | 负责人 |
| --- | --- | --- | --- |
| 预热 | T-3 到 T-1 | 小程序推荐位、朋友圈素材、老客私聊任务 | 营销运营 / 美容师 |
| 插秧 | T 日 | H5 发布、权益发放、目标客户触达、员工任务派发 | 店长 / 系统 |
| 养护 | T+1 到 T+7 | 未打开客户二次提醒、已打开客户预约跟进、已预约客户到店提醒 | 美容师 / 前台 |
| 收割 | 到店日 | 前台识别来源、服务接待、核销、办卡/复购/升单 | 前台 / 顾问 |
| 补苗 | T+7 到 T+14 | 未转化客户换话术或换权益，沉睡客户人工电话 | 店长 / 员工 |
| 复盘 | 周期结束 | 汇总触达、预约、到店、收入、ROI、员工执行 | 店长 / 老板 |

### 6.6 流程五：发布触点和派发任务

触点需要按客户和渠道拆分：

| 触点 | 适合场景 | 数据回流 |
| --- | --- | --- |
| 营销 H5 | 新客拓客、活动落地页、分享链接 | 浏览、点击、留资、预约、线索归因 |
| Ami Glow 小程序 | 老客服务、预约、会员权益、推荐位 | 曝光、点击、预约、行为事件 |
| Ami Aura Lite 终端 | 到店服务、推荐、核销、前台/顾问承接 | 推荐事件、收银、订单归因 |
| 管理端任务 | 店长分配、美容师跟进、顾问电话 | 任务状态、跟进结果、员工转化 |
| 自动营销 | 长期复购、生日、到期、沉睡唤醒 | 执行记录、触达、归因 |
| 线下物料 | 门店海报、二维码、活动展架 | 通过二维码、活动码或手工来源回流 |

### 6.7 流程六：到店承接

客户预约或到店后，前台和顾问需要看到：

- 客户来自哪个插秧策略、活动、页面或员工任务。
- 当前权益、有效期、适用项目和使用限制。
- 客户所属田块，例如沉睡补苗、体验转卡、复购成熟、高价值维护。
- 推荐下一步，例如先做肌肤检测、先核销体验、推荐疗程卡、邀请转介绍。
- 风险提示，例如近期投诉、频繁拒绝、触达过载、权益不可叠加。

### 6.8 流程七：复盘与补苗

复盘不是只看活动收入，需要拆成漏斗：

```text
目标客户 -> 实际触达 -> 打开/点击 -> 领取/留资 -> 预约 -> 到店 -> 核销 -> 订单 -> 复购/转介绍
```

补苗策略：

| 问题 | 判断方式 | 补苗动作 |
| --- | --- | --- |
| 触达低 | 发送失败、客户不可达、渠道不匹配 | 换渠道、改为员工电话、更新联系方式 |
| 打开低 | 标题/内容弱、时机不对 | AI 改文案、换发送时间、换图片 |
| 预约低 | 权益弱、入口复杂、顾客顾虑多 | 增加预约理由、简化表单、员工私聊 |
| 到店低 | 预约提醒不足、时间不合适 | 到店提醒、改约、前台确认 |
| 成交低 | 接待弱、权益不匹配、项目不匹配 | 顾问话术复盘、换方案、服务流程训练 |
| 复购低 | 服务后无跟进、护理周期未维护 | 自动回访、护理计划、下一次预约 |

| 问题 | 产品影响 | 插秧营销需要补齐 |

| 页面 / 模块 | 插秧营销中的职责 |

## 8. AI 应用点

### 8.1 AI 客户分田

目标：把客户从静态标签升级为动态客户田块。

AI 输入：

- 客户基础资料：年龄、性别、来源、标签、会员等级。
- 消费记录：项目、商品、卡项、客单价、最近消费时间。
- 服务记录：预约、到店、核销、疗程消耗。
- 营销记录：触达、打开、点击、领取、转化、拒绝。
- 预测分：流失风险、复购概率、营销响应分、LTV。

AI 输出：

| 输出 | 示例 |
| --- | --- |
| 客户田块 | 沉睡补苗田、复购成熟田、转介绍种子田 |
| 推荐原因 | 90 天未到店，但历史客单高，最近领取过护理券 |
| 优先级 | 高 / 中 / 低 |
| 推荐动作 | 顾问电话、发送回店礼、预约肌肤检测 |
| 风险提示 | 近 30 天已触达 3 次，建议人工关怀而不是群发 |

### 8.2 AI 插秧方案生成

用户输入：

```text
帮我做一套 60 天未到店客户的补苗营销，目标是本周带回 20 个预约。
```

AI 应生成：

- 目标客群和筛选条件。
- 客群规模、预计触达、预计预约和预计成本。
- 推荐权益和替代权益。
- H5 / 小程序内容结构。
- 员工话术。
- 自动触达节奏。
- 需要人工确认的动作。
- 复盘指标。

输出必须是可编辑草稿，而不是直接执行。

### 8.3 AI 内容与话术生成

内容生成不应脱离业务对象，需要绑定客户田块、项目、权益和渠道。

| 场景 | 生成内容 | 约束 |
| --- | --- | --- |
| 新客拓客 | H5 标题、项目介绍、预约理由 | 不夸大疗效，不使用医疗化表述 |
| 沉睡补苗 | 道歉关怀、回店礼说明、电话话术 | 语气真诚，不制造羞辱感 |
| 复购提醒 | 护理周期提醒、项目价值解释 | 基于服务周期和历史项目 |
| 转介绍 | 推荐朋友话术、奖励说明 | 规则清楚，避免虚假承诺 |
| 节日营销 | 朋友圈文案、小程序推荐位文案 | 与门店真实权益一致 |
| 员工跟进 | 一对一私聊话术、异议处理 | 按客户历史和关系亲疏调整 |

### 8.4 AI 权益匹配

AI 不是随机推荐优惠，而是根据客户价值、复购概率、库存、项目毛利和活动目标匹配权益。

| 客群 | 权益策略 |
| --- | --- |
| 高价值客户 | 专属服务、赠品、生日礼，不优先大额折扣 |
| 沉睡客户 | 回店礼、肌肤检测、限时护理体验 |
| 新客 | 低门槛体验、预约礼、推荐好友奖励 |
| 复购成熟客户 | 项目组合、疗程包、护理计划 |
| 价格敏感客户 | 满减、限时券、组合权益 |
| 高转介绍潜力客户 | 推荐奖励、好友体验权益、裂变红包 |

### 8.5 AI 员工任务编排

AI 应把策略拆成员工可执行的任务：

| 任务类型 | 任务内容 | 验收结果 |
| --- | --- | --- |
| 电话跟进 | 联系沉睡客户，确认回店意愿 | 已预约 / 拒绝 / 未接 / 待跟进 |
| 微信私聊 | 发送专属护理建议和权益 | 已回复 / 已领取 / 已预约 |
| 到店承接 | 客户到店时完成检测、核销和方案推荐 | 已核销 / 已成交 / 未成交原因 |
| 服务后转介绍 | 服务满意后邀请推荐朋友 | 已分享 / 已推荐 / 推荐成交 |
| 补苗任务 | 未响应客户换话术或换权益 | 二次触达结果 |

AI 需要解释为什么任务分给某个员工，例如该员工是客户历史服务人、转化率高、当前任务负荷低。

### 8.6 AI 漏斗诊断与补苗

AI 对活动漏斗进行诊断：

| 漏斗异常 | AI 判断 | 推荐动作 |
| --- | --- | --- |
| 触达率低 | 联系方式无效或渠道失败 | 更新联系方式、改员工电话 |
| 打开率低 | 标题弱或发送时间不佳 | 改标题、换发送时段 |
| 点击高但预约低 | 页面权益不清或预约入口弱 | 优化 H5 首屏和预约按钮 |
| 预约高但到店低 | 缺少到店提醒或时间冲突 | 增加前台确认和改约入口 |
| 到店高但成交低 | 接待或权益匹配不足 | 复盘顾问话术、调整项目方案 |
| 成交后复购低 | 服务后没有养护 | 自动生成护理回访和复购提醒 |

### 8.7 AI 复盘报告

周期结束后，AI 自动生成复盘：

- 本轮目标和实际达成。
- 触达、打开、预约、到店、核销、成交、复购、转介绍漏斗。
- 不同客户田块的转化表现。
- 不同权益和内容表现。
- 员工执行和转化排行。
- ROI 和成本说明。
- 下轮补苗建议。
- 可沉淀为模板的打法。

### 8.8 AI 风险控制

AI 应内置营销合规与经营风险提示：

| 风险 | 控制要求 |
| --- | --- |
| 过度触达 | 客户级频控、渠道频控、拒绝后冷却期 |
| 虚假承诺 | 文案审核，避免医疗疗效、绝对化用语 |
| 权益亏损 | 权益成本、毛利、叠加规则校验 |
| 越权执行 | 批量触达、发券、发布页面必须确认或审批 |
| 敏感客户 | 投诉、退款、黑名单、特殊备注客户不自动营销 |
| 数据误判 | AI 建议必须展示数据依据和不确定性 |

### 9.4 营销种子库

营销种子库不等同于素材库，应包含“可被策略执行的资产”：

### 9.5 任务与执行

| 字段 | 说明 |

MVP 可以先用现有 `MarketingActivity`、`MarketingAutomationStrategy`、`Promotion`、`MarketingPage`、任务模型承接，新增字段保存插秧策略 JSON。稳定后再拆独立表。

## 13. 验收标准

### 13.1 产品验收

- 店长可以从营销工作台看到客户田块和今日插秧任务。
- 营销运营可以基于一个目标生成可编辑插秧策略草稿。
- 策略可以绑定客户、权益、H5、小程序/终端触点和员工任务。
- 员工可以看到客户跟进理由、话术和可用权益。
- 客户预约或到店时，前台能看到营销来源和权益。
- 策略结束后能看到漏斗复盘、员工执行和补苗建议。

### 13.2 数据验收

- 客户名单来自真实客户、预约、订单、卡项、营销行为和预测数据。
- H5 行为、线索、预约、订单可以归因到策略或页面。
- 自动营销触达和订单可形成归因。
- 员工任务结果能进入效果复盘。
- AI 建议必须展示数据依据，不能只输出结论。

### 13.3 风险验收

- 批量触达、批量发券、发布页面需要人工确认。
- 已投诉、黑名单、近期拒绝、触达过载客户不进入自动触达。
- 权益叠加、有效期、适用项目和成本需要校验。
- 文案不得包含医疗化、绝对化或虚假承诺。

## 14. 落地建议

第一阶段不要把插秧营销做成新的孤立大模块，而应优先用现有智能营销能力拼成闭环：

1. 在营销工作台增加“客户田块 + 今日插秧 + 补苗建议”。
2. 在智能推荐中补客户田块和推荐原因。
3. 在活动/自动营销草稿中增加插秧策略结构。
4. 在推广资产中强化“权益 + 页面 + 话术 + 任务模板”的组合资产。
5. 在效果分析中补插秧漏斗和员工任务结果。
6. 在 Ami_Agent 中新增“营销增长 Agent”提示词和工具编排，让用户可以用自然语言生成策略草稿。

这样可以在不推翻现有数字化底座的前提下，把“营销活动、自动触达、推广资产、员工任务、终端承接、数据复盘”串成可交付的插秧营销闭环。

## 15. 后续开发拆分建议

| 阶段 | 任务 | 交付物 |
| --- | --- | --- |
| 阶段 1 | 客户田块和工作台 | 田块算法、今日插秧列表、客户名单预览 |
| 阶段 2 | 插秧策略草稿 | 策略表单、AI 生成方案、权益/页面/话术绑定 |
| 阶段 3 | 员工任务闭环 | 任务派发、跟进结果、员工转化排行 |

| 阶段 6 | 模板与多店复制 | 标准打法模板、总部下发、多店对比 |

优先级建议：先做“沉睡补苗”和“复购成熟”两个田块，因为这两类不依赖外部渠道，能最快验证客户召回、员工执行和订单归因闭环。
