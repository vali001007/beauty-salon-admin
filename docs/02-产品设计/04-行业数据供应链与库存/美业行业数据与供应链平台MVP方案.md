说明：本文仍作为“两平台协同 MVP”的总览方案。若产品架构拆分为行业数据平台和供应链平台，具体边界以《行业数据平台与供应链平台拆分对比方案》为准。

最新决策：当前优先建设行业数据平台，供应链平台不进入首期 MVP，只在行业数据平台中预留未来供应链平台对接字段和接口。最新执行口径以《美业行业数据平台MVP需求文档》为准。

## 1. MVP 结论

美业行业数据与供应链平台的 MVP 不应只做“行业模板库”和“商品报价库”，必须把“项目 BOM”纳入首版核心能力。

原因是项目 BOM 是美业经营闭环的连接点：

1. 连接项目模板和耗品模板：知道一个服务项目标准消耗哪些商品/耗品。
2. 连接供应链和成本：知道每次服务的标准耗材成本和可采购 SKU。
3. 连接 Ami_Core 库存：服务完成后可按 BOM 扣减库存或生成实际消耗记录。
4. 连接经营利润：项目收入减去 BOM/实耗成本和提成，才能看真实毛利。
5. 连接终端服务：Ami Aura Lite 可提示美容师准备耗品、用量和注意事项。
6. 连接智能补货：按项目预约、服务频次、BOM 用量预测耗品需求。

因此，最新 MVP 定义为：

> 建设一个独立的“美业行业配置库 + 项目 BOM 库 + 供应链 SKU/报价库”，并通过 API 供 Ami_Core 调用，让门店能从行业模板一键生成项目、商品、BOM、薪酬参考和服务知识，同时支持基础采购需求。

## 2. MVP 范围

### 2.1 首版必须包含

| 模块 | MVP 内容 | 目标 |
| --- | --- | --- |
| 行业服务项目模板 | 录入 50 个常见服务项目模板 | 让门店可一键创建项目 |
| 项目 BOM 模板 | 为 50 个服务项目配置标准耗品 BOM | 让项目具备成本、扣耗和采购基础 |
| 商品/耗品模板 | 录入 100 个常见耗品/商品模板 | 支撑 BOM 和供应链 SKU 映射 |
| 供应链 SKU/报价 | 接入或录入首批供应商 SKU 和报价 | 支撑采购参考价和供应商选择 |
| 岗位薪酬模板 | 录入 10 个岗位薪酬模板 | 支撑员工岗位和绩效配置参考 |
| 服务 SOP 和卫生安全知识库 | 建立项目 SOP、禁忌、卫生要求 | 支撑 AI、终端和培训 |
| Ami_Core 调用 | 项目、商品、员工配置页显示行业参考 | 让行业平台真正进入业务流程 |
| 基础采购需求 | Ami_Core 可基于商品/耗品发起采购需求 | 打通供应链入口 |

### 2.2 首版暂不包含

| 暂不做 | 原因 |
| --- | --- |
| 自动爬取全网价格 | 授权和稳定性不可控，MVP 先用人工模板、供应商报价、门店采购数据 |
| 在线支付和自动结算 | 会显著扩大财务和合规复杂度 |
| 复杂对账 | MVP 先记录采购单、收货、发货状态 |
| 自动跨供应商拆单 | 依赖供应商覆盖和价格策略成熟度 |
| 供应链金融 | 与当前 MVP 目标无关 |
| 医疗美容诊疗知识 | 合规风险高，首版只做生活美容服务知识 |
| 面向第三方售卖行业大数据 | 首版先服务 Ami_Core，不做外部商业化 |

## 3. 项目 BOM 在 MVP 中的定义

项目 BOM 是某个服务项目在标准服务流程下预计消耗的商品、耗品、一次性用品、仪器耗材和辅助物料清单。

### 3.1 BOM 示例

项目：深层补水护理

| BOM 项 | 类型 | 标准用量 | 单位 | 是否必需 | 成本用途 | 服务用途 |
| --- | --- | --- | --- | --- | --- | --- |
| 洁面乳 | 院装耗品 | 5 | ml | 是 | 计入耗材成本 | 清洁 |
| 软膜粉 | 院装耗品 | 25 | g | 是 | 计入耗材成本 | 补水敷膜 |
| 精华液 | 院装耗品 | 3 | ml | 是 | 计入耗材成本 | 导入 |
| 一次性面巾 | 一次性耗材 | 1 | 片 | 是 | 计入耗材成本 | 卫生 |
| 棉片 | 一次性耗材 | 4 | 片 | 是 | 计入耗材成本 | 擦拭 |
| 导入仪探头套 | 仪器耗材 | 1 | 个 | 否 | 可选计入 | 仪器护理 |

### 3.2 BOM 字段建议

```text
IndustryProjectBomTemplate
- id
- serviceTemplateId
- bomVersion
- status: draft | pending_review | published | offline
- totalStandardCost
- costCurrency
- sourceId
- reviewerId
- publishedAt

IndustryProjectBomItemTemplate
- id
- bomTemplateId
- productTemplateId
- supplySkuId
- itemRole: main_material | auxiliary_material | disposable | instrument_consumable | disinfectant
- standardQty
- unit
- lossRate
- required: true | false
- allowSubstitute: true | false
- substituteGroupCode
- costIncluded: true | false
- serviceStep
- note
```

### 3.3 BOM 口径

1. 标准用量用于成本估算、服务准备和补货预测。
2. 实际用量由 Ami_Core 的服务消耗记录或库存流水沉淀。
3. 项目毛利优先使用实际耗材成本；没有实际扣减时，用 BOM 标准成本估算。
4. BOM 需要版本管理。项目采用模板后，Ami_Core 应保存一份门店项目 BOM 快照，避免行业模板更新后影响历史利润。
5. BOM 中的商品模板可以映射多个供应链 SKU，用于比价和替代采购。

## 4. MVP 产品模块

### 4.1 独立平台模块

#### 4.1.1 行业服务项目模板

能力：

1. 新建、编辑、审核、发布服务项目模板。
2. 支持项目分类、服务时长、建议价格、适用人群、禁忌、标准流程。
3. 支持关联项目 BOM 模板。
4. 支持关联 SOP 和卫生安全知识条目。
5. 支持发布给 Ami_Core 调用。

MVP 数量：

- 50 个服务项目模板。
- 每个模板至少包含：分类、名称、建议时长、建议价区间、适用人群、禁忌、SOP 摘要。

#### 4.1.2 项目 BOM 模板

能力：

1. 为服务项目维护标准耗品清单。
2. 支持从商品/耗品模板中选择 BOM 项。
3. 支持标准用量、单位、损耗率、是否必需、是否计入成本。
4. 支持显示标准成本合计。
5. 支持将 BOM 项映射到供应链 SKU 和供应商报价。
6. 支持发布版本，并供 Ami_Core 采用。

MVP 数量：

- 50 个服务项目均需配置 BOM。
- 每个项目建议配置 3-8 个 BOM 项。
- 至少 80% 的 BOM 项需映射到商品/耗品模板。
- 至少 50% 的 BOM 项需映射到可采购 SKU 或报价。

#### 4.1.3 商品/耗品模板库

能力：

1. 维护商品/耗品标准名称、分类、品牌、规格、单位、条码、参考成本。
2. 支持区分零售商品、院装耗品、一次性耗材、仪器耗材、消毒用品。
3. 支持单位换算，例如 ml、g、片、盒、支、套。
4. 支持被项目 BOM 引用。
5. 支持映射供应链 SKU。

MVP 数量：

- 100 个常见商品/耗品模板。
- 至少覆盖面部护理、身体护理、美睫、美甲、头皮护理、消毒和一次性用品。

#### 4.1.4 供应链 SKU 和报价

能力：

1. 维护供应商。
2. 维护供应商 SKU。
3. 维护报价、起订量、有效期、服务区域。
4. 支持 SKU 映射到商品/耗品模板。
5. 支持 Ami_Core 查询可采购 SKU 和报价。

MVP 数量：

- 至少 3 个供应商样本。
- 至少 50 个供应链 SKU。
- 至少 50 条有效报价。

#### 4.1.5 岗位薪酬模板

能力：

1. 维护岗位、等级、底薪区间、提成区间、手工费区间。
2. 支持按城市/门店定位配置薪酬参考。
3. 支持 Ami_Core 员工配置页调用。

MVP 数量：

- 10 个岗位薪酬模板。
- 建议包含美容师、顾问、前台、店长、店助、美发师、美甲师、美睫师、培训师、运营督导。

#### 4.1.6 知识库

能力：

1. 维护服务 SOP、禁忌提醒、卫生安全、销售话术、培训知识。
2. 支持关联服务项目和岗位。
3. 支持审核发布。
4. 支持 AI Gateway 和 Ami Aura Lite 调用已发布知识。

MVP 数量：

- 20 条服务 SOP。
- 10 条卫生安全知识。
- 20 条销售/顾问话术。

### 4.2 Ami_Core 侧模块

#### 4.2.1 项目配置调用行业模板

能力：

1. 项目管理增加“从行业模板创建”入口。
2. 选择项目模板后展示建议价、建议时长、SOP、BOM 标准成本。
3. 可一键创建 Ami_Core 项目。
4. 创建时同步生成门店项目 BOM。
5. 支持店长调整 BOM 用量、替换耗品。

落地对象：

- `Project`
- `ProjectBomItem`
- `Product`

#### 4.2.2 商品/耗品配置调用供应链

能力：

1. 商品管理增加“从行业商品/供应链 SKU 创建”入口。
2. 商品详情显示参考采购价、供应商报价和可替代 SKU。
3. 商品被 BOM 引用时，展示引用项目数量。
4. 库存低于阈值时，可发起采购需求。

落地对象：

- `Product`
- `StockBatch`
- `StockMovement`

#### 4.2.3 服务消耗与库存扣减

能力：

1. 服务完成时，可按项目 BOM 生成默认消耗清单。
2. 美容师或店长可确认实际用量。
3. 确认后写入库存扣减流水。
4. 若未确认实际用量，经营利润可按 BOM 标准成本估算，并标记为估算。

#### 4.2.4 经营利润

能力：

1. 项目毛利计算使用实际耗材优先。
2. 无实际耗材时使用 BOM 标准成本。
3. 缺 BOM 时明确提示 `missing_bom`。
4. 项目编辑页展示 BOM 标准成本、建议售价和毛利率预估。

#### 4.2.5 基础采购需求

能力：

1. 从商品、库存、BOM 缺口发起采购需求。
2. 查询供应链平台的可采购 SKU 和报价。
3. 创建采购需求并记录状态。
4. 收货后由 Ami_Core 执行本地入库。

## 5. 关键流程

### 5.1 行业项目模板和 BOM 发布

```mermaid
flowchart LR
  A["行业数据运营创建项目模板"] --> B["维护标准 BOM"]
  B --> C["BOM 项映射商品/耗品模板"]
  C --> D["商品/耗品映射供应链 SKU"]
  D --> E["审核项目模板和 BOM"]
  E --> F["发布版本"]
  F --> G["Ami_Core 查询并采用"]
  G --> H["生成门店 Project + ProjectBomItem"]
```

### 5.2 门店创建项目

```mermaid
flowchart LR
  A["店长进入 Ami_Core 项目管理"] --> B["选择从行业模板创建"]
  B --> C["查看建议价/时长/BOM 成本/SOP"]
  C --> D["调整价格和 BOM 用量"]
  D --> E["保存项目"]
  E --> F["生成门店项目和项目 BOM"]
  F --> G["后续用于服务扣耗和利润核算"]
```

### 5.3 服务完成扣减耗材

```mermaid
flowchart LR
  A["终端完成服务"] --> B["读取项目 BOM"]
  B --> C["生成默认耗材消耗清单"]
  C --> D["美容师确认实际用量"]
  D --> E["写入 StockMovement"]
  E --> F["更新库存和项目毛利"]
```

### 5.4 BOM 驱动采购

```mermaid
flowchart LR
  A["系统发现库存不足"] --> B["查看被哪些项目 BOM 引用"]
  B --> C["计算未来预约/服务需求"]
  C --> D["查询供应链 SKU 和报价"]
  D --> E["生成采购建议"]
  E --> F["店长确认采购需求"]
```

## 9. 验收标准

### 9.1 行业模板和 BOM

1. 平台可创建 50 个服务项目模板。
2. 每个服务项目模板可维护并发布项目 BOM。
3. BOM 可关联商品/耗品模板。
4. BOM 可关联供应链 SKU 或报价。
5. Ami_Core 可查询服务项目模板详情和 BOM。
6. Ami_Core 可一键采用模板，生成本地项目和 `ProjectBomItem`。
7. 店长可在 Ami_Core 调整采用后的 BOM，不影响行业平台原模板。

### 9.2 成本和库存

1. 项目编辑页能展示 BOM 标准成本。
2. 服务完成时可按 BOM 生成默认消耗清单。
3. 确认实际消耗后可写入库存扣减流水。
4. 项目利润在有实际耗材时使用实际成本。
5. 项目利润在无实际耗材时使用 BOM 标准成本，并标记为估算。
6. 项目缺 BOM 时明确提示缺口，不能静默当作 0 成本。

### 9.3 供应链

1. 平台可维护供应商、SKU 和报价。
2. 商品/耗品模板可映射供应链 SKU。
3. Ami_Core 可按 BOM 中的耗品查询可采购 SKU。
4. Ami_Core 可创建采购需求。
5. 采购需求状态可同步回 Ami_Core。

### 9.4 AI 和终端

1. Ami Aura Lite 可读取项目 BOM，展示服务前耗品准备清单。
2. AI Gateway 只能调用已发布 SOP、禁忌和卫生安全知识。
3. AI 生成建议时能引用项目 BOM 成本或耗品信息，但不能编造供应商报价。

## 10. 里程碑

### M1：行业模板和 BOM 底座

周期：2 周

交付：

- 服务项目模板。
- 商品/耗品模板。
- 项目 BOM 模板。
- BOM 成本预估。
- 模板审核发布。

### M2：Ami_Core 采用模板

周期：2 周

交付：

- Ami_Core 查询服务模板和 BOM。
- 从模板创建项目。
- 同步生成门店项目 BOM。
- 项目编辑页显示行业 BOM 参考和标准成本。

### M3：供应链 SKU 和采购入口

周期：3 周

交付：

- 供应商、SKU、报价。
- BOM 项映射可采购 SKU。
- Ami_Core 查询报价。
- Ami_Core 发起采购需求。

### M4：服务扣耗和经营利润联动

周期：3 周

交付：

- 服务完成生成默认耗材清单。
- 实际耗材确认和库存扣减。
- 项目毛利按实耗/BOM 计算。
- 缺 BOM、缺实耗等数据质量提示。

## 11. 产品优先级建议

最高优先级不是先做“很多数据”，而是先把一个完整闭环跑通：

1. 一个行业项目模板能带出标准 BOM。
2. BOM 能带出标准耗品。
3. 耗品能映射供应链 SKU 和报价。
4. Ami_Core 能采用项目、商品和 BOM。
5. 服务完成能按 BOM 扣耗或生成实际消耗。
6. 经营利润能看到项目真实成本或估算成本。
7. 库存不足能从 BOM 反推采购需求。

只要这个闭环跑通，后续扩充 50 个项目、100 个耗品、更多供应商就是数据运营和规模化问题；如果没有 BOM 闭环，行业数据和供应链都会停留在“参考资料”，无法真正进入 Ami_Core 的经营决策。
