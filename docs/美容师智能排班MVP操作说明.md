# 美容师智能排班 MVP 操作说明

日期：2026-06-08

## 适用对象

门店店长、运营管理员、实施顾问。

## 使用入口

进入管理端：

```text
门店管理 -> 排班管理
```

页面顶部提供四类操作：

- 智能排班：配置并生成建议班表。
- 检查冲突：对当前页面班表重新评分并检查硬冲突。
- 需求热力图：查看本周预约需求、建议人手和实际在岗差距。
- 发布智能排班：将预览班表写入正式排班。

## 推荐操作流程

### 1. 选择排班周期

在排班页面顶部先确认当前查看周，也可以在“智能排班”配置里选择：

- 当前查看周
- 本周
- 下周

推荐日常使用“下周”，避免误改本周已执行中的班表。

### 2. 生成智能排班预览

点击“智能排班”，确认配置：

- 生成方式：默认“复制上周并优化”。
- 优先目标：默认“覆盖预约”。
- 高峰最低在岗：默认 3 人，作用于周六、周日下午 14:00-17:00。
- 保留已确认预约美容师：默认开启。
- 允许覆盖忙碌时段：默认关闭。
- 允许覆盖请假时段：默认关闭。

点击“生成预览”后，系统只生成预览，不会直接写入正式排班。

### 3. 查看评分和风险

预览面板会展示：

- 总评分
- 预约覆盖率
- 高峰覆盖率
- 硬冲突数量
- 提醒数量
- 业务解释
- 冲突列表

状态为“预览未发布”时，代表当前班表还没有写入数据库正式排班。

### 4. 人工调整

店长可以继续在班表格子中手动调整：

- 正常
- 忙碌
- 请假

调整后点击“检查冲突”，系统会基于当前页面班表重新计算评分和冲突。

### 5. 发布排班

当硬冲突数量为 0 时，可以点击“发布智能排班”。

发布前系统会弹出确认提示，并在后端再次校验：

- 是否存在请假冲突。
- 是否存在忙碌和预约冲突。
- 已确认预约对应时段是否仍可服务。
- 是否有重复排班。

校验通过后，系统写入正式排班，并记录智能排班运行记录。

## 不可发布的典型情况

- 美容师请假时段被安排为正常在岗。
- 已确认预约对应时段被设置为请假或忙碌。
- 同一美容师同一时间段存在重复或重叠排班。
- 已确认预约没有被班表覆盖。

## 数据留痕

每次生成预览会记录一次智能排班运行记录。

发布成功后，运行记录会更新为 published，并保存：

- 输入参数
- 生成班表
- 评分
- 冲突和风险摘要
- 发布时间

发布成功还会记录一次 Ami 智能贡献，用于后续统计智能排班为门店节省的人工时间。

## 当前 MVP 边界

本版本不做：

- 大模型直接生成最终班表。
- 自动修改客户已确认预约。
- 跨门店调配美容师。
- 员工端换班审批。
- 复杂劳动合同和薪资工时规则。

## 验收检查清单

- 能对下周生成智能排班预览。
- 预览不会直接写入正式排班。
- 能看到评分、预约覆盖率、高峰覆盖率和冲突列表。
- 有硬冲突时不能发布。
- 无硬冲突时可以发布并刷新正式排班。
- 已确认预约时段不会被自动清空。
- 请假时段不会被自动安排为正常在岗。
- 能查看需求热力图，并识别人手不足时段。
