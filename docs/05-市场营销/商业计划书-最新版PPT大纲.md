# Ami 智慧美业商业计划书 - 最新版 PPT 大纲

更新时间：2026-06-22
用途：用于生成投资人/合作伙伴路演 PPT。
版本定位：在旧版“美业管理平台商业计划书”基础上，吸收最近的 Ami_Aura 数字员工、API + Agent、经营利润、行业数据平台、供应链平台方案，形成新版叙事。

## 一句话版本

Ami 要做的不是一套普通美业管理软件，而是面向中小美容院和区域连锁的“经营智能体系统”：用 Ami_Core 管经营数据，用 Ami_Aura 做门店数字员工，用行业数据平台沉淀标准项目/BOM/知识库，用供应链平台连接耗品采购和履约，最终帮助门店从“靠经验经营”升级为“用数据和智能体经营”。

## 第 2 页｜为什么是现在

### 核心结论

美业门店正在从“手工管理 + 老板经验”进入“数据经营 + AI 助理 + 供应链效率”的拐点。

### 内容要点

- 门店经营压力上升：获客更贵、员工流动高、复购不稳定、利润口径不清。
- 传统系统只管收银和会员，难以回答“哪项业务赚钱、哪类客户该跟进、哪些耗材要补”。
- AI 已经可以进入门店一线任务，但必须绑定真实业务数据、权限、审计和执行闭环。
- 中小门店不需要重型 ERP，更需要能快速落地的经营智能体。

### 建议画面

行业变化四象限：人工经验、传统管理软件、重 ERP、经营智能体。

## 第 3 页｜核心痛点

### 核心结论

美容院不是没有系统，而是缺少一套能把客户、服务、利润、员工、库存和营销串起来的经营闭环。

### 内容要点

- 看不清真实利润：办卡、充值带来现金流，但不等于利润。
- 服务过程不透明：项目耗材、工时、提成、履约成本没有被准确记录。
- 客户运营断层：客户画像、营销推荐、活动转化和复购跟进没有闭环。
- 一线执行不稳定：前台、顾问、美容师高度依赖个人经验。
- 库存与供应链割裂：低库存、临期、异常消耗和补货采购没有系统联动。

### 建议画面

门店经营漏斗或问题链路图：客户 -> 服务 -> 收银 -> 库存 -> 利润 -> 复购。

## 第 4 页｜我们的答案

### 核心结论

Ami 用“经营中枢 + 数字员工 + 行业数据 + 供应链协同”构建美业门店的智能经营基础设施。

### 内容要点

- Ami_Core：门店经营中枢，承载客户、订单、项目、库存、员工、权限、营销和利润。
- Ami_Aura：店内数字员工工作台，辅助前台、顾问、美容师、店长完成业务任务。
- Ami_Glow：客户侧小程序，承接预约、会员、消费记录、营销触达和服务体验。
- 行业数据平台：沉淀标准项目、标准耗品、项目 BOM、岗位薪酬和知识库。
- 供应链平台：基于标准品、库存、BOM 和补货建议连接采购履约。

### 建议画面

五层产品架构图：客户侧、门店现场、管理后台、行业数据、供应链生态。

## 第 6 页｜核心闭环 1：经营利润

### 核心结论

用经营利润看板把“门店收了多少钱”升级为“门店到底赚不赚钱”。

### 内容要点

- 收入口径：服务收入、商品收入、办卡收入、充值收入、消课收入、净收入。
- 成本口径：耗材、员工提成/手工费、固定成本、营销费用、折旧等。
- 核心指标：毛利、净利、项目毛利、商品毛利、员工人效、会员卡履约风险。
- 业务价值：帮助老板发现低毛利项目、异常耗材、营销 ROI 差和预收履约风险。

### 建议画面

利润看板截图位：总览指标、项目毛利排行、异常提醒、门店排行。

## 第 7 页｜核心闭环 2：数字员工

### 核心结论

Ami_Aura 的长期定位不是一台收银屏，而是可被配置、考核和归因的门店数字员工。

### 内容要点

- 当前阶段：先作为一线智能终端，完成服务、收银、核销、办卡、充值、任务提醒。
- 下一阶段：升级为数字员工工作台，按岗位承担前台助理、顾问助理、美容师助理、店长助理、会员运营助理。
- 商业化方式：项目交付费 + 终端/席位服务费 + 值守服务费 + 可归因绩效提成。
- 关键前提：必须先补齐推荐采纳、订单归因、任务记录、绩效看板和结算审计。

### 建议画面

数字员工岗位卡片：前台助理、顾问助理、美容师助理、店长助理、会员运营助理。

## 第 8 页｜核心闭环 3：客户运营与智能营销

### 核心结论

客户运营从“群发活动”升级为“画像识别、推荐策略、活动/规则承接、效果复盘”的闭环。

### 内容要点

- 客户画像：消费频次、项目偏好、会员权益、沉睡风险、复购潜力。
- 智能推荐：推荐项目、推荐商品、优惠方式、触达渠道、执行方式。
- 两种执行方式：单次营销活动、自动营销规则。
- 效果复盘：浏览、报名、预约、核销、复购、转化金额。

### 建议画面

客户画像 -> 推荐卡 -> 活动/自动规则 -> 效果分析的流程图。

## 第 9 页｜核心闭环 4：行业数据平台

### 核心结论

行业数据平台是 Ami 做深美业的关键壁垒：标准项目、标准耗品和项目 BOM 是利润、库存、AI 推荐和供应链的共同底座。

### 内容要点

- 首期目标：优先建设可被 Ami_Core 采用的成熟行业配置资产。
- 核心资产：服务项目模板、项目 BOM 模板、标准商品/耗品模板、岗位薪酬模板、知识库。
- 业务连接：门店可一键采用行业模板，快速生成本店项目、商品、BOM 和经营口径。
- 长期价值：沉淀美业数据资产，为 AI 推荐、利润估算、补货建议和供应链匹配提供依据。

### 建议画面

行业数据资产库示意：模板发布 -> 门店采用 -> 经营数据回流 -> 模板优化。

## 第 10 页｜核心闭环 5：供应链平台

### 核心结论

供应链不是单独做采购商城，而是从门店库存、BOM 消耗和补货建议自然延伸出来的交易场景。

### 内容要点

- 入口：项目 BOM、库存扣耗、安全库存、临期/缺货预警、补货建议。
- 能力：供应商目录、标准品映射、采购订单、入库、对账、供应商服务。
- 收入：采购 GMV 分成、供应商服务费、优选目录推广、供应链金融预留。
- 节奏：先用行业数据平台做好标准品和 BOM 映射，再接入真实供应商 SKU 和履约。

### 建议画面

BOM 消耗 -> 库存预警 -> 补货建议 -> 采购订单 -> 入库 -> 供应商结算。

## 第 11 页｜技术与数据架构

### 核心结论

Ami 的技术路线是“业务 API 底座 + Agent 编排 + 多端协同”，不是让 AI 直接操作页面。

### 内容要点

- server-v2：统一业务 API，承载客户、订单、库存、营销、终端、利润、行业数据、供应链。
- 管理端：配置、审核、看板、权限、经营决策。
- Ami_Aura Lite：一线任务入口和店内 Agent 工作台。
- Ami_Glow：客户侧服务入口。
- Agent 模式：AI 调用受控 API，写操作必须经过权限、预检查、确认、审计和幂等保护。

### 建议画面

技术架构图：用户端 -> Agent/终端 -> API Gateway -> 业务服务 -> 数据库/行业数据/供应链。

## 第 12 页｜商业模式

### 核心结论

收入结构围绕“项目交付 + 终端席位 + AI/Agent 服务 + 绩效分成 + 供应链分成”建立结果导向商业模式。

### 内容要点

| 收入来源 | 说明 | 阶段 |
|---|---|---|
| 项目交付费 | 按门店或连锁客户收取初始化、配置、培训和数据治理费用 | 立即可售 |
| 终端/席位服务费 | Ami_Aura 按设备、席位或服务包收费 | 样板店阶段 |
| AI/Agent 服务包 | 客户画像、推荐、问答、营销文案、经营分析、任务编排 | V1 阶段 |
| 数字员工值守费 | 按岗位包/月，替代模糊的 AI 工时费 | 任务闭环后 |
| 可归因绩效提成 | 对明确归因的增量成交收取分成 | 归因审计成熟后 |
| 供应链分成 | 耗品采购 GMV 分成或供应商服务费 | 供应链接入后 |

### 建议画面

收入阶梯图：项目交付 -> 智能终端/席位 -> AI/Agent 服务 -> 绩效分成 -> 供应链生态。

## 第 13 页｜定价与套餐建议

### 核心结论

早期不要卖复杂概念，先用“门店经营系统交付 + 智能终端 + AI/Agent 服务包”建立可验收价值。

### 内容要点

| 方案 | 目标客户 | 建议包含 |
|---|---|---|
| 单店启动交付包 | 预算敏感单店 | 客户、预约、订单、项目、基础库存、基础报表、数据初始化 |
| 单店增长交付包 | 有增长诉求单店 | 启动包 + Ami_Aura + 客户画像 + 智能营销 + 利润看板 |
| 连锁经营交付包 | 3-30 家区域连锁 | 多门店、权限、总部看板、统一项目/商品/营销/库存、实施培训 |
| 数字员工服务包 | 样板门店/高端门店 | Ami_Aura 岗位包 + 任务看板 + 归因分析 + 绩效试点 |
| 供应链协同包 | 耗材消耗明显门店 | 行业 BOM + 补货建议 + 供应商目录 + 采购协同 |

### 建议画面

交付包矩阵，突出从一次性交付到持续服务、分成和供应链协同的升级路径。

## 第 14 页｜目标客户与切入策略

### 核心结论

首批客户应聚焦“服务流程复杂、会员复购强、耗材消耗可标准化”的门店，而不是泛泛覆盖所有小店。

### 内容要点

- 优先客户：中高端美容院、皮肤管理中心、SPA 会所、3-30 家区域连锁。
- 共同特征：有会员/次卡、顾问销售、项目复购、耗材扣耗、员工提成和经营复盘需求。
- 样板店策略：先打通 3-5 家高配样板店，证明利润、复购、效率和供应链价值。
- 扩张策略：从同城区域连锁和供应商渠道切入，再复制到更多城市。

### 建议画面

客户分层图：样板店 -> 区域连锁 -> 单店规模化 -> 供应链生态。

## 第 15 页｜竞争优势

### 核心结论

Ami 的优势不在“也有会员收银”，而在“美业深度数据 + 店内数字员工 + 利润/库存/营销闭环 + 供应链入口”。

### 内容要点

- 比通用管理软件更懂美业：项目、次卡、核销、耗材、提成、复购、顾问销售。
- 比传统美业软件更智能：AI 推荐、智能问答、终端任务、自动营销、经营分析。
- 比单点 AI 工具更可落地：所有 AI 动作绑定业务 API、权限、审计和数据闭环。
- 比单纯供应链平台更靠近交易入口：从真实库存、BOM 和补货建议触发采购需求。

### 建议画面

二维竞争图：美业深度 × AI/数据闭环能力。

## 第 16 页｜增长路径

### 核心结论

增长不靠一次性卖软件，而靠“样板门店效果证明 + 区域连锁复制 + 供应链伙伴协同”。

### 内容要点

- 阶段 1：完成核心经营闭环，服务 3-5 家样板店。
- 阶段 2：沉淀标准实施包，复制到 50 家种子门店。
- 阶段 3：通过区域代理、培训机构、供应商和行业社群获客。
- 阶段 4：行业数据平台和供应链平台形成生态增长。
- 阶段 5：数字员工按岗位打包，提升 ARPU 和客户粘性。

### 建议画面

增长飞轮：门店数据 -> AI 效果 -> 样板案例 -> 获客 -> 行业模板 -> 供应链交易。

## 第 17 页｜关键指标

### 核心结论

投资人需要看到两类指标：门店经营改善指标和平台商业化指标。

### 内容要点

门店侧指标：

- 老板查看利润看板频次
- 项目毛利改善
- 耗材异常下降
- 客户复购触达转化
- 美容师任务完成率
- 库存缺货/临期下降

平台侧指标：

- 付费门店数
- 单店 ARPU
- Ami_Aura 部署数
- AI/Agent 任务调用量
- 推荐采纳率
- 可归因成交额
- 供应链采购 GMV
- 月留存率和续费率

### 建议画面

双指标仪表盘：经营价值指标 + 商业化指标。

## 第 18 页｜产品路线图

### 核心结论

路线图应先闭合真实业务，再扩大智能体和生态能力。

### 内容要点

| 阶段 | 目标 | 关键交付 |
|---|---|---|
| P0：业务事实闭合 | 让核心门店业务可信 | 客户、订单、项目、库存、核销、收银、服务记录 |
| P1：经营利润闭环 | 看清真实利润 | 利润看板、项目毛利、商品毛利、员工人效、会员履约 |
| P2：行业数据接入 | 建立标准配置资产 | 项目模板、BOM、标准耗品、岗位薪酬、知识库 |
| P3：数字员工任务 | 证明一线执行价值 | 任务中心、推荐采纳、归因事件、绩效看板 |
| P4：供应链协同 | 连接采购交易 | 补货建议、供应商目录、采购订单、入库对账 |
| P5：Agent 生态 | 扩展多角色智能体 | 多 Agent 协作、客户侧入口、外部系统连接 |

### 建议画面

路线图时间轴，按“经营闭环 -> 数据平台 -> 数字员工 -> 供应链生态”展开。

## 第 19 页｜融资用途

### 核心结论

融资资金应优先用于验证商业闭环，而不是过早铺大团队。

### 内容要点

- 产品研发：完善 Ami_Core、Ami_Aura、经营利润、行业数据、供应链基础能力。
- AI/Agent 能力：任务编排、推荐引擎、权限审计、归因分析、语义查询。
- 样板客户：打磨 3-5 家样板门店的真实经营案例。
- 供应链合作：接入核心耗品供应商和标准品目录。
- 销售与交付：建立可复制实施包、培训体系和渠道合作。
- 数据安全与合规：权限、审计、门店隔离、客户隐私保护。

### 建议画面

资金用途饼图或分阶段资金表。

## 第 20 页｜风险与应对

### 核心结论

最大风险不是技术不可行，而是价值证明、归因结算和门店执行习惯。

### 内容要点

| 风险 | 影响 | 应对 |
|---|---|---|
| 门店数据不完整 | AI 和利润分析不可信 | 先做业务事实闭合和基础数据治理 |
| 数字员工提成争议 | 门店接受度下降 | 先用值守费和服务包，绩效分成只做可归因场景 |
| 供应链履约复杂 | 平台早期运营压力大 | 先做标准品/BOM/补货建议，再分阶段接入供应商 |
| 客户教育成本高 | 销售周期拉长 | 样板店案例、经营改善数据、渠道合作 |
| 同质化竞争 | 价格战风险 | 深耕美业口径、行业数据资产、终端任务闭环 |

## 生成 PPT 时的建议

1. 前 6 页不要堆功能，先讲清楚“为什么门店需要经营智能体”。
2. 产品能力页要用闭环图，不要只放模块列表。
3. 商业模式页要突出阶段性：早期卖项目交付和终端服务，后期再卖绩效分成和供应链生态。
4. 数字员工页要避免直接说“AI 工时 + 提成”，建议表达为“值守服务费 + 可归因绩效分成”。
5. 财务预测页如果要上正式路演，需要补最新真实样板店数据、渠道 CAC、交付成本和续费假设。
6. 市场规模和竞品数据建议在生成正式 PPT 前做一次外部数据核验，避免沿用旧版未核验数字。
